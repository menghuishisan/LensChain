# LensChain 学生实验功能测试报告

## 测试概述

本报告记录了学生端实验功能的测试发现，基于页面点击操作验证实验功能的正确性。

---

## 发现的问题

### 1. 仿真引擎服务未运行 🔴 高优先级

**问题描述:**
- 启动实验后，实验状态显示为"异常"
- 仿真会话创建失败，错误信息显示仿真引擎服务无法连接
- 错误信息: "创建仿真会话失败: 错误码: 50010, 消息: 仿真会话创建失败: rpc error: code = Unavailable desc = connection error: desc = \"transport: Error while dialing: dial tcp 127.0.0.1:50061: connectex: No connection could be made because the target machine actively refused it.\""
- 仿真功能无法使用（显示"可视化内容还在准备中，请稍后刷新查看"）

**影响位置:**
- 学生实验详情页 (`/student/experiment-instances/{id}`)
- 仿真面板
- 所有需要仿真引擎的实验

**问题原因:**
- SimEngine 服务未在 127.0.0.1:50061 端口运行
- 仿真引擎服务可能未启动或端口配置错误
- 后端尝试连接仿真引擎时被拒绝连接

**建议修复:**
- 启动 SimEngine 服务，确保其在 127.0.0.1:50061 端口监听
- 检查 SimEngine 配置文件，确认端口配置正确
- 添加仿真引擎服务健康检查
- 改进错误提示，明确告知用户仿真引擎未运行
- 在启动实验前检查仿真引擎服务是否可用

### 2. 实验检查点验证409错误 🔴 高优先级

**问题描述:**
- 点击检查点的"单项验证"按钮时，后端返回409 Conflict错误
- 检查点验证功能无法正常使用

**错误信息:**
```
Failed to load resource: the server responded with a status of 409 (Conflict)
@ http://localhost:8080/api/v1/experiment-instances/{id}/checkpoints/verify
```

**影响位置:**
- 学生实验详情页 (`/student/experiment-instances/{id}` - 检查点tab)
- 点击"单项验证"按钮时

**问题原因:**
- 实验状态为"异常"时，检查点验证可能被拒绝
- 后端可能对实验状态有前置检查，异常状态下不允许验证

**建议修复:**
- 检查后端逻辑，确保实验异常状态下也能进行验证（或给出明确提示）
- 前端应根据实验状态禁用相关按钮或给出提示
- 改进错误提示，明确告知用户为何无法验证

### 3. 实验暂停/恢复功能409错误 🔴 高优先级

**问题描述:**
- 点击"暂停"或"恢复"按钮时，后端返回409 Conflict错误
- 实验暂停/恢复功能无法正常使用

**错误信息:**
```
Failed to load resource: the server responded with a status of 409 (Conflict)
@ http://localhost:8080/api/v1/experiment-instances/{id}/pause
@ http://localhost:8080/api/v1/experiment-instances/{id}/resume
```

**影响位置:**
- 学生实验详情页 (`/student/experiment-instances/{id}`)
- 点击"暂停"或"恢复"按钮时

**问题原因:**
- 实验状态为"异常"时，暂停/恢复操作可能被拒绝
- 后端可能对实验状态有前置检查，异常状态下不允许暂停/恢复

**建议修复:**
- 检查后端逻辑，确保实验异常状态下也能进行暂停/恢复（或给出明确提示）
- 前端应根据实验状态禁用相关按钮或给出提示
- 改进错误提示，明确告知用户为何无法暂停/恢复

### 4. 实验重启按钮导致实验销毁 ✅ 已修复

**问题描述:**
- 之前点击"重启"按钮后，实验状态从"异常"变为"已销毁"
- 重启功能实际执行了销毁操作，与预期不符

**修复状态:**
- ✅ 已修复 - 重启按钮现在会创建新的实验实例，而不是销毁当前实例
- 点击"重启"后，系统会创建新的实验实例（尝试次数+1）

**验证结果:**
- 点击"重启"按钮后，页面导航到新的实验实例
- 新实例显示"尝试第 4 次"，表明重启操作正常工作

### 5. 实验操作历史详情显示问题 ✅ 已修复

**问题描述:**
- 之前实验操作历史页面显示操作记录，但所有记录的详情列都显示"-"
- 用户无法看到具体的操作详情信息

**修复状态:**
- ✅ 已修复 - 操作历史详情现在正常显示
- 详情列显示完整的操作信息，包括报告ID、快照ID、尝试次数、模板ID等

**验证结果:**
- 操作历史页面现在显示：
  - "报告ID：2052589786241175552 附带文件：false"
  - "快照ID：2052589606683021312 快照类型：5"
  - "尝试次数：3 模板ID：920000000000008006"

### 测试结果总结

**核心问题:**
- ❌ **仿真引擎服务未运行** - SimEngine 服务未在 127.0.0.1:50061 端口运行，导致仿真会话创建失败，实验状态显示"异常"
- 此问题影响所有需要仿真的实验：
  - 混合实验（PoW 挖矿与链观察混合实验）
  - 纯仿真实验（共识机制可视化对比实验）
  - EVM合约混合实验（EVM 合约开发与执行可视化混合实验）
- ❌ **Kubernetes服务未运行** - Kubernetes API 服务未在 127.0.0.1:54682 端口运行，导致命名空间创建失败
- 此问题影响所有真实环境实验（不依赖仿真的实验）：
  - 以太坊本地开发与部署实践
  - Fabric 单人多节点网络搭建实验
  - EVM 全栈 DApp 开发实验
  - 共享链基础设施上的 DApp 部署实验
