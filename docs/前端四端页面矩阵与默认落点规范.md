# 前端四端页面矩阵与默认落点规范

> 状态：✅ 已确认
> 目的：作为前端权限审查、路由治理、导航收敛和后续整改的统一基线文档。
> 依据：`docs/modules/01-08/*/04-前端页面设计.md`

---

## 一、适用范围

本文档只定义前端四个业务端及其页面归属：

- 学生端
- 教师端
- 学校管理员端
- 超级管理员端

同时补充两类非业务端页面：

- 公共未登录页面
- 已登录公共页面

---

## 二、总规则

### 2.1 四端页面归属规则

1. 每个业务页面必须明确归属到一个端，或明确标记为“已登录公共页面”。
2. 业务端页面不得因为组件复用而改变页面归属。
3. 已登录公共页面只允许承载真正跨角色一致的页面，不得承载某一端专属业务页面。
4. 学校管理员是“教师兼任的管理角色”，但“教师端页面”和“学校管理员端页面”仍需严格分开治理，不能用一个页面同时承担两个端的职责。

### 2.2 默认落点规则

四个端均不设置“导航页 / 工作台页 / 角色首页”。

用户登录成功后，直接进入当前端侧边栏第一个功能页：

| 角色 | 默认落点 | 说明 |
|------|----------|------|
| 学生 | `/student/courses` | 进入“我的课程” |
| 教师 | `/teacher/courses` | 进入“课程教学” |
| 学校管理员 | `/admin/users` | 进入“用户管理” |
| 超级管理员 | `/admin/schools` | 进入“学校管理” |

补充约束：

- 不再保留 `/student`、`/teacher`、`/admin`、`/super` 作为独立工作台页面。
- 如保留这些路由，也只能作为无状态跳转壳，立即重定向到各自默认落点。

### 2.3 公共页面规则

以下页面允许作为公共页存在：

- 未登录公共页：登录、SSO、入驻申请、申请查询
- 已登录公共页：个人中心、修改密码、消息中心、消息详情、通知偏好

以下页面不得放入公共页：

- 课程讨论区、课程公告页、帖子详情页以外的业务页
- 任何教师专属、学生专属、学校管理员专属、超级管理员专属页面

## 九、后续审查口径

后续前端审查、重构和权限整改时，统一按以下口径执行：

1. 先看页面是否属于正确的端。
2. 再看该端路由 `layout` 是否只放行正确角色。
3. 再看页面级 `PermissionGate` 是否与文档一致。
4. 再看共享页面是否被错误放宽到管理端。
5. 最后再看组件复用是否引入职责混杂。

---

## 十、整改目标

基于本规范，前端后续整改应达到以下状态：

- 四端不再存在独立工作台页。
- 登录后直接进入各端第一个业务页。
- 课程、实验、CTF、成绩、通知等页面按端严格收口。
- “共享页面”只保留真正跨角色一致的页面。
- 学校管理员与教师的职责边界在路由、导航、页面和组件层同时清晰。
