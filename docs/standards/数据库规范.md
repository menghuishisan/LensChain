# 数据库设计规范

> 本规范适用于链镜平台所有数据库表设计，所有模块必须严格遵守。

---

## 一、命名规范

| 项目 | 规范 | 示例 |
|------|------|------|
| 表名 | 小写下划线，复数形式 | `users`、`courses`、`login_logs` |
| 字段名 | 小写下划线 | `created_at`、`phone_number` |
| 主键 | 统一命名 `id` | `id BIGINT PRIMARY KEY` |
| 外键字段 | `关联表单数_id` | `user_id`、`school_id` |
| 索引名 | `idx_表名_字段名` | `idx_users_phone` |
| 唯一索引 | `uk_表名_字段名` | `uk_users_phone` |
| 联合索引 | `idx_表名_字段1_字段2` | `idx_users_school_id_student_no` |

---

## 二、通用字段

每张业务表必须包含以下通用字段：

```sql
id          BIGINT       PRIMARY KEY,          -- 雪花算法生成
created_at  TIMESTAMP    NOT NULL DEFAULT NOW(), -- 创建时间（UTC）
updated_at  TIMESTAMP    NOT NULL DEFAULT NOW(), -- 更新时间（UTC）
deleted_at  TIMESTAMP    NULL                    -- 软删除时间，NULL表示未删除
```

---

## 三、数据类型约定

| 场景 | 类型 | 说明 |
|------|------|------|
| 主键/外键 | `BIGINT` | 雪花算法ID，不使用自增 |
| 短文本（名称等） | `VARCHAR(N)` | 按实际需要指定长度 |
| 长文本（描述等） | `TEXT` | 无长度限制 |
| 枚举/状态 | `SMALLINT` | 配合代码中常量定义，不使用数据库ENUM |
| 布尔值 | `BOOLEAN` | 字段名以 `is_` 前缀 |
| 金额/精确数值 | `DECIMAL(10,2)` | 不使用浮点数 |
| 时间 | `TIMESTAMP` | 统一UTC存储 |
| JSON数据 | `JSONB` | PostgreSQL JSONB类型 |

---

## 四、索引设计原则

1. **主键索引**：每张表必须有主键
2. **外键索引**：所有外键字段必须建立索引
3. **查询索引**：高频查询条件字段建立索引
4. **唯一索引**：业务唯一性约束使用唯一索引而非应用层判断
5. **联合索引**：遵循最左前缀原则，高区分度字段在前
6. **避免过度索引**：单表索引数量不超过6个

---

## 五、软删除规范

- 所有业务表使用 `deleted_at` 字段实现软删除
- `deleted_at IS NULL` 表示未删除
- 查询时默认过滤已删除记录（ORM层统一处理）
- 唯一索引需考虑软删除：使用部分索引 `WHERE deleted_at IS NULL`

```sql
-- 示例：手机号唯一索引（仅对未删除记录生效）
CREATE UNIQUE INDEX uk_users_phone ON users(phone) WHERE deleted_at IS NULL;
```

---

## 六、多租户数据隔离

- 涉及学校数据的表必须包含 `school_id` 字段
- 查询时必须带上 `school_id` 条件（中间件层统一注入）
- 超级管理员可跨学校查询

---

## 七、审计字段

关键业务表可选增加审计字段：

```sql
created_by  BIGINT  NULL,  -- 创建人ID
updated_by  BIGINT  NULL   -- 最后修改人ID
```
