# 实验环境模块 — 功能需求说明

> 模块状态：✅ 已确认
> 文档版本：v4.0

## 一、模块概述

实验环境模块是链镜平台的核心模块之一，为区块链教学提供真实的、可操作的实验环境。本模块采用 **三层架构**（镜像层 → 拓扑层 → 实验层），基于 Docker + K8s 容器编排技术，并集成自研 **ChainMirror SimEngine 可视化仿真引擎**，支持教师通过可视化编排器自由组合镜像、拓扑、工具和仿真场景来设计任意实验。

### 核心设计理念

- **平台提供零件库，教师组装实验**：平台不预设实验模板，而是提供完整的镜像库、工具库、仿真场景库，教师根据教学需要自行编排
- **三种实验模式**：纯仿真实验（SimEngine驱动，无需容器）、真实环境实验（容器驱动）、混合实验（容器 + SimEngine联动）
- **场景无限制**：支持有链实验和无链实验（如密码学原理、P2P网络、共识算法原理等）
- **自研仿真引擎**：ChainMirror SimEngine 采用独立微服务 + 场景算法容器化架构，每个仿真场景的算法实现打包为独立容器镜像（10-50MB），通过 gRPC 与 SimEngine Core 通信，前端按领域共享渲染器只负责渲染与交互采集
- **环境编排服务层**：镜像库、编排引擎、容器管理、网络隔离、资源配额、快照等基础设施抽象为独立的公共服务层，供实验模块和CTF竞赛模块（模块05）共同复用

### 三层架构

```
┌─────────────────────────────────────┐
│          实验层（教师定义）            │
│  实验说明 + 步骤引导 + 检查点 + 评分  │
├─────────────────────────────────────┤
│          拓扑层（编排模板）            │
│  单人单节点/单人多节点/多人组网/共享   │
├─────────────────────────────────────┤
│          镜像层（基础组件库）          │
│  链节点/中间件/工具/环境基础 四类镜像  │
└─────────────────────────────────────┘
         ↕ WebSocket 数据通道
┌─────────────────────────────────────────────┐
│         SimEngine 可视化仿真引擎               │
│                                              │
│  前端渲染层：8个领域渲染器 + 动画调度器 + 交互管理器 │
│         ↕ WebSocket                          │
│  SimEngine Core（独立微服务）：                  │
│    会话管理 + 场景调度 + 联动引擎 + 仿真时钟      │
│         ↕ gRPC                               │
│  场景算法容器（按需启停，10-50MB 轻量镜像）        │
└─────────────────────────────────────────────┘
```

## 三、用户故事

### US-01：教师创建实验模板

```
作为教师，我希望通过可视化编排器创建实验模板，
选择实验类型、配置仿真场景和/或容器镜像、编写实验说明、定义检查点，
以便学生可以在仿真或真实环境中完成实验。
```

**验收条件**：
- 可视化编排器支持三种实验类型选择：纯仿真 / 真实环境 / 混合
- 纯仿真实验：可从仿真场景库选择场景、配置联动组、设置场景参数
- 真实环境实验：可选择拓扑模式、从四类镜像库选择镜像并配置参数、从工具库选配工具
- 混合实验：同时配置容器环境和仿真场景，配置数据采集通道（容器端口 → 仿真场景）
- 可编写Markdown格式的实验说明和分步骤引导
- 可定义自动检查点（后端脚本和/或前端状态断言）和手动评分项
- 可配置初始化脚本（真实环境/混合实验）
- 可设置资源限制（CPU/内存/磁盘/时长）
- 保存后平台自动生成K8s编排配置（真实环境/混合实验）

### US-02：学生启动并完成实验

```
作为学生，我希望在课时页面一键启动实验环境，
通过Web终端、IDE、仿真面板等工具完成实验操作，
并通过检查点验证我的实验结果。
```

**验收条件**：
- 纯仿真实验即开即用（<1s），真实环境/混合实验30s-2min内环境就绪
- 环境就绪后自动执行初始化脚本（真实环境/混合实验）
- 可通过实验配置的各种工具操作环境
- 可随时触发检查点验证查看进度
- 可暂停/恢复/重新开始实验
- 提交后自动检查点评分即时反馈

### US-03：教师监控学生实验

```
作为教师，我希望实时查看所有学生的实验进度和环境状态，
必要时可以远程查看学生操作并提供指导。
```

**验收条件**：
- 实时显示每个学生的检查点完成情况
- 显示每个学生的环境状态和资源占用
- 可实时查看学生终端操作（只读）
- 可向学生发送指导消息
- 可强制回收异常环境

### US-04：多人协作组网实验

```
作为教师，我希望设计多人协作实验，
让多个学生各自扮演不同角色（如Org1管理员、Org2管理员、Orderer运维），
各自的容器自动组网形成真实的分布式网络。
```

**验收条件**：
- 教师可定义多个角色及每个角色的容器配置
- 支持三种分组方式：教师手动/学生自选/系统随机
- 学生加入时选择或被分配角色
- 各角色容器自动组网，共享链网络
- 同组学生网络互通，不同组完全隔离
- 支持组级检查点（如整个网络联通验证）和个人检查点
- 组内支持实时聊天和查看组员终端

### US-05：可视化仿真交互

```
作为学生，我希望通过SimEngine仿真面板直观地理解区块链原理，
可以操控节点、注入故障、发起攻击，通过时间控制器控制仿真进度，
观察完整的过程动画，多个仿真场景之间通过共享状态空间自然联动。
```

**验收条件**：
- SimEngine采用独立微服务 + 场景算法容器化架构，算法在后端容器中运行，前端只负责渲染
- 8个领域渲染器按场景所属领域自动匹配，教师和学生不感知渲染器的存在
- 三种时间控制模式：过程化场景支持播放/暂停/单步/回退/变速（0.5x-2x）/重置；交互响应式场景无时间控件；持续运行式场景支持暂停/恢复/变速
- 每个场景有独立的交互操作面板（非通用按钮），操作类型因场景而异
- 所有操作和状态变化以过程化动画呈现，动画与仿真时钟同步
- 同一联动组内的场景共享底层状态对象，一个场景的操作自然影响其他场景的实际仿真结果（联动可选，教师可启用或关闭）
- 支持状态快照和回退，可回到任意历史时刻
- 仿真面板支持拖拽布局、调整大小、全屏查看
- 数据通过内嵌指标卡片、悬停详情、状态变化高亮等方式融入可视化画面

### US-06：超级管理员管理镜像仓库

```
作为超级管理员，我希望管理平台的镜像仓库，
维护官方镜像的版本更新，审核教师上传的自定义镜像，
确保镜像安全可靠。
```

**验收条件**：
- 可上传/更新/下架官方镜像
- 镜像支持多版本共存（如geth:1.13和geth:1.14）
- 教师上传的自定义镜像需经过安全扫描和资源限制检查
- 审核通过后镜像可被教师使用
- 可查看镜像使用统计

## 四、业务规则

### 4.1 镜像管理规则

| 编号 | 规则 | 说明 |
|------|------|------|
| BR-01 | 镜像四类分类 | 链节点镜像、中间件镜像、工具镜像、环境基础镜像 |
| BR-02 | 镜像多版本共存 | 同一镜像支持多个版本，如 geth:1.13、geth:1.14 |
| BR-03 | 官方镜像免审核 | 平台维护的官方镜像直接可用 |
| BR-04 | 自定义镜像需审核 | 教师上传的自定义镜像需管理员审核（安全扫描+资源检查）后方可使用 |
| BR-05 | 镜像不可删除如被引用 | 被实验模板引用的镜像不可下架或删除 |
| BR-06 | 链生态可选 | 实验模板不强制要求链节点镜像，支持无链实验（密码学、P2P等） |

### 4.2 拓扑与编排规则

| 编号 | 规则 | 说明 |
|------|------|------|
| BR-07 | 四种拓扑模式 | 单人单节点、单人多节点、多人协作组网、共享基础设施 |
| BR-08 | 可视化编排器 | 教师通过可视化界面配置，平台自动生成K8s编排配置 |
| BR-09 | 高级教师可查看YAML | 高级教师可查看和微调生成的K8s YAML配置 |
| BR-10 | 网络自动隔离 | 每个实验实例创建独立的K8s Network Policy，实验间完全隔离 |
| BR-11 | 多人组网自动打通 | 同组学生的容器自动组网互通，不同组完全隔离 |
| BR-12 | 初始化脚本支持 | 容器启动后自动执行教师配置的初始化脚本 |

### 4.3 生命周期规则

| 编号 | 规则 | 说明 |
|------|------|------|
| BR-13 | 按需创建 | 学生点击启动时创建容器环境（真实环境/混合实验）或初始化SimEngine（纯仿真实验） |
| BR-14 | 7状态流转 | creating → initializing → running → paused/completed/expired/error，任意状态可强制destroyed |
| BR-15 | 空闲超时回收 | 无操作30分钟自动保存快照并回收容器（超时时间可配置） |
| BR-16 | 最长运行时间 | 超过教师设置的最长运行时间强制快照并回收 |
| BR-17 | 自动快照 | 运行中每5分钟自动保存快照 |
| BR-18 | 快照恢复 | 下次启动时可选择从快照恢复或重新开始 |
| BR-19 | 课程结束强制回收 | 课程状态变为"已结束"时强制回收所有关联实验环境 |
| BR-20 | 暂停与恢复 | 学生可主动暂停实验（保存快照+挂起容器），恢复时从快照继续 |

### 4.4 判题与评分规则

| 编号 | 规则 | 说明 |
|------|------|------|
| BR-21 | 三种判题模式 | 纯自动（检查点脚本全自动）、纯手动（教师评分）、混合（自动+手动加权） |
| BR-22 | 检查点脚本 | 自动检查点通过脚本进入容器执行验证（真实环境），或通过SimEngine状态断言验证（纯仿真），返回通过/失败+得分 |
| BR-23 | 手动评分项 | 教师根据学生操作记录、快照、实验报告手动评分 |
| BR-24 | 混合模式权重 | 混合模式下自动部分和手动部分各占权重，权重由教师配置 |
| BR-25 | 成绩回传课程 | 实验得分自动回传到课程成绩体系，参与加权计算 |
| BR-26 | 多次尝试 | 学生可重新开始实验多次尝试，以最后一次提交为准（或教师配置取最高分） |

### 4.5 资源管理规则

| 编号 | 规则 | 说明 |
|------|------|------|
| BR-27 | 三级资源配额 | 学校级（总CPU/内存/存储）→ 课程级（最大并发/每人上限）→ 实验级（单实验资源限制） |
| BR-28 | 资源超限排队 | 超出课程并发上限时新请求进入排队等待 |

## 六、可视化仿真引擎体系

> 完整的SimEngine架构设计、43个仿真场景详细规格、9个联动组定义、领域渲染器、时间控制模式、gRPC协议、数据采集等详见 **`06-可视化仿真引擎设计.md`**。本节仅概述引擎在功能需求层面的定位。

### 6.1 引擎架构概述

ChainMirror SimEngine 是平台自研的可视化仿真引擎，采用 **独立微服务 + 场景算法容器化** 三层架构：

```
┌─────────────────────────────────────────┐
│         前端渲染层（随平台前端部署）         │
│  8个领域渲染器 + 动画调度器 + 交互管理器    │
│  + 时间控制器（按场景模式条件显示）          │
├─────────────────────────────────────────┤
│         SimEngine Core（独立微服务）        │
│  会话管理 + 场景调度 + 联动引擎 + 仿真时钟  │
│  + 状态管理 + 快照栈 + WebSocket Hub       │
├─────────────────────────────────────────┤
│         场景算法容器（按需启停）             │
│  每个场景独立容器镜像（10-50MB），gRPC通信   │
│  语言无关（Go/Python/Rust/Node）           │
└─────────────────────────────────────────┘
```

- **前端渲染层**：8 个领域渲染器按场景所属领域自动匹配，接收后端推送的状态数据进行渲染，采集用户交互发送操作指令，不包含任何算法实现
- **SimEngine Core**：独立微服务（长驻），包含会话管理、场景调度器、联动引擎、仿真时钟（支持变速/暂停/回退）、事件系统、状态管理、快照栈、WebSocket Hub
- **场景算法容器**：每个仿真场景的算法实现打包为独立容器镜像，通过 gRPC 与 SimEngine Core 通信，由编排服务层按需启停

## 七、实验实例状态流转

```
学生点击启动
     │
     ├── 纯仿真实验 → 直接进入运行中(running)
     │
     ├── 真实环境/混合实验 ↓
     │
     ▼
创建中(creating) ── 失败 → 创建失败(failed)
     │
     ▼
初始化中(initializing) ── 失败 → 创建失败(failed)
     │
     ▼
运行中(running)
     │
     ├── 学生暂停 → 暂停(paused)
     │                    │
     │              学生恢复 → 运行中(running)
     │
     ├── 学生提交/结束 → 已完成(completed)
     ├── 空闲超时/时长超限 → 已过期(expired)
     └── 异常崩溃 → 异常(error)

已完成/已过期/异常 ── 重新开始 → 创建中(creating)
任意状态 ── 强制销毁(教师/管理员) → 已销毁(destroyed)
```
