## 二、模式判定与 4 模式统一布局

### 2.1 4 模式

| 模式 | 编码 | 触发条件 | 主区布局 | 时间控制 | InteractionForm | 联动可视化 | SharedStatePanel |
|---|---|---|---|---|---|---|---|
| **A 单仿真** | `single` | 1 场景 | 单画布全宽 | 单场景控件，⏮ 启用 | 1 个面板 | 无 | 不显示 |
| **B 对照** | `comparison` | 多场景且未启用联动 | grid 1×N（≤4） / focus + 缩略图（≥5） / carousel（≥5 学生选） | 默认全局同步，可切独立时钟；⏮ 全部置灰 | N 个面板（每场景独立） | 单场景 link_indicator（如各成员属于联动组但本实验未启用联动） | 不显示 |
| **C 联动** | `linkage` | 多场景且启用联动组 | 同 B 主区，但禁用 solo 视图 | **强制时钟同步**；⏮ 全部置灰 | N 个面板，触发联动的 action 标 🔗 徽章 | 单场景 link_indicator + cross-canvas-overlay M8 弧线 | 抽屉，顶部 [📊] 触发 |
| **D 混合** | `hybrid` | 单/多仿真 + 容器（experiment_type=3） | 仿真主区 + 独立 Web Terminal Tab（页面级 Tab） | sim 通道走 WS action；container 通道走 Web Terminal | container 通道 action 标 ⛓ 徽章并跳 Web Terminal | 视联动配置而定 | 视联动配置而定 |

### 2.2 模式自动判定规则

模式由三个字段组合判定，**学生不可手切**（教师配实验时已确定形态）：

```typescript
function detectMode(
  experimentType: 1 | 2 | 3,         // 1=纯仿真 2=真实 3=混合
  sceneCount: number,
  hasActiveLinkGroup: boolean
): 'single' | 'comparison' | 'linkage' | 'hybrid' {
  if (experimentType === 3) return 'hybrid'
  if (sceneCount === 1) return 'single'
  if (hasActiveLinkGroup) return 'linkage'
  return 'comparison'
}
```

教师配置实验时，三字段已确定；学生进入实验后模式即固定。教师仍可在干预面板临时切视图（如 focus 单个场景细看），但模式不变。

### 2.3 实验类型 × 模式兼容

| 实验类型 | 可达模式 |
|---|---|
| 纯仿真（type=1） | A / B / C |
| 真实环境（type=2） | 不进入 SimEnginePage |
| 混合（type=3） | D（且仿真侧若多场景启用联动也显示 C 风格的联动可视化） |

## 三、多场景响应式布局

### 3.1 默认布局（grid 模式）

| 场景数 | 默认布局 | 单画布尺寸（视口 1920×1080） |
|---|---|---|
| 1 | 单画布全宽 | 1640×720 |
| 2 | 1×2 横排 | 820×720 |
| 3 | 1×3 横排 | 547×720 |
| 4 | 2×2 网格 | 820×360 |
| **≥5** | **强制 focus 或 carousel**（不做 3×2，避免单画布太挤） | 详 §3.2 / §3.3 |

**grid 单画布最小高度 360px**，低于则降级到 focus 模式。

### 3.2 focus 模式（场景 ≥5 默认）

```
┌──────────────────┬────────┐
│                  │ ☑ 场景 A │
│                  ├────────┤
│                  │ ☐ 场景 B │
│  主聚焦场景        ├────────┤
│  ≥ 720×480      │ ☐ 场景 C │
│                  ├────────┤
│                  │ ☐ 场景 D │
└──────────────────┴────────┘
主区 flex-1               右侧 width 200px
```

- **触发**：grid 模式下双击任一画布进入 focus；URL `?focus=:scene_code`
- **退出**：顶部 [返回 grid] 按钮（不设 ESC 快捷键）
- **缩略图**：仍保持画布渲染（联动因果继续传递），尺寸 100×60，单击切聚焦
- **联动场景的跨画布弧线 M8**：仍指向缩略图（视觉简化但保持因果可见）

### 3.3 carousel 模式（场景 ≥5 学生可选）

```
┌─────────────────────────┐
│ ◀      场景 A         ▶ │
│                          │
│     1/4 ●○○○            │
└─────────────────────────┘
```

- **触发**：场景 ≥5 时学生顶部菜单 [视图: grid / focus / carousel] 切换
- **切换**：[◀] [▶] 按钮 或 鼠标拖动
- 适合"逐个细看不对照"场景

### 3.4 默认视图判定

```typescript
function defaultLayout(sceneCount: number, viewportWidth: number): 'grid' | 'focus' {
  if (sceneCount <= 4 && viewportWidth >= 1280) return 'grid'
  return 'focus'
}
```

### 3.5 尺寸约束表

| 场景 | 容器最小宽 | 容器最小高 |
|---|---|---|
| 单画布全宽（A） | 800 | 480 |
| 横排 2 列（B 对照 2 场景） | 600 | 480 |
| 横排 3 列（B 对照 3 场景） | 500 | 480 |
| 2×2 网格（B/C 4 场景） | 500 | **360**（grid 模式底线） |
| focus 主区（≥5） | 720 | 480 |
| focus 缩略图 | 100 | 60 |

## 四、时间控件 UI

### 4.1 三模式控件清单

| 模式 | 按钮 |
|---|---|
| `process` | ▶ 播放 / ⏸ 暂停 / ⏭ 单步前进 / ⏮ 单步回退 / 速度 0.5× / 1× / 1.5× / 2× / ⟲ 重置 |
| `continuous` | ⏸ 暂停 / 恢复 / 速度 0.5× / 1× / 1.5× / 2× |
| `reactive` | **不显示时间控件**（操作即响应） |

按钮颜色统一使用 shadcn/ui Button 的 outline / default variant，无自定义配色。

### 4.2 ⏮ 单步回退按钮显示规则

| 使用场合 | ⏮ 状态 | 原因 |
|---|---|---|
| A 单仿真 · process | **启用**（高亮 outline） | 学生看错一步可退一步 |
| B 对照 · 任一 process | **置灰** + tooltip "对照模式不支持单步回退" | 多画布独立回退会造成跨画布因果弧线 M8 状态不一致 |
| C 联动 · 任一场景 | **置灰** + tooltip "联动模式不支持单步回退，请重置" | 联动共享状态不可部分回退 |
| D 混合 | **置灰** + tooltip "混合实验不支持单步回退" | 容器侧真实链不可回退 |
| reactive / continuous | **不显示** | 该两模式本就不面向 tick 推进 |

最多可连续点击 ⏮ 20 次（环形快照缓冲）；超出后服务端返回 `buffer_exhausted`，前端 toast 提示"已回退到最早可用快照，请使用 ⟲ 重置"。

### 4.3 速度切换

四档变速按钮：`0.5×` / `1×` / `1.5×` / `2×`。0.5× 为"慢速"，便于详看动画细节。变速只影响 tick 推进速率，不压缩 MicroStep 子步骤的动画时长（详 06.md §5.3）。

### 4.4 1× 基础配速参考

学生应在 1× 速率下能"看清每步又不等太久"。下表为协议层量化（详 06.md §5.3）：

| 模式 | 单 MicroStep（1×） | 单一主流程完成（1×） | 典型场景 |
|---|---|---|---|
| `process` | 1.2-2.0s | 30-50s | PBFT 5 阶段一轮 ≈ 40s |
| `continuous` | 0.6-1.2s | 5-10s | PoW 单出块周期 ≈ 6s |
| `reactive` | 0.3-0.8s | 1-3s | SHA-256 单轮压缩 ≈ 1.5s |

阶段切换过渡 200-300ms 渐变；联动事件 MicroStep 固定 600-1000ms。

## 五、InteractionForm 组件

### 5.1 数据来源

```
HTTP GET /api/v1/experiment-instances/:id/sim-scenes/:scene_code/interaction-schema
       └─ 浏览器缓存 24h
       └─ WS schema_invalidated 信号触发立即作废 + 重新拉取
```

返回 `InteractionDefinition.actions[]`（详 06.md §6.3）。

### 5.2 ActionDef 渲染规则

每个 `ActionDef` 渲染为一个按钮 + 字段组（按 trigger 类型）：

| trigger | 行为 |
|---|---|
| `submit` | 按钮点击展开字段表单 → 学生填字段 → 提交 |
| `immediate` | 按钮点击立即触发，无字段 |
| `hold` | 按住按钮持续触发（如"持续 spam 交易"），松开停止 |

按钮按 `category` 着色：

| category | 颜色 | 图标前缀 | 备注 |
|---|---|---|---|
| `param_tune` | 蓝色 | [⚙ 调参] | 改数值参数 |
| `attack_inject` | 红色 | [⚠ 攻击] | **额外加二次确认弹窗** |
| `primary` | 绿色 | [▶ 操作] | 走正常流程 |
| `observe` | 灰色 | [👁 观察] | 切视图 / 不改状态 |

### 5.3 FieldDef.type → shadcn/ui 映射

| FieldDef.type | shadcn/ui 组件 | 备注 |
|---|---|---|
| `string` | `<Input>` | 单行文本 |
| `number` | `<Input type="number">` | 带 min / max / step |
| `boolean` | `<Switch>` | |
| `select` | `<Select>` | options ≥5 时用 |
| `enum` | `<RadioGroup>` | options ≤4 时优先 |
| `range` | `<Slider>` + 当前值 Label | 调比例 / 调权重 |
| `multi_select` | `<Checkbox>` 组（横向） | 多选；options ≥8 时改 `<Select multiple>` |
| `json` | `<Textarea>` + 客户端 `JSON.parse` 校验 | 复杂数据 |

`options_from: "state.nodes"` 等动态选项：前端从 `RenderEnvelope.data` 或 primitives 列表实时读取，每次打开字段重新解析。

### 5.4 联动徽章与混合通道徽章

按钮上叠加角标：

| 徽章 | 触发条件 | 视觉 |
|---|---|---|
| 🔗 链 | `writes_owned_fields` 非空 | 按钮右上角小图标 + tooltip 显示写入的联动字段列表 |
| ⛓ 真链 | `hybrid_channel = "container"` | 按钮右上角金色图标 + tooltip "执行后跳转 Web Terminal" |

### 5.5 冷却 (cooldown_ms)

按钮触发后若有 `cooldown_ms`，按钮置灰显示倒计时（环形进度），到时恢复。

### 5.6 多场景模式下的位置

| 模式 | InteractionForm 位置 |
|---|---|
| A 单仿真 | 主画布下方折叠面板 |
| B / C grid | **每场景画布下方各自独立** InteractionForm |
| B / C focus | 仅主聚焦场景显示 InteractionForm；缩略图区不显示 |
| B / C carousel | 当前展示场景下方显示 |
| D 混合 sim 部分 | 同 A / B / C 规则 |
| D 混合 container 通道 | 不在 InteractionForm 渲染，跳 Web Terminal Tab |

`actions=0` 时 InteractionForm 整体折叠隐藏。

## 六、联动可视化 UI

### 6.1 单场景画布内（A / B / C / D 通用）

| 原语 | 渲染规则 |
|---|---|
| `link_indicator`（layer=overlay） | 画布右上角徽章（24×24px）；颜色按 LinkGroup 配色：attack 系=红 / network 系=蓝 / crypto 系=青 / economic 系=金 / consensus 系=紫；status: idle=半透明 / active=持续亮 / recent=1 秒高亮脉冲 |
| `external_event_marker`（layer=overlay） | 临时浮窗（120×40px），标注"刚刚因联动场景 X 的事件而更新"，按 `fade_ms` 自动 fade out |
| 多组归属（N:M） | 多个 `link_indicator` 在画布右上角并排或堆叠，每组一个 |

### 6.2 跨画布因果弧线（cross-canvas-overlay，仅 C 联动 + 多场景同屏启用）

```
MultiSceneContainer
  └─ <svg class="cross-canvas-overlay">  全局叠加层
      └─ <path class="link-arc">  贝塞尔弧线
      └─ <polygon class="link-arrow">  头部箭头
      └─ <text class="link-label">  短文字标签（如 "↗ hashrate 0.55"）
```

**绘制流程**：

1. WS 接收 `RenderEnvelope.link_triggers[]` 含 `source_anchor_id` / `target_anchor_id`
2. 检测 `source_scene` 是否在当前并排视图（grid / focus 主区与缩略图均算）
3. 若在：从 `source_anchor_id` 对应原语的 DOM 几何中心拉弧线到 `target_anchor_id` 几何中心
4. 弧线持续 1.5-2s 后 fade out
5. 若 `source_scene` 不在并排视图：降级为单场景 `external_event_marker`

**配色按 LinkGroup 类目**：attack=红 / network=蓝 / crypto=青 / economic=金 / consensus=紫 / blockchain-integrity=灰。

### 6.3 SharedStatePanel 抽屉（仅 C 联动模式）

```
顶部 [📊 共享状态] 按钮 → 右侧抽屉 slide-in（width 360px）

┌──────────────────────────┐
│ pbft-attack-group  ▼     │
│ ┌──────────────────────┐ │
│ │ view_number      0    │ │  owner: pbft-consensus
│ │ byzantine_nodes  []   │ │  owner: pbft-byzantine
│ │ partition_active false│ │  owner: network-partition
│ └──────────────────────┘ │
│                          │
│ pow-attack-group  ▶      │  （折叠）
└──────────────────────────┘
```

- 每个 LinkGroup 一个手风琴 section
- 每字段一行：字段名 / 当前值 / owner 场景标签
- 字段值变化 1 秒黄色高亮闪烁
- 多组归属（N:M）每组独立 section

**默认显隐**：教师默认抽屉可见；学生默认隐藏（教师可在实验配置开放给学生）。

### 9.1 用途与归属

> **本节是 P-24 学生实验监控面板（`04-前端页面设计.md` §P-24）在"实验含 SimEngine 仿真场景"时的增强视图**。

>
> 路由保持 `/teacher/courses/:id/experiment-monitor`（与 P-24 一致）；视图切换按钮在顶部 [📊 表格] / [🖼 缩略图]，缩略图视图为含仿真实验的默认。

### 9.2 用途

教师课堂上同时观察全班学生（最多 60 人）的实验进度、异常情况，并支持远程协助 / 干预。

### 9.3 三列布局

```
┌────────────┬───────────────────────────────┬──────────────┐

│ width 240  │ flex-1                        │ width 360    │
│            │                               │              │
│ ☐ 张三 🟢  │ ┌──┐ ┌──┐ ┌──┐ ┌──┐         │ 张三 / 学号    │
│ ☐ 李四 🟢  │ │张三│ │李四│ │王五│ │赵六│         │ 当前 tick: 32 │
│ ☐ 王五 🟡  │ └──┘ └──┘ └──┘ └──┘         │ 阶段：Prepare │
```

### 9.4 学生列表（左列）

- 每行：☐ 选择框 + 姓名 + 状态点（🟢 正常 / 🟡 落后 / 🔴 异常 / ⚫ 离线）
- 多选 + 顶部 [批量干预] 按钮 → 弹出干预面板（同 §7.2 但作用于多人）
- 排序：默认按状态降序（🔴→🟡→🟢→⚫），可切换姓名 / 进度排序
- 筛选：全部 / 异常 / 已完成 / 落后 30+ tick

### 9.5 缩略图网格（中列）

#### 9.5.1 布局

- 自适应列数：视口 ≥1920 取 5 列；≥1280 取 4 列；< 1280 取 3 列
- 每个缩略图：240×120 px
- 顶角徽章：当前 tick / 阶段；右下角故障图标（如有 `error_overlay` 原语）
- 异常学生：红边框 2px + 顶部 toast “学生 X 实验异常”
- 双击缩略图 = 选中学生 + 进入详情面板

#### 9.5.2 推送模型（快照轮询 + 事件推送混合）

不使用高 fps 连续帧（减带宽与需要并不匹配），采用二者结合：

| 通道 | 频率 | 负载包内容 |
|---|---|---|
| **状态快照轮询**（默认 channel） | 每 2 秒 1 帧（0.5 fps 但使用静态快照不是动画） | `RenderEnvelope.primitives` 子集（仅 layer ∈ {background, content}，不下发动画 / 联动 / 教师附加层） + tick / 阶段徽章 |
| **事件推送**（高优先 channel） | 事件驱动（实时） | `error_overlay` 出现 / `student_alert` / `student_join` / `student_leave` / 进入阶段跳转 |

**为什么不是 fps**：缩略图 论上实 就是“带东西在动”的静态预览，教师需要的是“进度是否推进”与“异常是否出现”，不需看动画。快照轮询 + 事件推送两者结合，既避免 1 fps 动画“卡顿不清晰”，又保证异常即时跟进。

**带宽估算**：60 学生 × 快照 5 KB / 帧 × 0.5 fps ≈ 150 KB/s，可接受。

**选中学生后详情面板（右列）**则启用全量 WS 推送（同学生本人频率），仅针对 1 个学生，负载可忽。

**高负载限定**：同屏学生 ≥ 80 人时，服务端自动降级为 4 秒一帧（0.25 fps）；学生 ≥ 120 人时提示教师切换为表格视图。

### 9.6 详情面板（右列）

学生选中后展示：

- 基础信息：姓名 / 学号 / 当前 tick / 时间模式 / 阶段
- 操作历史时间线：最近 10 个 action（action_code / 参数 / 时间）
- 异常日志：error_overlay 历史 + 故障原因
- 干预操作（教师专属）：
  - 📺 远程查看：跳转到学生 SimEnginePage 只读模式（与学生同屏）
  - 💬 私聊：弹出消息框，仅该学生可见
  - ⏭ 帮跨步：调用 `force_step` 干预 action（详 06.md §14.5）
  - ⟲ 强制重置：调用 `force_reset`
  - 🚪 踢出：调用 `kick_student`

### 9.7 顶部全班操作栏

```
┌────────────────────────────────────────────────────────┐
│ 实验X · 学生监控    [📢 广播] [⏸ 集中暂停] [▶ 集中恢复] │
│ 在线 38/40    异常 2    完成 5    平均 tick 24         │
└────────────────────────────────────────────────────────┘
```

- 实时统计：在线 / 异常 / 完成数 / 平均 tick
- 全班干预按钮：广播消息 / 集中暂停所有学生 / 集中恢复（详 06.md §14.5）
- 后端 API：`POST /api/v1/teacher/experiments/:id/intervene`（同 §7.2）

### 9.8 推送通道

```
WS /api/v1/ws/teacher-monitor/:experiment_id?token=<jwt>
```

| 后端 → 前端消息 | 频率 | 说明 |
|---|---|---|
| `student_thumbnail` | 每 2 秒一帧（高负载降为 4 秒） | 单学生静态快照 （§9.5.2） |
| `student_progress` | 每 2 秒批量 | 学生 tick / 阶段更新 |
| `student_alert` | 事件驱动 | 学生异常告警（实时） |
| `student_join` / `student_leave` | 事件驱动 | 上下线 |
| `student_phase_transition` | 事件驱动 | 跳阶段时提前推送，避免轮询间隔错过关键状态 |

干预指令通过 HTTP `POST /api/v1/teacher/experiments/:id/intervene` 下发，不走 WS。

## 十一、自定义仿真场景上传页（教师，P-29）

### 11.1 用途与归属

> **本节是 04-前端 §P-29 自定义仿真场景上传页的展开设计**。
>
> 路由：`/teacher/sim-scenarios/upload`
> 权限：教师（含学校管理员兼任）
>
> 用途：教师按 06.md §14 SDK 扩展规范贡献自定义场景：L1 / L2 通过平台 API 配置，L3 通过 npm 包上传。

### 11.2 三步上传流程

```
┌─────────────────────────────────────────────┐
│ ① 元信息    ② 容器/渲染器    ③ 提交审核        │
│ ●━━━━━━━━━━●━━━━━━━━━━○                    │
└─────────────────────────────────────────────┘
```

### 11.3 步骤 ① 元信息

```
scene_code 命名空间：teacher_<school_id>__<code>
（前端自动加前缀，教师只填 <code> 部分）

中文名：     [______________________________]
英文名：     [______________________________]
描述：      [______________________________]
            [______________________________]
领域分类：   [节点与网络 / 共识过程 / 密码学 / 数据结构 /
             交易生命周期 / 智能合约 / 攻击与安全 / 经济模型 /
             generic 教师扩展 ▾]
难度等级：   [L1 / L2 / L3 ▾]
时间模式：   [process / reactive / continuous ▾]
```

校验规则：
- scene_code 唯一性（命名空间内）
- 平台保留前缀 `pow-` / `pbft-` / `sha-` 等不允许使用
- 教师扩展场景默认归 `generic` 领域，可选其他领域但需说明理由

### 11.4 步骤 ② 容器与渲染器

#### 11.4.1 L1 / L2 模式（无需上传容器）

```
渲染层级：    [● L1 平台原语组合]
              [○ L2 平台 +1 原语函数]
算法逻辑：    [文本框：JavaScript 代码（在沙箱 worker 内执行）]
默认参数：    [文本框：JSON Schema]
ActionDef：   [可视化编辑器：定义按钮 + 字段（与 06.md §6.3 一致）]
```

- L1：完全使用 47 内置原语组合，平台沙箱内执行 JS 算法逻辑
- L2：可声明 1 个原语函数（自定义图元），由平台审核员审 + 平台批准后生效

#### 11.4.2 L3 模式（npm 包上传）

```
渲染层级：    [● L3 npm 自定义渲染器]
npm 包名：    @chainmirror/renderer-<school_id>__<package>
              （前端自动加命名空间前缀）
版本号：      [1.0.0]
.tgz 文件：   [拖拽或点击上传 .tgz]
gRPC 镜像：   [选择已有镜像 / 上传新镜像]
gRPC 端口：   [50051]
资源限额：    CPU [0.5]  Mem [512Mi]
```

- 平台自动 npm install 验证包结构（必须 export 默认渲染器组件）
- gRPC 镜像走 P-28 自定义镜像上传流程独立审核
- 包大小上限 5 MB（平台规则）

### 11.5 步骤 ③ 提交审核

```
┌─────────────────────────────────────────┐
│ 审核流程预估：3-5 个工作日                │
│                                         │
│ 提交内容：                                │
│   - 元信息（scene_code / 中文名 / 领域）  │
│   - 算法逻辑 / npm 包                     │
│   - 平台先行预览（教师可在沙箱启动一次）   │
│                                         │
│ ☑ 我已阅读并同意《教师扩展场景规范》        │
│                                         │
│      [取消]                  [提交审核]  │
└─────────────────────────────────────────┘
```

提交后：
- 写入 `sim_scenarios` 表 status=pending_review
- 超管在 P-04 §10.6 审核中 Tab 看到该条目
- 教师在"我的提交"页查看进度
- 通过：status=published，教师 + 全平台可在 P-21 step ③ 选用
- 退回：status=rejected，附评论，教师可改后重新提交

### 11.6 教师扩展加入既有联动组

L1 / L2 场景如需加入既有联动组（如 `pbft-attack-group`），在元信息步骤勾选目标组：

- 必须实现该组 schema 中的全部字段（owner 检查）
- 提交后由超管审核（不允许影响既有 owner 字段）
- 通过后 `template_sim_scenes.link_group_id` 可关联

## 十二、URL 路由结构

### 12.1 学生路由

```
/student/experiments/:id                          → 实验详情页（含 Tab：仿真/容器终端/日志/作业提交）
/student/experiments/:id?tab=sim                  → 直接打开仿真 Tab
/student/experiments/:id?tab=sim&focus=:scene     → 多场景下进入 focus 模式
/student/experiments/:id?tab=sim&view=carousel    → 强制 carousel 视图
/student/experiments/:id?tab=sim&fullscreen=1     → 全屏（隐藏顶部导航）
```

### 12.2 教师路由

```
/teacher/experiment-templates/create              → P-21 创建实验模板（step ? 中 step=3 进入 SimEngine 子向导，详 §八）
/teacher/experiment-templates/create?step=3&sub=sim-config  → SimEngine 子向导（query 参数，不开独立路由）
/teacher/experiment-templates/:id/edit            → 编辑模板
/teacher/courses/:id/experiment-monitor           → P-24 多学生监控（含仿真时启用 §九 增强视图）
/teacher/experiment-instances/:id/assist          → P-25 远程协助
/teacher/sim-scenarios/upload                     → P-29 自定义场景上传（详 §十一）
/admin/sim-scenarios                              → P-04 场景库管理（详 §十）
```

URL query 参数仅影响视图切换，不改变模式判定（A/B/C/D 由实验配置决定，URL 仅 focus / carousel / fullscreen）。

### 12.3 不同模式 URL 与组件矩阵

| 模式 | URL 默认 | 主区组件 | 侧栏组件 | InteractionForm | SharedStatePanel | annotation | 教师干预 |
|---|---|---|---|---|---|---|---|
| A 单仿真 | `?tab=sim` | 单画布 | 指标 + MicroSteps | 1 个 | ❌ | 教师 only | 教师 only |
| B 对照 | `?tab=sim` 或 `?tab=sim&focus=:scene` | grid / focus / carousel | 指标聚合 | N 个 | ❌ | 教师 only | 教师 only |
| C 联动 | `?tab=sim` 或 `?tab=sim&focus=:scene` | grid / focus + cross-canvas-overlay | 指标 + 联动状态 | N 个（带 🔗 徽章） | ✓（教师默认开） | 教师 only | 教师 only |
| D 混合 | `?tab=sim` 或 `?tab=terminal` | 仿真主区 / Web Terminal Tab | 指标 + 容器健康 | N 个（含 ⛓ 真链徽章） | 视联动配置 | 教师 only | 教师 only |

## 十三、关键决策汇总（LAYOUT-XX）

### 13.1 整体布局（LAYOUT-01..06）

| 决议号 | 决策 | 落地组件 |
|---|---|---|
| **LAYOUT-01** | 三段式默认 + 全屏切换 | TopBar / 主区 / ControlBar / InteractionForm |
| **LAYOUT-02** | 统一页面 + 教师附加层 | `useAuth().role` 切组件，不开两套路由 |
| **LAYOUT-03** | 路径参数 + focus / carousel / fullscreen query | §十二 URL 结构 |
| **LAYOUT-04** | InteractionForm 画布下方折叠 | 单/多场景一致；actions=0 隐藏 |
| **LAYOUT-05** | SharedStatePanel 顶部按钮触发右侧抽屉 | 默认折叠；多场景模式下侧栏自动让位 |
| **LAYOUT-06** | annotation 工具栏左侧浮动竖排 + 顶部 [✏️] 按钮 toggle | 教师 only；学生 role 隐藏；**不设快捷键** |

### 13.2 模式与控制（LAYOUT-07..13）

| 决议号 | 决策 | 补充说明 |
|---|---|---|
| **LAYOUT-07** | 三字段（experiment_type / sceneCount / hasActiveLinkGroup）自动判定模式，学生不可手切 | 教师配实验时已确定形态 |
| **LAYOUT-08** | C 联动模式禁用 solo（单场景独占视图） | solo 会暂停其他场景 → 联动因果断 → 与学习初衷违背 |
| **LAYOUT-09** | B 对照默认全局同步 + 独立控制 toggle | 学生对比研究有时需独立调步 |
| **LAYOUT-10** | C 联动禁用独立控制；教师紧急解锁能力放干预面板 | 联动组规则强制同步；调试 / 应急开后门 |
| **LAYOUT-11** | D 混合容器操作走独立 Web Terminal Tab | 职责清晰，复用现有组件 |
| **LAYOUT-12** | 教师 SharedStatePanel 抽屉默认可见 / 学生默认隐藏 + 教师可配置开放 | 防止学生被"剧透"联动结果 |
| **LAYOUT-13** | 多学生监控复用 P-24 路由 `/teacher/courses/:id/experiment-monitor`，含仿真时启用增强视图 | 仿真增强视图 = 缩略图网格 + 三列布局（§九） |

### 13.3 多场景视觉优化（LAYOUT-14..19）

| 决议号 | 决策 | 落地 |
|---|---|---|
| **LAYOUT-14** | 5+ 场景不做 3×2，强制 focus / carousel | §3.1 响应式表 |
| **LAYOUT-15** | grid 模式单画布 `min-h-[360px]` 底线 | §3.5 尺寸约束 |
| **LAYOUT-16** | focus 模式：主区 + 右侧缩略图竖排；双击进入 / [返回 grid] 按钮退出 | §3.2；**不设快捷键** |
| **LAYOUT-17** | carousel 模式：场景 ≥5 时学生可选；[◀] [▶] 按钮切换 | §3.3 |
| **LAYOUT-18** | 默认视图判定函数：场景数 + 视口宽度联合决定 | §3.4 |
| **LAYOUT-19** | focus 下联动场景 cross-canvas-overlay 弧线仍指向缩略图 | §3.2 |
