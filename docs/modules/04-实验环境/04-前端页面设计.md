**向导步骤条件逻辑：**

实验类型选择后，步骤导航栏根据类型动态调整显示的步骤：

| 步骤 | 纯仿真 | 真实环境 | 混合 |
|------|--------|---------|------|
| ① 基本信息与类型选择 | 显示 | 显示 | 显示 |
| ② 镜像编排 | **跳过**（灰色不可点击） | 显示 | 显示 |
| ③ 工具与仿真场景 | 显示（仅仿真场景区域） | 显示（仅工具区域） | 显示（工具+仿真场景+数据采集通道） |
| ④ 实验说明 | 显示（无初始化脚本区域） | 显示 | 显示 |
| ⑤ 检查点与评分 | 显示（仅SimEngine状态断言+手动评分） | 显示（仅脚本检查点+手动评分） | 显示（脚本+状态断言+手动评分） |
| ⑥ 资源与发布 | 显示（无资源限制、无K8s预览） | 显示 | 显示 |

- 纯仿真实验选中后，步骤②变为灰色不可点击，步骤导航自动从①跳到③
- 切换实验类型时，弹出确认对话框："切换实验类型将清除已配置的容器/仿真场景，是否继续？"
- 拓扑模式选择区域仅在真实环境/混合实验时显示，纯仿真时隐藏

**镜像配置模板自动填充：**

教师从镜像库点击 `[+]` 添加镜像到编排画布时，平台自动从镜像配置模板填充默认值：
- 默认端口映射（`default_ports`）→ 自动填入容器端口配置
- 默认环境变量（`default_env_vars`）→ 自动填入环境变量列表
- 默认挂载卷（`default_volumes`）→ 自动填入挂载卷配置
- 建议资源（`resource_recommendation`）→ 自动填入 CPU/内存/磁盘限制
- 条件环境变量：当编排画布中已存在触发条件的容器时（如已有 couchdb），自动切换条件变量值并注入关联变量

教师可在容器配置弹窗中修改所有自动填充的默认值。

**典型搭配提示：**

添加镜像后，编排画布顶部根据 `typical_companions` 显示搭配提示：
- **required 级别**：红色提示条 — "⚠ fabric-peer 需要 fabric-orderer 作为依赖，请添加"，同时左侧镜像库中对应镜像显示 🔴 标记
- **recommended 级别**：蓝色提示条 — "💡 推荐为 fabric-peer 搭配 couchdb（支持富查询状态数据库）"
- **optional 级别**：不主动提示，仅在镜像文档侧边栏中展示

## 三、全局交互规范

### 3.1 实验环境状态与页面操作映射

| 实例状态 | 学生可操作 | 教师可操作 |
|---------|-----------|-----------|
| 创建中 | 查看启动进度 | 查看状态 |
| 初始化中 | 查看初始化进度 | 查看状态 |
| 运行中 | 操作终端/IDE/仿真面板、验证检查点、暂停、提交 | 监控、远程协助、发送消息、强制回收 |
| 暂停 | 恢复、重新开始 | 查看状态 |
| 已完成 | 查看结果、查看操作历史、提交报告、重新开始 | 评分、查看操作日志 |
| 已过期 | 查看结果、重新开始 | 查看状态 |
| 异常 | 查看错误信息、重新开始 | 查看错误、强制回收 |
| 已销毁 | 查看历史记录 | 查看历史记录 |
| 创建失败 | 查看错误信息、重试 | 查看错误信息 |

### 3.2 WebSocket 实时更新规范

| 场景 | WebSocket通道 | 更新内容 |
|------|-------------|---------|
| 实验启动进度 | 实例状态通道 | 容器创建进度、初始化进度 |
| 实验运行状态 | 实例状态通道 | 状态变更、空闲警告、时长警告 |
| 检查点验证结果 | 实例状态通道 | 检查点通过/失败、得分更新 |
| 教师指导消息 | 实例状态通道 | 消息内容、发送时间 |
| 教师监控面板 | 监控通道 | 学生状态变更、检查点完成、异常告警 |
| 组内聊天 | 分组聊天通道 | 聊天消息、系统通知 |
| SimEngine仿真数据 | SimEngine数据通道 | 场景状态推送、联动事件、时间控制同步 |
| 远程协助终端 | 终端流通道 | 学生终端实时输出 |

### 3.3 心跳与超时机制

- 前端每60秒向后端发送心跳上报
- 心跳响应中包含剩余时间和空闲警告标志
- 空闲超时前5分钟弹出警告弹窗，用户可点击"继续实验"重置计时
- 时长超限前10分钟弹出警告弹窗，提醒尽快完成并提交
- 超时后自动保存快照并跳转到实验结果页

### 3.4 SimEngine 仿真面板交互规范

- **领域渲染器自动匹配**：8个领域渲染器（节点与网络/共识过程/密码学运算/数据结构/交易生命周期/智能合约/攻击与安全/经济模型）根据场景所属领域自动匹配，教师和学生不感知渲染器的存在
- **三种时间控制模式**：
  - 过程化场景（process）：完整时间控件 — 播放/暂停/单步/回退/变速/重置
  - 交互响应式场景（reactive）：无时间控件，操作即响应
  - 持续运行式场景（continuous）：暂停/恢复/变速
- **变速范围**：0.5x / 1x / 1.5x / 2x 四档
- **场景专属交互面板**：每个场景通过 gRPC GetInteractionSchema 声明交互操作定义，前端根据 schema 动态生成操作面板
- **联动组可视化**：同一联动组内的场景共享底层状态，操作一个场景自然影响其他场景；联动组标识显示在场景面板角标，联动中的场景边框高亮
- **状态快照与回退**：关键帧快照（每50 tick）+ 增量快照（每 tick），支持回退到任意历史时刻
- **数据融入可视化**：仿真数据通过内嵌指标卡片、悬停详情、状态变化高亮等方式融入可视化画面，不设独立数据面板
- 场景面板支持拖拽排列和调整大小（基于栅格系统，12列）
- 支持全屏查看单个场景
- 仿真数据通过 WebSocket 实时推送（SimEngine数据通道）
- 布局配置自动保存到用户偏好

### 3.5 终端交互规范

- Web终端基于 xterm.js 实现
- 支持多容器切换（下拉选择）
- 支持终端分屏（水平/垂直拆分）
- 支持复制粘贴（Ctrl+Shift+C / Ctrl+Shift+V）
- 终端命令自动记录到操作日志
- 教师远程查看时为只读模式

### 3.6 响应式布局

- 实验操作主页（P-41）最低分辨率要求：1280×720
- 面板支持拖拽调整大小
- 支持全屏模式（隐藏顶部导航栏）
- 移动端不支持实验操作页面，仅支持查看实验列表和结果
