② DOM 渲染 4 个（强文本/交互/a11y 必走 DOM）：code_block（语法高亮+复制）/ math_formula（KaTeX）/ register_row（HTML 表格）/ error_overlay（Modal）

③ DOM 浮层 4 个（覆盖在 canvas 之上，不画入 canvas）：label（节点旁文字 a11y）/ tooltip（hover 气泡）/ annotation（教师标注）/ math_pipeline（公式行 + canvas 主流混合）

④ Layout 容器 6 个（自身不绘制，仅为子原语解算坐标）：stack / ring_layout / matrix_layout / horizontal_lane / tree_layout / graph_layout

33 + 4 + 4 + 6 = 47 全覆盖。
内容驱动尺寸：node radius clamp [16, 28]px；水平 step = max(2r+8, label.width+8)；每原语 minH/prefH/maxH 三档；Σprefh > 可用区时压到 minH，反之按权重再分。
零遮挡铁律（layoutSolver + React 强制执行，禁止任何 absolute 覆盖式弹层）：
• 右侧抽屉（SharedState / 干预面板）= position:fixed 滑入，主区 margin-right:+360px 让位，不覆盖画布
• 左侧标注工具栏 atb = 浮动竖排，主画布 padding-left:48px 让位
• InteractionForm = 画布下方 flex 子项，永远在画布之下，不浮动
• ControlBar = panel 最底部 flex 子项，sticky，画布缩放时不被压掉
• tooltip / 节点 label = DOM 浮层，智能避让（首选右侧 → 顶部 → 底部 → 左侧）+ 边界碰撞检测自动翻转
• ErrorOverlay = 居中 Modal + 半透明遮罩（这是唯一允许的全覆盖，因为它本就是阻断性状态）
• 对照模式 focus 切换 = 主舞台与缩略图位置互换，不是浮层；缩略图始终在底部 flex row 中
• 跨画布 M8 弧线（联动模式）= SVG pointer-events:none，不拦截交互

⑥ 5 种播放态 — 状态机与可视化

SimEngine 内置 FSM：idle → running ⇄ paused → finished，外加 error 兜底。每态触发不同的 UI 反馈。

idle · 未启动 running · 运行中 paused · 已暂停 finished · 已完成 error · 异常
状态UI 反馈ControlBar 按钮可用性InteractionForm
idle 画布空白 + 中央 "▶ 点击播放" 大按钮；MicroSteps 空；指标全 0 仅 ▶ 可点；其它灰 所有 action 灰禁用
running 动画流畅；联动徽章脉动；Tick 高频更新 ⏸ ⏭ ⟲ 可点；▶ 灰 调参 / 攻击 / 触发 全部可点；前置满足才亮
paused 画布静止；HUD 顶部黄色 "⏸ 已暂停 · 第 6 步" banner；动画 freeze 在当前帧 ▶ ⏭ ⏮ ⟲ 可点 调参 / 观察可点；动画类攻击灰禁用
finished 画布锁定；右上 ✅ 完成徽章；自动展开"📤 提交报告"对话；MicroSteps 全部为 dn 仅 ⟲ 重置可点 仅 "查看回放 / 下载快照" 两键
error 顶部红色 banner: "⚠ algo-runner exited (code 137) — 内存溢出"；画布半透明遮罩；ErrorOverlay 卡片显示堆栈 仅 ⟲ 重置可点 全灰；仅 "查看日志 / 重启容器" 两键
