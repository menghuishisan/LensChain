# 实验环境模块 — 验收标准

> 模块状态：✅ 已确认
> 文档版本：v3.0

---

## 一、功能验收条件

### AC-01：超级管理员上传官方镜像

```
Given 超级管理员已登录系统
When 填写镜像名称、分类、生态、默认配置，上传至少一个版本并提交
Then 镜像以"正常"状态创建成功，出现在镜像仓库列表中
And 教师可在实验编排时选择该镜像
```

### AC-02：教师上传自定义镜像

```
Given 教师已登录系统
When 填写镜像信息并上传自定义镜像
Then 镜像以"待审核"状态创建，教师可在"我的镜像"中查看
And 教师不可在实验编排中使用该镜像（需审核通过后方可使用）
```

### AC-03：超级管理员审核镜像

```
Given 存在一个待审核状态的教师自定义镜像
When 超级管理员审核通过该镜像
Then 镜像状态变为"正常"，教师可在实验编排中使用
```

```
Given 存在一个待审核状态的教师自定义镜像
When 超级管理员审核拒绝并填写拒绝原因
Then 镜像状态变为"审核拒绝"，教师可查看拒绝原因
```

### AC-04：镜像下架保护

```
Given 某镜像被至少一个实验模板引用
When 超级管理员尝试下架该镜像
Then 系统拒绝操作，提示"该镜像被N个实验模板引用，不可下架"
```

```
Given 某镜像未被任何实验模板引用
When 超级管理员下架该镜像
Then 镜像状态变为"已下架"，不再出现在教师的镜像选择列表中
```

### AC-05：教师创建实验模板（完整流程）

```
Given 教师已登录系统
When 通过可视化编排器完成以下步骤：
  1. 选择实验类型（纯仿真/真实环境/混合）
  2. 填写基本信息，选择拓扑模式（真实环境/混合实验时）
  3. 从镜像库选择镜像并配置容器参数（真实环境/混合实验时）
  4. 选配仿真场景、配置联动组和场景参数（纯仿真/混合实验时）
  5. 编写实验说明（Markdown）和初始化脚本
  6. 定义检查点（自动脚本/SimEngine状态断言+手动评分项）并配置分值
  7. 设置资源限制，点击"保存草稿"
Then 实验模板以"草稿"状态保存成功
And 真实环境/混合实验：平台自动生成K8s编排配置
And 纯仿真实验：无需K8s编排配置
```

```
Given 教师拥有一个草稿状态的实验模板，已配置至少一个容器或一个仿真场景，且至少一个检查点
When 教师点击"发布"
Then 实验模板状态变为"已发布"，可被课时关联使用
```

```
Given 教师拥有一个草稿状态的实验模板，未配置任何容器且未配置任何仿真场景
When 教师点击"发布"
Then 系统拒绝发布，提示"请至少配置一个容器或一个仿真场景"
```

### AC-06：实验模板编辑限制

```
Given 实验模板状态为"草稿"
When 教师编辑模板的任意配置
Then 修改保存成功
```

```
Given 实验模板状态为"已发布"且未被任何课时引用
When 教师编辑模板
Then 修改保存成功
```

```
Given 实验模板状态为"已发布"且已被课时引用
When 教师编辑模板的基本信息、实验说明、检查点
Then 修改保存成功（不影响已启动的实验实例）
```

### AC-07：实验模板删除

```
Given 实验模板未被任何课时引用
When 教师点击删除
Then 模板被软删除，不再出现在列表中
```

```
Given 实验模板已被课时引用
When 教师尝试删除
Then 系统拒绝操作，提示"该模板已被课时引用，不可删除"
```

### AC-08：实验模板克隆

```
Given 教师查看自己的实验模板或共享实验库中的模板
When 点击"克隆"
Then 系统创建新的草稿模板，包含：基本信息、实验类型、容器配置、仿真场景配置、检查点、初始化脚本、标签
And 不包含：使用统计、共享状态
And 新模板与原模板完全独立
```

### AC-09：实验模板共享

```
Given 教师的实验模板状态为"已发布"
When 教师开启"共享到平台实验库"
Then 模板出现在共享实验库中，其他教师可浏览和克隆
```

```
Given 实验模板状态为"草稿"
When 教师尝试开启共享
Then 系统拒绝，提示"仅已发布的模板可共享"
```

### AC-10：学生启动实验环境

```
Given 学生已加入课程，课时关联了已发布的纯仿真实验模板
When 学生点击"启动实验环境"
Then 系统创建实验实例，SimEngine Core 初始化会话并按需启动场景算法容器
And 实验即开即用（<1s），学生可通过SimEngine仿真面板操作
```

```
Given 学生已加入课程，课时关联了已发布的真实环境/混合实验模板
When 学生点击"启动实验环境"
Then 系统创建实验实例，状态依次流转：创建中 → 初始化中 → 运行中
And 环境就绪后自动执行初始化脚本
And 学生可通过Web终端/IDE/仿真面板操作环境
And 整个启动过程在30秒-2分钟内完成
```

```
Given 学生之前有暂停的实验实例
When 学生选择"从快照恢复"启动
Then 系统从最新快照恢复环境，状态变为运行中
And 环境恢复到暂停前的状态
```

### AC-11：实验环境资源限制

```
Given 学生已有2个运行中的实验（达到个人并发上限）
When 学生尝试启动第3个实验
Then 系统拒绝，提示"已达个人并发实验上限"
```

```
Given 课程并发实验数已达上限
When 新学生尝试启动实验
Then 系统将请求加入排队队列，显示排队位置和预计等待时间
And 资源释放后自动创建排队中的实验
```

```
Given 学校资源配额已用尽
When 学生尝试启动实验
Then 系统拒绝，提示"学校资源配额不足"
```

### AC-12：实验暂停与恢复

```
Given 学生的实验实例状态为"运行中"
When 学生点击"暂停"
Then 系统自动保存快照，挂起容器，实例状态变为"暂停"
```

```
Given 学生的实验实例状态为"暂停"
When 学生点击"恢复"
Then 系统从快照恢复容器，实例状态变为"运行中"
And 环境恢复到暂停前的状态
```

### AC-13：实验重新开始

```
Given 学生的实验实例状态为"已完成"/"已过期"/"异常"
When 学生点击"重新开始"
Then 系统销毁旧环境，创建全新实验实例
And attempt_no 递增
And 新实例从初始状态开始
```

### AC-14：检查点验证

```
Given 学生的实验实例状态为"运行中"
When 学生点击"验证检查点"（指定某个自动检查点）
Then 系统在目标容器中执行检查脚本
And 返回通过/失败结果和得分
And 结果实时显示在检查点面板中
```

```
Given 学生点击"验证所有检查点"
When 系统执行所有自动检查点脚本
Then 返回每个检查点的通过/失败结果和得分
And 更新总得分
```

### AC-15：提交实验

```
Given 学生的实验实例状态为"运行中"
When 学生点击"提交实验"
Then 系统触发所有自动检查点的最终验证
And 自动评分即时反馈
And 手动评分项标记为"待评分"
And 实例状态变为"已完成"
And 成绩回传到课程成绩体系
```

### AC-16：空闲超时回收

```
Given 学生的实验实例运行中，空闲超时设置为30分钟
When 学生无任何操作超过25分钟
Then 系统通过WebSocket推送空闲超时警告
```

```
Given 学生收到空闲超时警告后仍无操作
When 空闲时间达到30分钟
Then 系统自动保存快照并回收容器
And 实例状态变为"已过期"
```

### AC-17：最长运行时间限制

```
Given 实验模板设置最长运行时间为120分钟
When 实验运行时间达到110分钟
Then 系统通过WebSocket推送时长超限警告（剩余10分钟）
```

```
Given 实验运行时间达到120分钟
When 学生未主动提交
Then 系统自动触发所有检查点验证，保存快照并回收容器
And 实例状态变为"已过期"
```

### AC-18：自动快照

```
Given 学生的实验实例状态为"运行中"
When 每5分钟定时任务触发
Then 系统自动保存增量快照
And 快照记录出现在快照列表中
```

### AC-19：教师监控学生实验

```
Given 教师进入课程实验监控面板
When 页面加载完成
Then 实时显示每个学生的实验状态、检查点进度、资源占用
And 通过WebSocket实时更新
And 异常实例高亮显示并置顶
```

### AC-20：教师远程协助

```
Given 教师在监控面板点击某学生的"协助"按钮
When 进入远程协助页
Then 教师可实时查看学生终端操作（只读，不可输入）
And 教师可发送指导消息，消息实时推送到学生实验界面
And 教师可切换查看不同容器的终端
```

### AC-21：教师手动评分

```
Given 学生已提交实验，存在手动评分项
When 教师对每个手动评分项逐项评分（给分+评语），提交评分结果
Then 手动评分完成，总分自动计算
And 学生可查看评分结果和教师评语
And 成绩回传到课程成绩体系
```

```
Given 判题模式为"混合"，自动权重60%，手动权重40%
When 自动部分得分60/60，手动部分得分35/40
Then 总分 = (60/60 × 60%) + (35/40 × 40%) = 60% + 35% = 95分（满分100时）
```

### AC-22：教师强制回收环境

```
Given 教师在监控面板发现异常或超时的学生实验
When 教师点击"强制回收"并确认
Then 系统强制销毁该实验环境
And 实例状态变为"已销毁"
```

### AC-23：多人协作实验 — 教师手动分组

```
Given 教师创建了多人协作拓扑的实验模板，定义了3个角色
When 教师选择"教师手动分组"，创建分组并分配学生和角色
Then 分组创建成功，每个学生被分配到指定角色
```

### AC-24：多人协作实验 — 学生自选分组

```
Given 教师选择"学生自选分组"，创建了空分组
When 学生进入分组页面，选择一个分组和角色并加入
Then 学生成功加入分组，角色分配完成
```

```
Given 某分组已满
When 学生尝试加入该分组
Then 系统拒绝，提示"该分组已满"
```

```
Given 学生已在某分组中
When 学生尝试加入另一个分组
Then 系统拒绝，提示"您已在其他分组中"
```

### AC-25：多人协作实验 — 系统随机分组

```
Given 教师选择"系统随机分组"，设置每组人数
When 教师点击"随机分组"
Then 系统根据课程学生名单自动随机分组
And 角色随机分配
And 教师可查看分组结果并手动调整
```

### AC-26：多人协作实验 — 组网与隔离

```
Given 多人协作实验分组已就绪，所有组员启动实验
When 同组学生的容器全部启动完成
Then 同组学生的容器自动组网互通
And 不同组的容器完全网络隔离
```

### AC-27：多人协作实验 — 组内通信

```
Given 同组学生都在实验中
When 学生在组内聊天中发送消息
Then 消息通过WebSocket实时推送给所有组员
And 消息记录持久化保存
```

```
Given 同组学生都在实验中
When 学生查看组员终端
Then 可以只读方式查看组员的终端操作
```

### AC-28：多人协作实验 — 组级检查点

```
Given 实验模板定义了组级检查点（如"全网络联通验证"）
When 任一组员触发组级检查点验证
Then 系统在组级网络中执行验证脚本
And 结果对所有组员可见
```

### AC-29：SimEngine 仿真场景交互

```
Given 学生的实验包含仿真场景
When 学生在SimEngine仿真面板中进行交互操作（如宕掉节点、注入攻击、调整参数）
Then 操作通过WebSocket发送到SimEngine Core
And SimEngine Core 将操作指令通过gRPC转发到对应的场景算法容器
And 场景算法容器计算新状态并返回
And 新状态通过WebSocket推送到前端，领域渲染器更新可视化画面
And 同一联动组内的其他场景自然受到影响（如联动已启用）
```

### AC-30：SimEngine 仿真面板布局

```
Given 学生在实验操作页面的SimEngine仿真面板中
When 拖拽场景面板调整位置或大小
Then 布局立即更新（基于栅格系统，12列）
And 布局配置自动保存到用户偏好
And 支持全屏查看单个场景
```

### AC-31：SimEngine 时间控制

```
Given 仿真场景的时间控制模式为"过程化"（process）
When 学生操作时间控制器
Then 支持播放/暂停/单步/回退/变速（0.5x/1x/1.5x/2x）/重置
And 所有操作和状态变化以过程化动画呈现，动画与仿真时钟同步
```

```
Given 仿真场景的时间控制模式为"交互响应式"（reactive）
When 学生在场景操作面板中执行操作
Then 操作即响应，无时间控制器显示
```

```
Given 仿真场景的时间控制模式为"持续运行式"（continuous）
When 学生操作时间控制器
Then 支持暂停/恢复/变速（0.5x/1x/1.5x/2x），无回退和单步
```

### AC-32：超级管理员全局资源监控

```
Given 超级管理员进入全局资源监控页
When 页面加载完成
Then 显示全平台运行实例数、K8s集群状态、各学校资源使用率
And 显示异常实例告警列表
And 超级管理员可强制回收任意异常实例
```

### AC-33：学校资源配额管理

```
Given 超级管理员为某学校分配了资源配额
When 学校管理员查看本校资源配额页
Then 显示CPU/内存/存储/并发的配额和使用情况
And 学校管理员可为本校课程分配课程级配额
```

```
Given 课程级配额的并发上限超过学校级剩余可分配量
When 学校管理员尝试保存
Then 系统拒绝，提示"课程配额不能超过学校剩余可分配量"
```

### AC-34：实验统计

```
Given 教师进入课程实验统计页
When 页面加载完成
Then 显示每个实验模板的统计数据：
  - 完成率、平均得分、最高分、最低分
  - 平均用时、平均尝试次数
  - 各检查点通过率
  - 分数分布（90-100/80-89/70-79/60-69/60以下）
```

### AC-35：实验标签管理

```
Given 教师在实验模板编辑页
When 设置标签（链生态/实验类型/难度/自定义）
Then 标签保存成功
And 在实验模板列表和共享实验库中可按标签筛选
```

### AC-36：实验报告提交

```
Given 学生的实验需要提交实验报告
When 学生编写Markdown报告或上传PDF/Word文件并提交
Then 报告保存成功，教师可在评分页查看
```

```
Given 学生上传的报告文件超过50MB
When 提交报告
Then 系统拒绝，提示"报告文件不能超过50MB"
```

### AC-37：操作历史查看

```
Given 学生查看自己的实验操作历史
When 页面加载完成
Then 显示完整的操作记录：终端命令、检查点验证、状态变更等
And 支持按操作类型和容器筛选
```

```
Given 教师查看学生的操作历史
When 页面加载完成
Then 显示该学生的完整操作记录（用于评分参考和教学分析）
```

### AC-38：成绩回传课程

```
Given 学生完成实验并获得最终得分
When 实验关联了课程课时
Then 实验得分自动回传到课程成绩体系
And 在课程成绩权重配置中作为独立项参与加权计算
```

### AC-39：成绩策略

```
Given 实验模板成绩策略为"取最后一次提交"
When 学生多次尝试并提交
Then 以最后一次提交的得分为最终成绩
```

```
Given 实验模板成绩策略为"取最高分"
When 学生多次尝试并提交
Then 以所有提交中的最高分为最终成绩
```

### AC-40：课程结束环境回收

```
Given 课程状态变为"已结束"
When 定时任务检测到该课程仍有运行中的实验环境
Then 系统强制保存快照并回收所有关联实验环境
```

### AC-41：仿真场景审核

```
Given 教师通过SimScenario SDK开发并上传了自定义仿真场景（容器镜像）
When 超级管理员审核通过
Then 场景状态变为"正常"，教师可在实验编排中使用
And 场景算法容器镜像注册到平台镜像仓库
```

### AC-42：SimEngine 联动组

```
Given 教师在实验模板中配置了联动组（如"PBFT攻击组"包含PBFT共识+拜占庭攻击+网络分区）
And 教师启用了联动
When 学生在PBFT共识场景中注入拜占庭节点
Then 拜占庭攻击场景中自动反映该节点的异常行为
And 网络分区场景中显示受影响的消息传播变化
And 联动通过共享底层状态对象实现，非消息传递
```

```
Given 教师在实验模板中关闭了联动
When 学生操作某个场景
Then 其他场景不受影响，各场景独立运行
```

## 三、安全要求

| 要求 | 说明 |
|------|------|
| 容器网络隔离 | 每个实验实例创建独立的K8s Network Policy，实验间完全网络隔离 |
| 多人实验组间隔离 | 同组容器互通，不同组完全隔离，通过K8s Network Policy实现 |
| 容器资源限制 | 通过K8s ResourceQuota和LimitRange限制CPU/内存/磁盘，防止资源滥用 |
| 容器安全策略 | 禁止特权容器、禁止挂载宿主机目录、限制Linux Capabilities |
| 镜像安全扫描 | 教师上传的自定义镜像需经过安全扫描（漏洞检测）后方可使用 |
| 初始化脚本安全 | 初始化脚本在容器内执行，不可访问宿主机资源 |
| 检查点脚本安全 | 检查点脚本在目标容器内执行，有超时限制（默认10秒） |
| 数据隔离 | 学生只能访问自己的实验实例，教师只能访问本课程的实验数据 |
| school_id 隔离 | 所有实验数据带 school_id，跨学校不可访问 |
| 终端命令审计 | 学生终端命令完整记录，支持教师回查 |
| 教师远程协助只读 | 教师查看学生终端为只读模式，不可执行命令 |
| WebSocket 认证 | WebSocket连接需携带JWT Token，验证用户身份和权限 |
| 快照数据加密 | 快照数据存储在对象存储中，传输和存储均加密 |
| 实验报告文件校验 | 上传文件校验类型（PDF/Word）和大小（≤50MB） |
| 接口权限校验 | 所有API校验请求者角色和资源归属关系 |
| 资源配额强制执行 | 三级资源配额（学校/课程/实验）在后端强制校验，不可绕过 |
| 防止容器逃逸 | 使用K8s Pod Security Standards（Restricted级别）防止容器逃逸 |
| API限流 | 心跳上报接口限流（每用户每分钟最多2次），检查点验证限流（每用户每分钟最多10次） |

## 四、边界条件与异常处理

| 场景 | 处理方式 |
|------|---------|
| 实验模板无容器且无仿真场景时发布 | 拒绝发布，提示"请至少配置一个容器或一个仿真场景" |
| 实验模板检查点总分与total_score不一致 | 发布时警告提示，教师可选择继续或修改 |
| 混合模式权重之和≠100% | 拒绝保存，提示"自动权重和手动权重之和必须为100%" |
| 容器镜像拉取失败 | 实例状态变为"创建失败"，记录错误信息，学生可重试 |
| 容器启动失败 | 实例状态变为"创建失败"，记录错误信息，自动清理已创建的容器 |
| 初始化脚本执行超时 | 记录超时日志，实例状态仍变为"运行中"（脚本失败不阻塞实验） |
| 初始化脚本执行失败 | 记录错误日志，实例状态仍变为"运行中"，通知学生脚本执行异常 |
| 检查点脚本执行超时 | 返回"验证超时"，该检查点标记为未通过 |
| 检查点脚本执行异常 | 返回脚本错误输出，该检查点标记为未通过 |
| 容器运行中OOMKilled | 实例状态变为"异常"，记录错误信息，通知教师 |
| 容器运行中CrashLoopBackOff | 实例状态变为"异常"，记录错误信息，通知教师 |
| K8s节点故障 | 受影响的实例状态变为"异常"，K8s自动调度到其他节点恢复 |
| 快照保存失败 | 记录错误日志，不影响实验继续运行，下次定时任务重试 |
| 快照恢复失败 | 提示学生"快照恢复失败"，建议重新开始实验 |
| WebSocket连接断开 | 前端自动重连（指数退避，最大间隔30秒），重连后同步最新状态 |
| 心跳上报网络中断 | 前端本地记录活跃状态，网络恢复后补报，后端容忍短暂心跳丢失 |
| 学生并发启动同一实验 | 后端幂等处理，返回已存在的实例信息 |
| 多人实验部分组员未启动 | 已启动的组员可正常操作，组级检查点需所有组员就绪后方可验证 |
| 多人实验组员中途退出 | 该组员实例暂停，其他组员不受影响，组级检查点暂时不可验证 |
| 教师评分超过检查点满分 | 前端限制输入范围，后端二次校验拒绝 |
| 课程结束时学生正在操作 | 弹出警告"课程即将结束，实验环境将在10分钟后回收"，到时强制回收 |
| 资源配额动态调整 | 调低配额不影响已运行实例，仅影响新启动的实例 |
| 镜像版本更新 | 已运行的实例不受影响，新启动的实例使用新版本 |
| 大量学生同时启动实验 | 通过排队机制削峰，按请求时间顺序处理 |
| 仿真场景算法容器启动失败 | SimEngine Core 记录错误，该场景显示"加载失败"提示，不影响其他场景 |
| 仿真场景算法容器OOM | SimEngine Core 自动重启该场景容器，恢复最近快照状态 |
| SimEngine Core 连接断开 | 前端自动重连，重连后从最近快照恢复仿真状态 |
| 联动组中部分场景未加载 | 已加载的场景正常运行，联动效果在所有场景就绪后生效 |
| 仿真快照栈溢出（超过1000 tick） | 自动淘汰最早的快照，保留最近1000个 tick |
| Collector Agent 采集失败 | 记录错误日志，仿真面板显示"数据采集中断"提示，不影响容器实验操作 |
| 可视化渲染器加载失败 | 显示场景加载失败提示，不影响其他场景和实验操作 |
| 实验报告重复提交 | 覆盖上一次提交的报告内容 |
| 过期快照清理 | 已销毁实例的快照保留30天后自动清理 |
| 镜像配置模板缺失 | 自定义镜像无配置模板时，所有字段留空由教师手动填写 |
| 条件环境变量循环依赖 | 后端检测循环条件依赖，拒绝保存并提示教师 |
| 端口冲突修改后仍冲突 | 实时重新检测，持续显示冲突提示直到解决 |
| L5预检超时 | 预检执行超过10秒自动终止，返回"预检超时，请稍后重试" |
| 镜像预拉取节点离线 | 跳过离线节点，节点恢复后自动补拉 |
| 镜像预拉取磁盘空间不足 | 记录错误，管理后台显示告警，不影响其他节点拉取 |
| 教师强制添加不兼容工具后发布 | 允许发布，但在实验模板详情页显示兼容性提示 |
| 服务发现变量名冲突 | 容器名转换为环境变量名时去除特殊字符，冲突时追加序号 |
| 编排器向导中途切换实验类型 | 清除不适用的配置项，保留通用配置（基本信息、实验说明），弹窗确认 |
