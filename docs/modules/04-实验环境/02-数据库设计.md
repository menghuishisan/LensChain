## 一、ER 关系概览

```
images (镜像主表)
  │
  ├── 1:N ── image_versions (镜像版本)
  │
  └── N:1 ── image_categories (镜像分类)

experiment_templates (实验模板主表)
  │
  ├── 1:N ── template_containers (模板容器配置)
  │             │
  │             └── N:1 ── image_versions (镜像版本)
  │
  ├── 1:N ── template_checkpoints (检查点定义)
  │
  ├── 1:N ── template_init_scripts (初始化脚本)
  │
  ├── 1:N ── template_sim_scenes (模板仿真场景配置)
  │             │
  │             └── N:1 ── sim_scenarios (仿真场景库)
  │
  ├── 1:N ── template_tags (模板标签关联)
  │             │
  │             └── N:1 ── tags (标签表)
  │
  ├── 1:N ── template_roles (多人实验角色定义)
  │
  ├── 1:N ── experiment_instances (实验实例)
  │             │
  │             ├── 1:N ── instance_containers (实例容器)
  │             │
  │             ├── 1:N ── checkpoint_results (检查点结果)
  │             │
  │             ├── 1:N ── instance_snapshots (实例快照)
  │             │
  │             └── 1:N ── instance_operation_logs (操作日志)
  │
  └── 1:N ── experiment_groups (实验分组)
               │
               ├── 1:N ── group_members (分组成员)
               │
               └── 1:N ── group_messages (组内消息)

sim_scenarios (仿真场景库)
  │
  ├── 1:N ── sim_scenario_versions (场景版本历史，semver 锁版)
  │
  └── N:N ── sim_link_groups (联动组定义，schema 不可演进，code+version 双唯一)
              │  via sim_link_group_scenes

teacher_intervene_logs (教师 SimEngine 干预审计)

resource_quotas (资源配额)
```

## 二、表结构设计

### 2.1 image_categories — 镜像分类表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| name | VARCHAR(50) | NOT NULL | 分类名称 |
| code | VARCHAR(50) | NOT NULL, UNIQUE | 分类编码：chain_node / middleware / tool / base_env |
| description | VARCHAR(200) | NULL | 分类描述 |
| sort_order | INT | NOT NULL DEFAULT 0 | 排序序号 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |

**预设分类数据：**

| code | name |
|------|------|
| chain_node | 链节点镜像 |
| middleware | 中间件镜像 |
| tool | 工具镜像 |
| base_env | 环境基础镜像 |

### 2.2 images — 镜像主表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| category_id | BIGINT | NOT NULL, FK | 镜像分类ID |
| name | VARCHAR(100) | NOT NULL | 镜像名称（如 geth、fabric-peer） |
| display_name | VARCHAR(100) | NOT NULL | 显示名称（如 Go-Ethereum） |
| description | TEXT | NULL | 镜像描述 |
| icon_url | VARCHAR(500) | NULL | 镜像图标URL |
| ecosystem | VARCHAR(50) | NULL | 链生态：bitcoin / ethereum / fabric / chainmaker / fisco / solana / polkadot / cosmos / move / general |
| source_type | SMALLINT | NOT NULL DEFAULT 1 | 来源：1平台官方 2教师自定义 |
| uploaded_by | BIGINT | NULL | 上传者ID（教师自定义时） |
| school_id | BIGINT | NULL | 所属学校ID（教师自定义时） |
| status | SMALLINT | NOT NULL DEFAULT 1 | 状态：1正常 2待审核 3已下架 4审核拒绝 |
| review_comment | VARCHAR(500) | NULL | 审核意见 |
| reviewed_by | BIGINT | NULL | 审核人ID |
| reviewed_at | TIMESTAMP | NULL | 审核时间 |
| default_ports | JSONB | NULL | 默认端口映射：[{"port":8545,"protocol":"tcp","name":"RPC"}] |
| default_env_vars | JSONB | NULL | 默认环境变量：[{"key":"NETWORK_ID","value":"1337","desc":"网络ID"}] |
| default_volumes | JSONB | NULL | 默认挂载卷：[{"path":"/data","desc":"数据目录"}] |
| typical_companions | JSONB | NULL | 典型搭配：{"required":[{"image":"zookeeper","reason":"..."}],"recommended":[...],"optional":[...]} |
| required_dependencies | JSONB | NULL | 必要依赖镜像列表：["zookeeper","fabric-orderer"] |
| resource_recommendation | JSONB | NULL | 建议资源配置：{"cpu":"0.5","memory":"1Gi","disk":"2Gi"} |
| documentation_url | VARCHAR(500) | NULL | 镜像详细文档链接（如 /docs/images/geth） |
| tool_kind | VARCHAR(50) | NULL | 工具类型（与 §2.16 instance_containers.tool_kind 一致）：terminal / ide / desktop / explorer / monitor / null（链节点、中间件、基础环境等非工具镜像）；由镜像上架方在数据库声明，运行态 instance_containers.tool_kind 由此字段直接派生 |
| usage_count | INT | NOT NULL DEFAULT 0 | 使用次数（冗余计数） |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |
| deleted_at | TIMESTAMP | NULL | 软删除时间 |

**索引：**

```sql
CREATE INDEX idx_images_category_id ON images(category_id);
CREATE INDEX idx_images_ecosystem ON images(ecosystem);
CREATE INDEX idx_images_source_type ON images(source_type);
CREATE INDEX idx_images_status ON images(status);
CREATE INDEX idx_images_school_id ON images(school_id);
CREATE INDEX idx_images_uploaded_by ON images(uploaded_by);
```

**状态枚举：**

```go
const (
    ImageStatusActive    = 1  // 正常
    ImageStatusPending   = 2  // 待审核
    ImageStatusOffline   = 3  // 已下架
    ImageStatusRejected  = 4  // 审核拒绝
)
```

### 2.3 image_versions — 镜像版本表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| image_id | BIGINT | NOT NULL, FK | 所属镜像ID |
| version | VARCHAR(50) | NOT NULL | 版本号（如 1.13、2.5、latest） |
| registry_url | VARCHAR(500) | NOT NULL | 镜像仓库地址（如 registry.lianjing.com/geth:1.13） |
| image_size | BIGINT | NULL | 镜像大小（字节） |
| digest | VARCHAR(200) | NULL | 镜像摘要（SHA256） |
| min_cpu | VARCHAR(20) | NULL | 最低CPU要求（如 0.5） |
| min_memory | VARCHAR(20) | NULL | 最低内存要求（如 512Mi） |
| min_disk | VARCHAR(20) | NULL | 最低磁盘要求（如 1Gi） |
| is_default | BOOLEAN | NOT NULL DEFAULT FALSE | 是否为默认版本 |
| status | SMALLINT | NOT NULL DEFAULT 1 | 状态：1正常 2已废弃 |
| scan_result | JSONB | NULL | 安全扫描结果 |
| scanned_at | TIMESTAMP | NULL | 最后扫描时间 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |

**索引：**

```sql
CREATE INDEX idx_image_versions_image_id ON image_versions(image_id);
CREATE UNIQUE INDEX uk_image_versions_image_version ON image_versions(image_id, version);
```

| score_strategy | SMALLINT | NOT NULL DEFAULT 1 | 成绩策略：1取最后一次 2取最高分 |

### 2.8 sim_scenarios — 仿真场景库表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| name | VARCHAR(100) | NOT NULL | 场景名称（如 PBFT三阶段共识） |
| code | VARCHAR(100) | NOT NULL | 场景编码（唯一标识，如 pbft-consensus） |
| category | VARCHAR(50) | NOT NULL | 场景领域。平台内置场景使用 8 个标准领域：node_network / consensus / cryptography / data_structure / transaction / smart_contract / attack_security / economic；教师自定义场景允许使用扩展领域字符串，但运行时仍按字符串 category 与渲染器注册表匹配 |
| description | TEXT | NULL | 场景描述 |
| icon_url | VARCHAR(500) | NULL | 场景图标URL |
| thumbnail_url | VARCHAR(500) | NULL | 场景缩略图URL |
| source_type | SMALLINT | NOT NULL DEFAULT 1 | 来源：1平台内置 2教师自定义 |
| uploaded_by | BIGINT | NULL | 上传者ID（教师自定义时） |
| school_id | BIGINT | NULL | 所属学校ID（教师自定义时） |
| status | SMALLINT | NOT NULL DEFAULT 1 | 状态：1=published 已上架 / 2=pending_review 待审核 / 3=offline 已下架 / 4=rejected 审核拒绝 / 5=deprecated 已弃用（保留给历史实验，不可新选） |
| review_comment | VARCHAR(500) | NULL | 审核意见 |
| reviewed_by | BIGINT | NULL | 审核人ID |
| reviewed_at | TIMESTAMP | NULL | 审核时间 |
| algorithm_type | VARCHAR(100) | NOT NULL | 算法类型标识（如 pbft-consensus / sha256-hash / ecdsa-sign / pow-mining） |
| time_control_mode | VARCHAR(20) | NOT NULL DEFAULT 'process' | 时间控制模式：process（过程化）/ reactive（交互响应式）/ continuous（持续运行式） |
| extension_level | SMALLINT | NOT NULL DEFAULT 1 | SDK 扩展层级（详 06.md §14）：1=L1 平台原语组合（≥90% 场景） / 2=L2 平台 +1 原语函数 / 3=L3 自定义 npm 渲染器（教师扩展） |
| algorithm_logic_js | TEXT | NULL | L1 / L2 模式：场景算法 JavaScript 源码（在沙箱 Worker 内执行，平台审核）；L3 模式为空（算法在容器内） |
| npm_package_name | VARCHAR(200) | NULL | L3 模式：自定义渲染器 npm 包名，命名空间为 `@chainmirror/renderer-<school_id>__<package>` |
| npm_package_version | VARCHAR(50) | NULL | L3 模式：渲染器 npm 包版本号（与 npm_package_name 一一对应） |
| container_image_url | VARCHAR(500) | NULL | 场景算法容器镜像地址（平台内置场景预设；L3 教师自定义场景上传后填入） |
| container_image_size | BIGINT | NULL | 场景算法容器镜像大小（字节，通常10-50MB） |
| default_params | JSONB | NULL | 场景默认参数实例值（如 `{"node_count":4, ...}`） |
| default_params_schema | JSONB | NULL | 默认参数的 JSON Schema（教师配置时由前端按 schema 生成表单，详 06.2 §8.4） |
| interaction_schema | JSONB | NULL | 场景支持的交互操作定义（v2 ActionDef 格式，详 06.md §6.3） |
| data_source_mode | SMALLINT | NOT NULL DEFAULT 1 | 数据源模式：1仿真模式（场景算法容器驱动）2采集模式（Collector Agent从实验容器采集）3双模式 |
| default_size | JSONB | NULL | 默认尺寸：{"width":6,"height":4}（栅格单位） |
| delivery_phase | SMALLINT | NOT NULL DEFAULT 1 | 交付阶段：1/2/3（与 §十六交付计划对应，与 extension_level 正交） |
| current_version | VARCHAR(50) | NOT NULL DEFAULT '1.0.0' | 当前最新版本号；历史版本与版本切换详 §2.8.1 sim_scenario_versions |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |
| deleted_at | TIMESTAMP | NULL | 软删除时间 |

**shared_state_schema 字段结构（owner-based 模型，详 `06-可视化仿真引擎设计.md` §8.3）：**

**owner 校验：**
- 同一字段在一个 LinkGroup 内只能有 1 个 `owner`
- 教师在 P-21 step ③ 联动组选启用时，平台扫描已选场景对 schema 中 owner 字段的覆盖；某 owner 场景未选则 banner 警告并阻止启用

**派生规则（唯一来源）：**

`instance_containers.tool_kind` 在容器创建时由后端从 `images.tool_kind` **直接派生**，路径：

```
template_containers.image_version_id → image_versions.image_id → images.tool_kind
```

- 镜像上架时由平台/教师在 `images.tool_kind` 声明一次，所有引用该镜像的实例容器自动获得相同 tool_kind
- 后端 service 层不允许内置任何"镜像名→kind 映射表"，避免新工具上架必须改代码
- `images.tool_kind` 为 NULL 时，对应 `instance_containers.tool_kind` 也为 NULL（链节点、中间件、基础环境）

**真实环境实验终端约束（强制）：**

任何 `experiment_type ∈ {2 真实环境, 3 混合}` 的实验模板，其 `template_containers` 集合中**必须至少包含一个 `images.tool_kind = 'terminal'` 的镜像**（典型为 xterm-server）。该约束保证学生在真实环境实验中始终具备 Web 终端入口，对齐 04-前端页面设计.md §3.5 终端交互规范。

**proxy_url 生成规则：**

- 后端在容器状态变为"运行中"后，根据 Ingress/反向代理配置签发带短期 token 的访问 URL
- 格式示例：`https://lab.lianjing.com/instance/{instance_id}/{tool_kind}?token={jwt}`
- token 有效期 2 小时，过期后前端自动刷新（通过 WebSocket 推送新 URL 或前端轮询实例详情接口）
- 不同学校可配置不同域名前缀（租户隔离）

## 三、Redis 数据结构

### 3.1 实验实例状态缓存

```
# 实验实例实时状态（高频读取）
Key:   exp:instance:{instance_id}:status
Value: { "status": 3, "experiment_type": 1, "started_at": "2026-04-08T10:00:00Z", "last_active_at": "2026-04-08T10:25:00Z" }
TTL:   无过期（实例销毁时删除）
```

### 3.2 学生并发实验计数

```
# 学生当前运行中的实验数量（仅计算真实环境和混合实验，纯仿真不计入）
Key:   exp:student:{student_id}:running_count
Value: 整数（当前运行中的实验数）
TTL:   无过期（实验启动+1，停止-1）
```

### 3.3 课程并发实验计数

```
# 课程当前并发实验数（仅计算真实环境和混合实验）
Key:   exp:course:{course_id}:concurrency
Value: 整数
TTL:   无过期（实验启动+1，停止-1）
```

### 3.4 实验启动排队

```
# 资源不足时的排队队列
Key:   exp:queue:{course_id}
Value: List，元素为 { "student_id": xxx, "template_id": xxx, "requested_at": "..." }
TTL:   无过期（队列消费后移除）
```

### 3.5 实例活跃心跳

```
# 实例最后活跃时间（用于空闲超时检测）
Key:   exp:heartbeat:{instance_id}
Value: 最后活跃时间戳
TTL:   35分钟（超过30分钟无更新即视为空闲）
```

### 3.6 教师监控面板缓存

```
# 课程下所有实验实例的状态汇总（教师监控用）
Key:   exp:monitor:{course_id}:{template_id}
Value: Hash，field为student_id，value为 { "status": 3, "experiment_type": 2, "checkpoints_passed": 3, "checkpoints_total": 5 }
TTL:   30秒（定时刷新）
```

### 3.7 组内消息通道

```
# 组内实时消息（Pub/Sub通道）
Channel: exp:group:{group_id}:messages
```

### 3.8 SimEngine状态同步缓存

```
# SimEngine会话状态摘要（SimEngine Core通过WebSocket推送，用于教师监控仿真实验进度）
Key:   exp:sim:{instance_id}:state
Value: { "session_id": "xxx", "active_scenes": ["pbft-consensus","byzantine-attack"], "sim_tick": 152, "sim_time": 45.2, "speed": 1.0, "event_count": 128, "last_action": "inject_byzantine", "link_groups": ["pbft-attack-group"] }
TTL:   60秒（SimEngine Core定期推送刷新）
```

## 四、定时任务

| 任务 | 执行频率 | 说明 |
|------|---------|------|
| 空闲超时检测 | 每5分钟 | 检查运行中实例的最后活跃时间，超过空闲超时阈值的自动保存快照并回收 |
| 最长运行时间检测 | 每5分钟 | 检查运行中实例是否超过模板设置的最长运行时间，超时则强制快照并回收 |
| 自动快照 | 每5分钟 | 对运行中的实例自动保存增量快照（真实环境：容器快照；纯仿真：SimEngine状态快照） |
| 资源使用量同步 | 每1分钟 | 从K8s API同步各实例的实际CPU/内存使用量到数据库和Redis |
| 排队请求处理 | 每30秒 | 检查排队队列，当资源释放时自动创建排队中的实验实例 |
| 过期快照清理 | 每天凌晨3:00 | 清理已销毁实例的过期快照（保留最近30天） |
| 课程结束环境回收 | 每小时 | 检查已结束课程关联的运行中实验环境，强制回收 |
