## 一、ER 关系概览

```
users (用户主表)
  │
  ├── 1:N ── user_roles (用户角色关联)
  │             │
  │             └── N:1 ── roles (角色表)
  │                          │
  │                          └── 1:N ── role_permissions (角色权限关联)
  │                                        │
  │                                        └── N:1 ── permissions (权限表)
  │
  ├── 1:1 ── user_profiles (用户扩展信息)
  │
  ├── 1:N ── user_sso_bindings (SSO绑定记录)
  │
  ├── 1:N ── login_logs (登录日志)
  │
  └── 1:N ── operation_logs (操作日志)

schools (学校表，定义在模块02，此处仅引用)
  │
  └── 1:N ── users.school_id
```

## 二、表结构设计

### 2.1 users — 用户主表

> 存储所有用户的核心身份信息，是平台最基础的表。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| phone | VARCHAR(20) | NOT NULL | 手机号（全局唯一标识） |
| password_hash | VARCHAR(255) | NOT NULL | 密码哈希（bcrypt） |
| name | VARCHAR(50) | NOT NULL | 真实姓名 |
| school_id | BIGINT | NOT NULL, FK | 所属学校ID |
| student_no | VARCHAR(50) | NULL | 学号/工号（校内唯一） |
| status | SMALLINT | NOT NULL DEFAULT 1 | 账号状态：1正常 2禁用 3归档 |
| is_first_login | BOOLEAN | NOT NULL DEFAULT TRUE | 是否首次登录（需强制改密） |
| is_school_admin | BOOLEAN | NOT NULL DEFAULT FALSE | 是否兼任学校管理员 |
| login_fail_count | SMALLINT | NOT NULL DEFAULT 0 | 连续登录失败次数 |
| locked_until | TIMESTAMP | NULL | 锁定截止时间，NULL表示未锁定 |
| token_valid_after | TIMESTAMP | NOT NULL DEFAULT NOW() | Token生效时间基线，早于该时间签发的Access Token全部失效 |
| last_login_at | TIMESTAMP | NULL | 最后登录时间 |
| last_login_ip | VARCHAR(45) | NULL | 最后登录IP |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |
| created_by | BIGINT | NULL | 创建人ID |
| deleted_at | TIMESTAMP | NULL | 软删除时间 |

**索引：**

```sql
CREATE UNIQUE INDEX uk_users_phone ON users(phone) WHERE deleted_at IS NULL;
CREATE UNIQUE INDEX uk_users_school_student_no ON users(school_id, student_no) WHERE deleted_at IS NULL AND student_no IS NOT NULL;
CREATE INDEX idx_users_school_id ON users(school_id);
CREATE INDEX idx_users_status ON users(status);
CREATE INDEX idx_users_name ON users(name);
```

**状态枚举：**

```go
const (
    UserStatusActive   = 1  // 正常
    UserStatusDisabled = 2  // 禁用
    UserStatusArchived = 3  // 归档
)
```

### 2.2 user_profiles — 用户扩展信息表

> 存储非核心的用户信息，与主表分离以保持主表精简。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| user_id | BIGINT | NOT NULL, FK, UNIQUE | 关联用户ID |
| avatar_url | VARCHAR(500) | NULL | 头像URL |
| nickname | VARCHAR(50) | NULL | 昵称 |
| email | VARCHAR(100) | NULL | 邮箱 |
| college | VARCHAR(100) | NULL | 学院 |
| major | VARCHAR(100) | NULL | 专业 |
| class_name | VARCHAR(50) | NULL | 班级 |
| enrollment_year | SMALLINT | NULL | 入学年份 |
| education_level | SMALLINT | NULL | 学业层次：1专科 2本科 3硕士 4博士 |
| grade | SMALLINT | NULL | 年级 |
| remark | TEXT | NULL | 备注 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |

**索引：**

```sql
CREATE UNIQUE INDEX uk_user_profiles_user_id ON user_profiles(user_id);
CREATE INDEX idx_user_profiles_college ON user_profiles(college);
CREATE INDEX idx_user_profiles_education_level ON user_profiles(education_level);
```

**学业层次枚举：**

```go
const (
    EduLevelJuniorCollege = 1  // 专科
    EduLevelUndergraduate = 2  // 本科
    EduLevelMaster        = 3  // 硕士
    EduLevelDoctor        = 4  // 博士
)
```

### 2.3 roles — 角色表

> 定义系统中的角色，支持预设角色和自定义角色。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| code | VARCHAR(50) | NOT NULL, UNIQUE | 角色编码（如 super_admin） |
| name | VARCHAR(50) | NOT NULL | 角色显示名称 |
| description | VARCHAR(200) | NULL | 角色描述 |
| is_system | BOOLEAN | NOT NULL DEFAULT FALSE | 是否系统预设角色（不可删除） |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |

**预设角色数据：**

| code | name | is_system |
|------|------|-----------|
| super_admin | 超级管理员 | TRUE |
| school_admin | 学校管理员 | TRUE |
| teacher | 教师 | TRUE |
| student | 学生 | TRUE |

---

### 2.4 user_roles — 用户角色关联表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| user_id | BIGINT | NOT NULL, FK | 用户ID |
| role_id | BIGINT | NOT NULL, FK | 角色ID |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |

**索引：**

```sql
CREATE UNIQUE INDEX uk_user_roles_user_role ON user_roles(user_id, role_id);
CREATE INDEX idx_user_roles_role_id ON user_roles(role_id);
```

**业务约束（应用层实现）：**

- 学生角色不可与其他角色共存
- 教师角色仅可与学校管理员角色共存
- 超级管理员角色不可与其他角色共存

### 2.5 permissions — 权限表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| code | VARCHAR(100) | NOT NULL, UNIQUE | 权限编码（如 user:import） |
| name | VARCHAR(100) | NOT NULL | 权限显示名称 |
| module | VARCHAR(50) | NOT NULL | 所属模块（如 user, course） |
| description | VARCHAR(200) | NULL | 权限描述 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |

**索引：**

```sql
CREATE INDEX idx_permissions_module ON permissions(module);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| role_id | BIGINT | NOT NULL, FK | 角色ID |

### 2.7 user_sso_bindings — SSO绑定记录表

> 记录用户与各学校SSO系统的绑定关系。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| user_id | BIGINT | NOT NULL, FK | 用户ID |
| school_id | BIGINT | NOT NULL, FK | 学校ID |
| sso_provider | VARCHAR(20) | NOT NULL | SSO协议类型：cas / oauth2 |
| sso_user_id | VARCHAR(100) | NOT NULL | SSO系统中的用户标识（学号/工号） |
| bound_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 绑定时间 |
| last_login_at | TIMESTAMP | NULL | 最后一次SSO登录时间 |

**索引：**

```sql
CREATE UNIQUE INDEX uk_sso_bindings_school_sso_user ON user_sso_bindings(school_id, sso_user_id);
CREATE INDEX idx_sso_bindings_user_id ON user_sso_bindings(user_id);
```

---

### 2.8 login_logs — 登录日志表

> 记录所有登录/登出/踢下线等认证事件。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| user_id | BIGINT | NOT NULL | 用户ID（不设FK，避免删除用户后日志丢失） |
| action | SMALLINT | NOT NULL | 操作类型：1登录成功 2登录失败 3登出 4被踢下线 5账号锁定 |
| login_method | SMALLINT | NULL | 登录方式：1密码 2SSO-CAS 3SSO-OAuth2 |
| ip | VARCHAR(45) | NOT NULL | 客户端IP |
| user_agent | VARCHAR(500) | NULL | 浏览器UA |
| fail_reason | VARCHAR(200) | NULL | 失败原因（仅失败时记录） |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 记录时间 |

**索引：**

```sql
CREATE INDEX idx_login_logs_user_id ON login_logs(user_id);
CREATE INDEX idx_login_logs_created_at ON login_logs(created_at);
CREATE INDEX idx_login_logs_action ON login_logs(action);
```

**操作类型枚举：**

```go
const (
    LoginActionSuccess    = 1  // 登录成功
    LoginActionFail       = 2  // 登录失败
    LoginActionLogout     = 3  // 主动登出
    LoginActionKicked     = 4  // 被踢下线
    LoginActionLocked     = 5  // 账号锁定
)
```

---

### 2.9 operation_logs — 操作日志表

> 记录用户管理相关的关键操作（非登录类）。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| operator_id | BIGINT | NOT NULL | 操作人ID |
| action | VARCHAR(50) | NOT NULL | 操作类型（如 import_students, reset_password） |
| target_type | VARCHAR(50) | NOT NULL | 操作对象类型（如 user） |
| target_id | BIGINT | NULL | 操作对象ID |
| detail | JSONB | NULL | 操作详情（变更前后的数据快照） |
| ip | VARCHAR(45) | NOT NULL | 操作人IP |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 操作时间 |

**索引：**

```sql
CREATE INDEX idx_operation_logs_operator_id ON operation_logs(operator_id);
CREATE INDEX idx_operation_logs_action ON operation_logs(action);
CREATE INDEX idx_operation_logs_target ON operation_logs(target_type, target_id);
CREATE INDEX idx_operation_logs_created_at ON operation_logs(created_at);
```

## 三、Redis 数据结构

### 3.1 Token 会话管理

```
# 用户当前有效的Refresh Token（单设备登录，只存一个）
Key:   session:{user_id}
Value: { "refresh_token": "xxx", "device_info": "xxx", "login_at": 1712505600 }
TTL:   7天

# Access Token 黑名单（被踢下线时将未过期的Token加入黑名单）
Key:   token_blacklist:{access_token_jti}
Value: 1
TTL:   与该Token剩余有效期一致

# 用户Token生效时间基线缓存（用于强制下线后的快速校验）
Key:   token_valid_after:{user_id}
Value: RFC3339 时间字符串
TTL:   7天（重新登录或强制下线时刷新）
```

### 3.2 登录失败计数

```
# 登录失败计数器
Key:   login_fail:{phone}
Value: 失败次数（整数）
TTL:   与锁定策略中的计数窗口一致（如30分钟）

# 账号锁定标记
Key:   account_locked:{phone}
Value: 1
TTL:   锁定时长（可配置）
```

### 3.3 安全策略配置缓存

```
# 全局安全策略配置
Key:   config:security_policy
Value: { "max_fail_count": 5, "lock_duration_minutes": 15 }
TTL:   无过期（配置变更时主动刷新）
```
