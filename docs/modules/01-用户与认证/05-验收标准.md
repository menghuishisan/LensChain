# 用户与认证模块 — 验收标准

> 模块状态：✅ 已确认
> 文档版本：v1.0

---

## 一、功能验收条件

### 1.1 登录功能

#### AC-01：手机号+密码正常登录

```
Given  用户已被导入且账号状态为正常，且非首次登录
When   用户输入正确的手机号和密码点击登录
Then   登录成功，返回双Token，跳转到平台首页
And    Redis中记录该用户Session
And    记录登录成功日志
```

#### AC-02：首次登录强制改密

```
Given  用户账号的 is_first_login = true
When   用户输入正确的手机号和密码点击登录
Then   返回 temp_token（5分钟有效）
And    前端跳转到强制修改密码页面
And    用户设置符合复杂度要求的新密码后
Then   返回正式双Token，is_first_login 更新为 false
And    自动登录跳转首页
```

#### AC-03：密码错误计数与锁定

```
Given  安全策略配置为：最大失败5次，锁定15分钟
When   用户连续输入错误密码
Then   第1-4次返回"密码错误，还剩N次机会"
And    第5次返回"账号已锁定，请15分钟后重试"
And    记录账号锁定日志
And    锁定期间任何登录尝试均返回锁定提示
And    15分钟后自动解锁，失败计数归零
```

#### AC-04：单设备登录踢下线

```
Given  用户已在设备A登录
When   用户在设备B使用同一账号登录
Then   设备B登录成功，获得新Token
And    设备A的Session被删除，Access Token加入黑名单
And    设备A下次请求返回 code: 40107
And    设备A前端弹出"账号已在其他设备登录"提示
And    记录设备A被踢下线日志
```

#### AC-05：SSO登录成功

```
Given  学校已配置SSO（CAS或OAuth2.0），用户已被导入且学号匹配
When   用户选择学校 → 跳转SSO认证 → 认证成功回调
Then   系统通过「学校ID + 学号」匹配到用户
And    签发双Token，登录成功
And    更新 user_sso_bindings 表
```

#### AC-06：SSO登录失败（未导入）

```
Given  学校已配置SSO，但该学号未在平台中导入
When   用户通过SSO认证成功后回调
Then   返回"账号未开通，请联系管理员"
And    不创建任何用户记录
```

### 1.2 Token管理

#### AC-07：Access Token过期自动续期

```
Given  用户已登录，Access Token剩余有效期 < 5分钟
When   前端发起API请求
Then   前端先调用 refresh 接口获取新的双Token
And    用新Access Token重新发起原请求
And    用户无感知
```

#### AC-08：Refresh Token过期

```
Given  用户的Refresh Token已过期（超过7天未活跃）
When   前端尝试刷新Token
Then   返回 code: 40106
And    前端清除本地Token，跳转登录页
```

### 1.3 用户导入

#### AC-09：下载导入模板

```
Given  学校管理员进入导入页面
When   选择导入类型（学生/教师）并点击下载模板
Then   下载对应的Excel模板文件
And    模板包含所有字段列名、字段说明、示例数据
```

#### AC-10：导入预览正常数据

```
Given  管理员上传了填写正确的Excel文件
When   点击"上传并预览"
Then   显示解析结果统计（总计/有效/无效/冲突）
And    预览列表展示每行数据及其校验状态
And    有效数据标记为 ✓
```

#### AC-11：导入预览冲突处理

```
Given  Excel中某行手机号在系统中已存在
When   预览页面展示该行
Then   标记为 ⚠冲突，显示已存在用户的信息
And    管理员可选择"跳过"或"覆盖"
And    支持全局策略（全部跳过/全部覆盖）和逐条选择
```

#### AC-12：导入预览无效数据

```
Given  Excel中某行数据校验不通过（如手机号格式错误、姓名为空）
When   预览页面展示该行
Then   标记为 ✗无效，显示具体错误原因
And    无效数据不可导入
```

#### AC-13：确认导入并反馈结果

```
Given  管理员在预览页面确认冲突处理策略
When   点击"确认导入"
Then   后端执行导入，返回结果统计
And    弹窗显示：成功N条、失败N条、跳过N条、覆盖N条
And    可下载失败明细Excel（包含行号、数据、失败原因）
And    记录导入操作日志
```

#### AC-14：导入数据校验规则

```
校验项：
- 姓名：必填，长度1-50
- 手机号：必填，11位中国大陆手机号格式
- 学号/工号：必填，校内唯一
- 初始密码：必填，满足当前密码复杂度策略
- 邮箱：选填，格式校验
- 入学年份：选填，合理年份范围（2000-当前年份）
- 学业层次：选填，仅接受"专科/本科/硕士/博士"
```

### 1.4 用户管理

#### AC-15：禁用账号

```
Given  管理员在用户列表选择一个正常状态的用户
When   点击"禁用"并确认
Then   用户状态变更为禁用
And    该用户当前Session立即失效（踢下线）
And    该用户尝试登录时返回"账号已被禁用"
And    记录操作日志
```

#### AC-16：归档账号

```
Given  管理员选择一个正常状态的用户
When   点击"归档"并确认
Then   用户状态变更为归档
And    该用户当前Session立即失效
And    该用户的所有历史数据（课程、实验、竞赛）保留
And    该用户无法登录
```

#### AC-17：重置密码

```
Given  管理员在用户详情页点击"重置密码"
When   输入新密码并确认
Then   用户密码被重置
And    is_first_login 设为 true（下次登录需强制改密）
And    该用户当前Session立即失效
And    记录操作日志
```

#### AC-18：学籍变更

```
Given  管理员编辑学生的学籍信息（学院/专业/班级）
When   保存修改
Then   学籍信息更新成功
And    用户账号（手机号、密码、角色）不受影响
And    用户的历史学习数据不受影响
And    记录操作日志（含变更前后数据快照）
```

### 1.5 个人中心

#### AC-19：修改密码

```
Given  用户在个人中心点击"修改密码"
When   输入正确的旧密码和符合要求的新密码
Then   密码修改成功
And    当前Session保持有效（不踢下线）
And    新密码不能与旧密码相同
```

#### AC-20：编辑个人信息

```
Given  用户在个人中心编辑昵称/头像/邮箱
When   保存修改
Then   信息更新成功
And    姓名、手机号、学号、学籍信息不可编辑（灰色不可点击）
```

## 二、性能指标

| 指标 | 要求 |
|------|------|
| 登录接口响应时间 | ≤ 500ms（P95） |
| Token刷新接口响应时间 | ≤ 200ms（P95） |
| 用户列表查询响应时间 | ≤ 1s（P95，1000条数据以内） |
| 批量导入处理速度 | ≥ 100条/秒 |
| 批量导入最大支持量 | 单次 ≤ 5000条 |
| 导入预览响应时间 | ≤ 10s（5000条数据） |
| 并发登录支持 | ≥ 100 QPS |

## 三、安全要求

| 要求 | 说明 |
|------|------|
| 密码存储 | 使用 bcrypt 哈希，cost ≥ 10 |
| 密码传输 | HTTPS加密传输，前端不存储明文密码 |
| Token安全 | JWT使用HS256签名，密钥 ≥ 256位 |
| 手机号脱敏 | 个人中心和日志中手机号脱敏显示（138****8000） |
| SQL注入防护 | 所有查询使用参数化查询 |
| XSS防护 | 所有用户输入进行转义处理 |
| CSRF防护 | API使用Token认证，无需额外CSRF Token |
| 暴力破解防护 | 登录失败锁定 + 可配置策略 |
| 权限隔离 | 学校管理员只能操作本校数据，中间件层强制过滤 school_id |
| 日志安全 | 操作日志中不记录密码明文 |
| 导入文件安全 | 校验文件类型和大小，防止恶意文件上传 |

## 四、边界条件与异常处理

| 场景 | 处理方式 |
|------|---------|
| 上传非Excel/CSV文件 | 前端+后端双重校验，提示"仅支持.xlsx和.csv格式" |
| 上传空文件 | 提示"文件内容为空" |
| 上传超过5000行的文件 | 提示"单次导入不超过5000条" |
| Excel中存在空行 | 自动跳过空行 |
| Excel中字段前后有空格 | 自动去除前后空格 |
| 并发导入同一批数据 | 通过import_id保证幂等，同一批次不可重复执行 |
| 导入过程中服务器异常 | 事务回滚，返回错误提示，管理员可重新导入 |
| 管理员删除自己的账号 | 禁止操作，提示"不能删除自己的账号" |
| 超管禁用最后一个超管 | 禁止操作，提示"至少保留一个超级管理员" |
| 密码重置后用户正在线 | 立即踢下线，用户需用新密码重新登录 |
| SSO系统不可用 | 返回"学校认证服务暂时不可用，请使用手机号密码登录" |
| Redis不可用 | 降级处理：登录失败计数暂时不生效，记录错误日志告警 |
