## 二、页面详细设计

### P-01 登录页

**布局描述：** 左右分栏布局，左侧为品牌Hero展示区，右侧为登录表单区。

```
┌─────────────────────────────┬────────────────────────────┐
│                             │                            │
│       ┌─────────┐          │      链镜 LOGO              │
│       │ 区块链   │          │      区块链教学与实践平台     │
│       │ 动态插画  │          │                            │
│       │ / 动效   │          │   手机号  [______________]  │
│       └─────────┘          │   密  码  [______________]  │
│                             │                            │
│   链镜 — 区块链教学与实践平台 │      [     登  录     ]     │
│                             │                            │
│   "教学、实验、竞赛          │      ────── 或 ──────      │
│    三位一体的区块链学习平台"  │                            │
│                             │      [  学校SSO登录  ]      │
│   ● 多链全覆盖              │                            │
│   ● 动态实验环境             │                            │
│   ● CTF竞赛实战             │                            │
│                             │                            │
│                             │   © 2026 链镜平台           │
└─────────────────────────────┴────────────────────────────┘
```

**左侧Hero区说明：**
- 背景：深色/渐变主题色，可配置背景图或动态粒子效果
- 内容：平台品牌名称、Slogan、核心特性亮点（3-4条）
- 可选：区块链相关的动态插画或Lottie动效
- 响应式：窄屏时隐藏左侧Hero区，仅显示右侧登录表单

**右侧登录区说明：**
- 白色背景，居中显示登录表单
- 顶部显示LOGO和平台名称
- 表单区域包含手机号、密码输入框和登录按钮
- 分割线下方为SSO登录入口
- 底部显示版权信息

**交互流程：**

1. 用户输入手机号和密码，点击「登录」
2. 前端校验：手机号11位数字、密码非空
3. 调用 `POST /api/v1/auth/login`
4. 根据响应处理：
   - `is_first_login = true` → 跳转 P-03 强制改密页
   - 正常登录 → 存储双Token，跳转首页
   - 错误 → 显示对应错误提示
5. 点击「学校SSO登录」→ 跳转 P-02 学校选择页

**组件：**
- `LoginHero` — 左侧品牌展示区组件
- `LoginForm` — 右侧登录表单组件
- `SSOLoginButton` — SSO登录入口按钮

## 三、全局交互规范

### 3.1 Token 无感续期

1. 前端在每次API请求前检查Access Token是否即将过期（剩余<5分钟）
2. 如即将过期，先调用 `/api/v1/auth/token/refresh` 获取新Token
3. 用新Token替换旧Token后再发起业务请求
4. 如果Refresh Token也过期，跳转登录页

### 3.2 被踢下线处理

1. 任意API请求返回 `code: 40107`（Token无效）
2. 前端弹出提示："您的账号已在其他设备登录"
3. 用户点击确认后清除本地Token，跳转登录页

### 3.3 表格通用交互

- 支持列排序（点击表头）
- 支持多选（复选框）
- 操作按钮超过3个时收入「更多」下拉菜单
- 删除、禁用等危险操作需二次确认弹窗

### 3.4 表单通用交互

- 必填字段标红色星号 *
- 实时校验并显示错误提示
- 提交按钮在校验不通过时置灰
- 提交中显示loading状态，防止重复提交
