## 一、ER 关系概览

```
competitions (竞赛主表)
  │
  ├── 1:N ── competition_challenges (竞赛题目关联)
  │             │
  │             └── N:1 ── challenges (题目主表)
  │
  ├── 1:N ── competition_registrations (竞赛报名)
  │
  ├── 1:N ── teams (参赛团队)
  │             │
  │             └── 1:N ── team_members (团队成员)
  │
  ├── 1:N ── submissions (提交记录)
  │
  ├── 1:N ── leaderboard_snapshots (排行榜快照)
  │
  ├── 1:N ── announcements (竞赛公告)
  │
  ├── 1:N ── ad_rounds (攻防赛回合)
  │             │
  │             ├── 1:N ── ad_attacks (攻击记录)
  │             │
  │             └── 1:N ── ad_defenses (防守记录)
  │
  └── 1:N ── ad_token_ledger (Token流水账)

challenges (题目主表)
  │
  ├── 1:N ── challenge_contracts (题目合约)
  │
  ├── 1:N ── challenge_assertions (验证断言)
  │
  ├── 1:N ── challenge_reviews (审核记录)
  │
  └── 1:N ── challenge_verifications (预验证记录)

challenge_templates (参数化模板库)

ctf_resource_quotas (CTF资源配额)
```

## 二、表结构设计

### 2.1 competitions — 竞赛主表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| title | VARCHAR(200) | NOT NULL | 竞赛名称 |
| description | TEXT | NULL | 竞赛描述（Markdown） |
| banner_url | VARCHAR(500) | NULL | 竞赛横幅图片URL |
| competition_type | SMALLINT | NOT NULL | 竞赛类型：1解题赛 2攻防对抗赛 |
| scope | SMALLINT | NOT NULL DEFAULT 1 | 竞赛范围：1平台级 2校级 |
| school_id | BIGINT | NULL | 所属学校ID（校级竞赛时必填） |
| created_by | BIGINT | NOT NULL | 创建者ID（超级管理员/学校管理员） |
| team_mode | SMALLINT | NOT NULL DEFAULT 1 | 参赛模式：1个人赛 2自由组队 3指定组队 |
| max_team_size | INT | NOT NULL DEFAULT 1 | 团队最大人数（个人赛时为1） |
| min_team_size | INT | NOT NULL DEFAULT 1 | 团队最小人数 |
| max_teams | INT | NULL | 最大参赛队伍数（NULL不限） |
| status | SMALLINT | NOT NULL DEFAULT 1 | 状态：1草稿 2报名中 3进行中 4已结束 5已归档 |
| registration_start_at | TIMESTAMP | NULL | 报名开始时间 |
| registration_end_at | TIMESTAMP | NULL | 报名截止时间 |
| start_at | TIMESTAMP | NULL | 竞赛开始时间 |
| end_at | TIMESTAMP | NULL | 竞赛结束时间 |
| freeze_at | TIMESTAMP | NULL | 排行榜冻结时间（默认结束前1小时） |
| jeopardy_config | JSONB | NULL | 解题赛配置（解题赛时） |
| ad_config | JSONB | NULL | 攻防赛配置（攻防赛时） |
| rules | TEXT | NULL | 竞赛规则说明（Markdown） |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |
| deleted_at | TIMESTAMP | NULL | 软删除时间 |

**索引：**

```sql
CREATE INDEX idx_competitions_competition_type ON competitions(competition_type) WHERE deleted_at IS NULL;
CREATE INDEX idx_competitions_scope ON competitions(scope) WHERE deleted_at IS NULL;
CREATE INDEX idx_competitions_school_id ON competitions(school_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_competitions_status ON competitions(status) WHERE deleted_at IS NULL;
CREATE INDEX idx_competitions_created_by ON competitions(created_by) WHERE deleted_at IS NULL;
CREATE INDEX idx_competitions_start_at ON competitions(start_at) WHERE deleted_at IS NULL;
```

### 2.2 challenges — 题目主表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| title | VARCHAR(200) | NOT NULL | 题目名称 |
| description | TEXT | NOT NULL | 题目描述（Markdown） |
| category | VARCHAR(20) | NOT NULL | 题目类型：web / crypto / contract / blockchain / reverse / misc |
| difficulty | SMALLINT | NOT NULL | 难度：1Warmup 2Easy 3Medium 4Hard 5Insane |
| base_score | INT | NOT NULL | 基础分值（100-1000） |
| flag_type | SMALLINT | NOT NULL DEFAULT 1 | Flag类型：1静态Flag 2动态Flag 3链上状态验证 |
| static_flag | VARCHAR(500) | NULL | 静态Flag值（flag_type=1时，加密存储） |
| dynamic_flag_secret | VARCHAR(200) | NULL | 动态Flag密钥（flag_type=2时，用于HMAC-SHA256） |
| runtime_mode | SMALLINT | NOT NULL DEFAULT 1 | 链上运行时模式：1isolated 2forked |
| chain_config | JSONB | NULL | 链配置（智能合约题目时） |
| setup_transactions | JSONB | NULL | 题目初始化交易（链上题目时） |
| source_path | SMALLINT | NULL | 漏洞来源路径：1SWC导入 2参数化模板 3完全自定义 |
| swc_id | VARCHAR(20) | NULL | SWC编号（SWC导入时，如 SWC-107） |
| template_id | BIGINT | NULL, FK | 参数化模板ID（模板生成时） |
| template_params | JSONB | NULL | 模板参数值（模板生成时） |
| environment_config | JSONB | NULL | 环境配置（非合约题目的容器配置） |
| attachment_urls | JSONB | NULL | 附件URL列表（题目附件/源码包） |
| author_id | BIGINT | NOT NULL | 出题者ID（教师） |
| school_id | BIGINT | NOT NULL | 出题者所属学校ID |
| status | SMALLINT | NOT NULL DEFAULT 1 | 状态：1草稿 2待审核 3已通过 4已拒绝 |
| is_public | BOOLEAN | NOT NULL DEFAULT FALSE | 是否进入平台公共题库 |
| usage_count | INT | NOT NULL DEFAULT 0 | 被竞赛使用次数（冗余计数） |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |
| deleted_at | TIMESTAMP | NULL | 软删除时间 |

**索引：**

```sql
CREATE INDEX idx_challenges_category ON challenges(category) WHERE deleted_at IS NULL;
CREATE INDEX idx_challenges_difficulty ON challenges(difficulty) WHERE deleted_at IS NULL;
CREATE INDEX idx_challenges_flag_type ON challenges(flag_type) WHERE deleted_at IS NULL;
CREATE INDEX idx_challenges_runtime_mode ON challenges(runtime_mode) WHERE deleted_at IS NULL;
CREATE INDEX idx_challenges_author_id ON challenges(author_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_challenges_school_id ON challenges(school_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_challenges_status ON challenges(status) WHERE deleted_at IS NULL;
```

### 2.4 challenge_assertions — 验证断言表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| challenge_id | BIGINT | NOT NULL, FK | 所属题目ID |
| assertion_type | VARCHAR(30) | NOT NULL | 断言类型：balance_check / token_balance_check / storage_check / owner_check / event_check / code_check / custom_script |
| target | VARCHAR(200) | NOT NULL | 检查目标（合约名称/地址） |
| operator | VARCHAR(10) | NOT NULL | 比较运算符：eq / ne / gt / gte / lt / lte / contains |
| expected_value | TEXT | NOT NULL | 期望值 |
| description | VARCHAR(500) | NULL | 断言描述（如"合约ETH余额被掏空"） |
| extra_params | JSONB | NULL | 额外参数（如 storage_slot、token_address、event_signature 等） |
| sort_order | INT | NOT NULL DEFAULT 0 | 断言执行顺序 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |

**索引：**

```sql
CREATE INDEX idx_challenge_assertions_challenge_id ON challenge_assertions(challenge_id);
```

**断言类型枚举：**

```go
const (
    AssertionTypeBalance      = "balance_check"       // ETH余额检查
    AssertionTypeTokenBalance = "token_balance_check"  // ERC20代币余额检查
    AssertionTypeStorage      = "storage_check"        // 存储槽值检查
    AssertionTypeOwner        = "owner_check"          // 合约owner变更检查
    AssertionTypeEvent        = "event_check"          // 事件日志检查
    AssertionTypeCode         = "code_check"           // 合约代码检查（extcodesize）
    AssertionTypeCustomScript = "custom_script"        // 自定义脚本验证
)
```

## 三、Redis 数据结构

### 3.1 实时排行榜

```
# 解题赛排行榜（Sorted Set，score为总分）
Key:   ctf:leaderboard:{competition_id}
Value: Sorted Set，member=team_id，score=总分
TTL:   竞赛结束后24小时过期

# 攻防赛排行榜（Sorted Set，score为Token余额）
Key:   ctf:ad:leaderboard:{competition_id}:{group_id}
Value: Sorted Set，member=team_id，score=Token余额
TTL:   竞赛结束后24小时过期
```

### 3.2 排行榜冻结标记

```
# 排行榜是否处于冻结状态
```

## 四、定时任务

| 任务 | 执行频率 | 说明 |
|------|---------|------|
| 竞赛状态流转 | 每1分钟 | 检查竞赛时间，自动执行状态流转：报名开始→报名中，竞赛开始→进行中，竞赛结束→已结束 |
| 排行榜冻结 | 每1分钟 | 检查是否到达冻结时间，设置冻结标记，停止排行榜公开更新 |
| 排行榜快照 | 每5分钟 | 对进行中的竞赛保存排行榜快照到数据库（用于历史回溯） |
| 攻防赛回合推进 | 每10秒 | 检查攻防赛回合阶段时间，自动推进阶段：攻击→防守→结算→下一轮攻击 |
| 攻防赛自动结算 | 事件触发 | 结算阶段触发时，自动执行Token转账计算、更新余额、记录流水 |
| 环境资源回收 | 每5分钟 | 回收已结束竞赛的K8s环境资源（Namespace、Pod、PVC） |
| Token余额同步 | 每1分钟 | 将Redis中的Token余额同步到数据库teams表（攻防赛） |
| 动态分值同步 | 每1分钟 | 将Redis中的动态分值同步到数据库competition_challenges表（解题赛） |
| 竞赛归档 | 每天凌晨2:00 | 将结束超过30天的竞赛自动归档，清理Redis缓存数据 |
| 预验证环境清理 | 每30分钟 | 清理超过1小时未完成的预验证临时环境 |
