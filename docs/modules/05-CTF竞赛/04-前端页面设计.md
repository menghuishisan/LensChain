## 三、全局交互规范

### 3.1 竞赛状态与页面操作映射

| 竞赛状态 | 管理员可操作 | 选手可操作 |
|----------|-------------|-----------|
| 草稿 | 编辑、配置题目、删除 | 不可见 |
| 报名中 | 查看报名、发布公告 | 浏览、报名、组队 |
| 进行中 | 监控、发布公告、强制终止 | 解题/攻防、提交、查看排行榜 |
| 已结束 | 查看统计、归档 | 查看结果、排行榜 |
| 已归档 | 查看历史 | 查看历史 |

### 3.2 WebSocket 实时更新规范

| 频道 | 更新内容 | 更新频率 |
|------|---------|---------|
| 排行榜 | 排名变动、分数更新 | 每次正确提交触发 |
| 公告 | 新公告推送 | 管理员发布时触发 |
| 回合状态 | 阶段切换、倒计时同步 | 阶段切换时触发 |
| 攻击事件 | 攻击结果实时展示 | 每次攻击提交触发 |

### 3.3 心跳与断线重连

- 客户端每 30 秒发送心跳 ping
- 60 秒未收到心跳，服务端断开连接
- 客户端断线后自动重连（指数退避，最大间隔 30 秒）
- 重连后自动同步最新排行榜和回合状态

### 3.4 提交限流前端处理

| 场景 | 前端表现 |
|------|---------|
| 正常提交 | 提交按钮可用，显示剩余次数 |
| 接近限流 (4/5) | 提交按钮黄色警告，提示"本分钟剩余1次" |
| 触发限流 (5/5) | 提交按钮禁用，显示"请等待N秒" |
| 触发冷却期 | 提交按钮禁用，显示"冷却中，剩余N分钟" |

### 3.5 响应式布局

- PC 优先设计，最小宽度 1280px
- 解题赛主页（P-23）三栏布局在 < 1440px 时折叠为两栏（排行榜收起为侧边抽屉）
- 攻防赛主页（P-25）在 < 1280px 时攻击动态区域折叠
- 代码编辑器（P-11/P-24/P-27）最小高度 300px

### 3.6 公告通知交互

- 新公告通过 WebSocket 推送，顶部铃铛图标显示未读数
- 点击铃铛展开公告列表侧边栏
- 题目勘误类公告在对应题目详情页顶部显示黄色横幅提示
- 公告支持 Markdown 渲染
