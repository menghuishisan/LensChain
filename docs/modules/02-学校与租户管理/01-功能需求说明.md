# 学校与租户管理模块 — 功能需求说明

> 模块状态：✅ 已确认
> 文档版本：v1.0

---

## 一、模块概述

### 1.1 模块职责

学校与租户管理模块负责平台的多租户体系管理，包括：

- 学校入驻申请与审核流程
- 学校基本信息与配置管理
- SSO接入配置管理
- 学校授权有效期与生命周期管理（激活、冻结、注销）
- 多租户数据隔离策略的基础支撑

### 1.2 模块边界

| 本模块负责 | 本模块不负责 |
|-----------|-------------|
| 学校CRUD、入驻审核流程 | 学校内用户的管理（→ 模块01） |
| SSO接入参数配置 | SSO认证流程的执行（→ 模块01） |
| 学校有效期、冻结、注销 | 课程与学校的关联（→ 模块03） |
| 学校基本信息维护 | 实验环境资源分配（→ 模块04） |
| 首个校管账号的自动创建 | 校管后续管理用户（→ 模块01） |

### 1.3 依赖关系

```
本模块依赖：
  → 01-用户与认证（创建校管账号时调用用户模块）

本模块被以下模块依赖：
  ← 01-用户与认证（用户表通过 school_id 关联学校）
  ← 03-课程与教学（课程归属学校）
  ← 04-实验环境（实验环境按学校隔离）
  ← 05-CTF竞赛（竞赛可按学校组织）
```

## 二、功能清单

### 2.1 按角色分类

| S-SA-03 | 后台录入学校 | 线下沟通后直接在后台创建学校（跳过申请流程） |
| S-SA-04 | 学校列表管理 | 查看、搜索、筛选所有学校 |
| S-SA-05 | 编辑学校信息 | 修改学校基本信息 |
| S-SA-06 | 设置学校有效期 | 设置/修改学校的授权有效期 |
| S-SA-07 | 冻结学校 | 手动冻结学校，该校所有用户无法登录 |
| S-SA-08 | 解冻学校 | 解冻已冻结的学校，恢复正常使用 |
| S-SA-09 | 注销学校 | 软删除学校及其所有关联数据 |
| S-SA-10 | 恢复已注销学校 | 恢复软删除的学校 |

#### 学校管理员

| 编号 | 功能 | 说明 |
|------|------|------|
| S-AD-01 | 查看学校信息 | 查看本校基本信息和配置 |
| S-AD-02 | 编辑学校基本信息 | 修改本校LOGO、简介等展示信息 |
| S-AD-03 | 配置SSO接入 | 配置本校SSO参数（CAS/OAuth2端点、密钥等） |
| S-AD-04 | 测试SSO连接 | 配置完成后测试SSO连接是否正常 |
| S-AD-05 | 查看授权状态 | 查看本校的授权有效期和当前状态 |

#### 申请人（未注册用户）

| 编号 | 功能 | 说明 |
|------|------|------|
| S-AP-01 | 提交入驻申请 | 在公开页面填写学校信息和联系人信息提交申请 |
| S-AP-02 | 查询申请状态 | 通过手机号+验证码查询申请审核进度 |
| S-AP-03 | 重新申请 | 被拒绝后查看拒绝原因，修改信息重新提交 |

### 2.2 用户故事

#### US-01：学校在线申请入驻

```
作为 某高校的区块链课程负责人
我想要 在平台官网提交入驻申请
以便于 让我校师生使用该平台进行区块链教学

验收条件：
1. 平台提供公开的入驻申请页面（无需登录）
2. 填写学校名称、编码、地址、官网、LOGO
3. 填写联系人姓名、手机号、邮箱、职务/部门
4. 提交后显示申请编号，可用于后续查询
5. 申请人可通过手机号+短信验证码查询审核进度
```

#### US-02：超管审核入驻申请

```
作为 超级管理员
我想要 审核学校的入驻申请
以便于 控制平台的学校准入

验收条件：
1. 在申请列表中查看待审核的申请
2. 点击申请可查看完整的学校信息和联系人信息
3. 审核通过：
   - 学校状态变为"已激活"
   - 自动用联系人信息创建该校第一个管理员账号（角色：教师+学校管理员）
   - 管理员账号的初始密码由系统随机生成
   - 通过短信通知联系人：审核通过 + 账号信息 + 初始密码
4. 审核拒绝：
   - 必须填写拒绝原因
   - 申请状态变为"已拒绝"
   - 通过短信通知联系人：审核未通过 + 拒绝原因
```

#### US-03：超管后台直接录入学校

```
作为 超级管理员
我想要 在后台直接创建学校
以便于 线下沟通完成后快速开通学校

验收条件：
1. 填写学校基本信息和联系人信息
2. 设置授权有效期
3. 创建后学校直接为"已激活"状态（跳过审核）
4. 自动创建首个校管账号并短信通知
```

#### US-04：学校管理员配置SSO

```
作为 学校管理员
我想要 配置本校的SSO接入参数
以便于 本校师生可以通过学校统一认证登录平台

验收条件：
1. 选择SSO协议类型（CAS / OAuth2.0）
2. 填写对应的配置参数：
   - CAS：服务端URL、服务回调URL
   - OAuth2：授权端点、Token端点、Client ID、Client Secret、回调URL
3. 保存后可点击"测试连接"验证配置是否正确
4. 测试成功后SSO登录入口对本校用户生效
5. SSO未配置或测试未通过时，本校用户仅能使用手机号+密码登录
```

#### US-05：学校授权到期处理

```
作为 平台系统
我想要 在学校授权到期前后执行相应策略
以便于 保障平台的授权管理

验收条件：
1. 到期前7天：
   - 向该校所有管理员发送到期提醒通知
   - 学校管理后台显示到期提醒横幅
2. 到期当天：
   - 学校状态变为"缓冲期"
   - 该校用户仍可正常使用，但管理后台显示缓冲期倒计时
3. 缓冲期结束（到期后第7天）：
   - 学校状态变为"已冻结"
   - 该校所有用户无法登录
   - 已登录用户的Session立即失效
   - 所有数据保留，不删除
4. 超管可随时解冻学校，恢复正常使用
```

## 三、业务规则与约束

### 3.1 学校入驻规则

| 规则编号 | 规则描述 |
|---------|---------|
| BR-01 | 学校名称全局唯一，不可重复 |
| BR-02 | 学校编码全局唯一（如教育部编码），不可重复 |
| BR-03 | 入驻申请可通过在线表单或超管后台录入两种方式 |
| BR-04 | 同一手机号不可重复提交待审核的申请 |
| BR-05 | 被拒绝的申请可修改后重新提交，生成新的申请记录 |
| BR-06 | 审核通过后自动用联系人信息创建首个校管账号 |
| BR-07 | 首个校管账号角色为「教师 + 学校管理员」 |
| BR-08 | 首个校管的初始密码由系统随机生成，通过短信通知 |

### 3.2 学校管理规则

| 规则编号 | 规则描述 |
|---------|---------|
| BR-09 | 每个学校可有多个管理员，权限相同 |
| BR-10 | 学校管理员由教师兼任（is_school_admin = true） |
| BR-11 | 学校基本信息（LOGO、简介等）由校管维护 |
| BR-12 | SSO配置由校管自行配置和测试 |
| BR-13 | 学校注销为软删除，关联的所有用户同时软删除 |
| BR-14 | 恢复已注销学校时，关联用户同时恢复 |

### 3.3 有效期规则

| 规则编号 | 规则描述 |
|---------|---------|
| BR-15 | 学校必须设置授权有效期 |
| BR-16 | 到期前7天向校管发送提醒通知 |
| BR-17 | 到期后进入7天缓冲期，缓冲期内可正常使用 |
| BR-18 | 缓冲期结束后自动冻结，该校所有用户无法登录 |
| BR-19 | 冻结状态下数据全部保留，超管解冻后立即恢复 |
| BR-20 | 超管可随时手动冻结/解冻任意学校 |

### 3.4 数据隔离规则

| 规则编号 | 规则描述 |
|---------|---------|
| BR-21 | 采用共享数据库 + school_id 字段隔离方案 |
| BR-22 | 所有涉及学校数据的查询必须带 school_id 条件 |
| BR-23 | 校管只能查看和操作本校数据 |
| BR-24 | 超管可跨学校查看和操作所有数据 |

## 四、状态流转图

### 4.1 学校生命周期状态

```
  在线申请 / 超管录入
         │
         ▼
    ┌──────────┐     审核拒绝（填写原因）     ┌──────────┐
    │  待审核   │ ──────────────────────────▶ │  已拒绝   │
    │ (pending) │                              │(rejected)│
    └────┬─────┘                              └────┬─────┘
         │                                         │
    审核通过                                   修改后重新申请
    （自动创建校管）                                │
         │                                         ▼
         ▼                                    ┌──────────┐
    ┌──────────┐                              │  待审核   │
    │  已激活   │◄──── 超管解冻 ──────────┐    │(pending) │
    │ (active)  │                         │    └──────────┘
    └────┬─────┘                         │
         │                               │
    ┌────┴──────────┐                    │
    │               │                    │
  授权到期      超管手动冻结              │
    │               │                    │
    ▼               │                    │
┌──────────┐        │                    │
│  缓冲期   │        │                    │
│(buffering)│        │                    │
└────┬─────┘        │                    │
     │               │                    │
  缓冲期结束          │                    │
  （7天后）           │                    │
     │               │                    │
     ▼               ▼                    │
    ┌──────────────────┐                  │
    │     已冻结        │──────────────────┘
    │   (frozen)       │
    └────┬─────────────┘
         │
     超管注销
         │
         ▼
    ┌──────────┐
    │  已注销   │ ──── 超管恢复 ──▶ 已激活(active)
    │(cancelled)│
    └──────────┘
```
