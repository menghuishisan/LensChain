# 学校与租户管理模块 — 验收标准

> 模块状态：✅ 已确认
> 文档版本：v1.0

---

## 一、功能验收条件

### 1.1 入驻申请

#### AC-01：在线提交入驻申请

```
Given  用户访问入驻申请页面（无需登录）
When   填写完整的学校信息和联系人信息并提交
Then   申请提交成功，显示申请编号
And    申请状态为"待审核"
And    超管后台的申请列表中出现该申请
```

#### AC-02：申请字段校验

```
Given  用户在申请页面填写信息
When   学校名称为空 / 手机号格式错误 / 学校名称已存在
Then   前端实时提示对应错误
And    提交时后端再次校验，返回具体错误信息
```

#### AC-03：重复申请拦截

```
Given  某手机号已有一条待审核的申请
When   使用同一手机号再次提交申请
Then   返回"该手机号已有待审核的申请"
And    引导用户去查询页面查看进度
```

#### AC-04：查询申请状态

```
Given  申请人想查看审核进度
When   输入联系人手机号 + 短信验证码
Then   显示该手机号关联的所有申请记录
And    每条记录显示：学校名称、状态、申请时间、审核时间、拒绝原因（如有）
```

#### AC-05：被拒绝后重新申请

```
Given  某申请状态为"已拒绝"
When   申请人在查询页面点击"修改后重新申请"
Then   跳转到预填了上次信息的申请表单
And    申请人修改信息后提交
Then   生成新的申请记录，关联上一次申请ID
And    新申请状态为"待审核"
```

---

### 1.2 入驻审核

#### AC-06：审核通过

```
Given  超管在申请详情页审核一条待审核的申请
When   设置授权有效期并点击"审核通过"
Then   申请状态变为"已通过"
And    自动创建学校记录，状态为"已激活"
And    自动用联系人信息创建首个校管账号（角色：教师+学校管理员）
And    系统随机生成初始密码
And    通过短信通知联系人：审核通过 + 账号（手机号）+ 初始密码 + 平台地址
And    记录操作日志
```

#### AC-07：审核拒绝

```
Given  超管在申请详情页审核一条待审核的申请
When   填写拒绝原因并点击"拒绝"
Then   申请状态变为"已拒绝"
And    通过短信通知联系人：审核未通过 + 拒绝原因
And    拒绝原因不能为空
```

#### AC-08：超管后台直接创建学校

```
Given  超管在学校管理页面点击"创建学校"
When   填写学校信息、联系人信息、授权有效期并提交
Then   学校直接创建为"已激活"状态（跳过审核流程）
And    自动创建首个校管账号并短信通知
And    记录操作日志
```

---

### 1.3 学校管理

#### AC-09：冻结学校

```
Given  超管在学校列表选择一个已激活的学校
When   点击"冻结"并填写原因
Then   学校状态变为"已冻结"
And    该校所有用户的Session立即失效
And    该校用户尝试登录时返回"学校授权已过期，请联系管理员"
And    该校所有数据保留不删除
And    记录操作日志
```

#### AC-10：解冻学校

```
Given  超管选择一个已冻结的学校
When   点击"解冻"
Then   学校状态恢复为"已激活"
And    该校用户可正常登录
And    记录操作日志
```

#### AC-11：注销学校

```
Given  超管选择一个学校点击"注销"
When   二次确认后执行注销
Then   学校状态变为"已注销"，设置 deleted_at
And    该校所有用户同时软删除
And    该校所有用户Session立即失效
And    记录操作日志
```

#### AC-12：恢复已注销学校

```
Given  超管选择一个已注销的学校
When   点击"恢复"
Then   学校 deleted_at 清除，状态恢复为"已激活"
And    该校所有用户同时恢复（清除 deleted_at）
And    记录操作日志
```

### 1.4 授权有效期

#### AC-13：到期提醒

```
Given  某学校授权有效期将在7天内到期
When   定时任务执行到期检查（每天凌晨1:00）
Then   向该校所有管理员发送短信提醒
And    校管登录后台后顶部显示黄色到期提醒横幅
And    提醒仅发送一次，不重复发送
```

#### AC-14：到期转缓冲期

```
Given  某学校授权有效期已过
When   定时任务执行到期检查（每天凌晨0:30）
Then   学校状态自动变为"缓冲期"
And    该校用户仍可正常使用
And    校管后台显示红色缓冲期倒计时横幅
```

#### AC-15：缓冲期转冻结

```
Given  某学校处于缓冲期且已满7天
When   定时任务执行缓冲期检查（每天凌晨2:00）
Then   学校状态自动变为"已冻结"
And    该校所有用户Session立即失效
And    该校用户无法登录
```

### 1.5 SSO配置

#### AC-16：校管配置SSO

```
Given  校管进入SSO配置页面
When   选择协议类型，填写配置参数并保存
Then   配置保存成功
And    client_secret 加密存储
And    is_tested 重置为 false
And    SSO登录入口暂不对本校用户生效（需先测试通过）
```

#### AC-17：测试SSO连接

```
Given  校管已保存SSO配置
When   点击"测试连接"
Then   后端尝试连接SSO服务器
And    成功：显示"连接成功"，is_tested 设为 true
And    失败：显示具体错误信息，is_tested 保持 false
```

#### AC-18：启用SSO

```
Given  SSO配置已保存且测试通过
When   校管点击"启用SSO"
Then   is_enabled 设为 true
And    本校用户在登录页可看到SSO登录入口
And    SSO学校选择列表中出现本校
```

## 二、性能指标

| 指标 | 要求 |
|------|------|
| 学校列表查询响应时间 | ≤ 500ms（P95） |
| 入驻申请提交响应时间 | ≤ 1s（P95） |
| 审核通过处理时间 | ≤ 3s（含创建学校+创建用户+发短信） |
| SSO测试连接超时时间 | ≤ 10s |
| 定时任务执行时间 | ≤ 30s（100所学校规模） |
| 学校状态缓存命中率 | ≥ 95% |

## 三、安全要求

| 要求 | 说明 |
|------|------|
| SSO密钥加密 | client_secret 等敏感字段使用AES-256加密存储 |
| SSO密钥脱敏 | API返回时 client_secret 显示为 `******` |
| 申请查询验证 | 查询申请状态需手机号+短信验证码双重验证 |
| 数据隔离 | 校管只能查看和操作本校数据，中间件层强制过滤 |
| 注销二次确认 | 注销学校需传入 confirm=true，防止误操作 |
| 操作审计 | 所有学校状态变更操作记录操作日志 |
| 短信防刷 | 同一手机号60秒内不可重复发送验证码 |

## 四、边界条件与异常处理

| 场景 | 处理方式 |
|------|---------|
| 学校名称已存在 | 提交申请时返回"该学校名称已存在" |
| 学校编码已存在 | 提交申请时返回"该学校编码已存在" |
| 审核已通过的申请再次审核 | 返回"该申请已审核" |
| 冻结已冻结的学校 | 返回"该学校已处于冻结状态" |
| 解冻非冻结状态的学校 | 返回"仅冻结状态的学校可解冻" |
| 注销已注销的学校 | 返回"该学校已注销" |
| SSO配置不完整时测试 | 返回"请先完善SSO配置" |
| SSO服务器不可达 | 测试连接返回具体错误："无法连接到XXX端点" |
| 短信发送失败 | 记录失败日志，不阻塞审核流程，后台可手动重发 |
| 定时任务执行异常 | 记录错误日志，下次执行时重试 |
| 联系人手机号与已有用户冲突 | 创建校管时检测，冲突则提示超管手动处理 |
