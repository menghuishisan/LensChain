## 一、ER 关系概览

```
schools (学校主表)
  │
  ├── 1:N ── school_applications (入驻申请记录)
  │
  ├── 1:1 ── school_sso_configs (SSO配置)
  │
  ├── 1:N ── users (用户表，定义在模块01，通过 school_id 关联)
  │
  └── 1:N ── school_notifications (学校生命周期通知记录)
```

## 二、表结构设计

### 2.1 schools — 学校主表

> 存储所有入驻学校的核心信息和授权状态。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| name | VARCHAR(100) | NOT NULL | 学校名称 |
| code | VARCHAR(50) | NOT NULL | 学校编码（如教育部编码） |
| logo_url | VARCHAR(500) | NULL | 学校LOGO地址 |
| address | VARCHAR(200) | NULL | 学校地址 |
| website | VARCHAR(200) | NULL | 学校官网 |
| description | TEXT | NULL | 学校简介 |
| status | SMALLINT | NOT NULL DEFAULT 1 | 状态：1待审核 2已激活 3缓冲期 4已冻结 5已注销 6已拒绝 |
| license_start_at | TIMESTAMP | NULL | 授权开始时间 |
| license_end_at | TIMESTAMP | NULL | 授权结束时间 |
| frozen_at | TIMESTAMP | NULL | 冻结时间 |
| frozen_reason | VARCHAR(200) | NULL | 冻结原因（手动冻结时填写） |
| contact_name | VARCHAR(50) | NOT NULL | 联系人姓名 |
| contact_phone | VARCHAR(20) | NOT NULL | 联系人手机号 |
| contact_email | VARCHAR(100) | NULL | 联系人邮箱 |
| contact_title | VARCHAR(100) | NULL | 联系人职务/部门 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |
| created_by | BIGINT | NULL | 创建人ID（超管录入时记录） |
| deleted_at | TIMESTAMP | NULL | 软删除时间（注销时设置） |

**索引：**

```sql
CREATE UNIQUE INDEX uk_schools_name ON schools(name);
CREATE UNIQUE INDEX uk_schools_code ON schools(code);
CREATE INDEX idx_schools_status ON schools(status);
CREATE INDEX idx_schools_license_end_at ON schools(license_end_at);
CREATE INDEX idx_schools_contact_phone ON schools(contact_phone);
```

> 学校名称和学校编码按业务规则要求全局唯一，即使学校已注销也不可复用。

**状态枚举：**

```go
const (
    SchoolStatusPending   = 1  // 待审核
    SchoolStatusActive    = 2  // 已激活
    SchoolStatusBuffering = 3  // 缓冲期
    SchoolStatusFrozen    = 4  // 已冻结
    SchoolStatusCancelled = 5  // 已注销
    SchoolStatusRejected  = 6  // 已拒绝
)
```

### 2.2 school_applications — 入驻申请记录表

> 记录所有入驻申请的历史，包括被拒绝后重新申请的记录。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| school_name | VARCHAR(100) | NOT NULL | 申请的学校名称 |
| school_code | VARCHAR(50) | NOT NULL | 申请的学校编码 |
| school_address | VARCHAR(200) | NULL | 学校地址 |
| school_website | VARCHAR(200) | NULL | 学校官网 |
| school_logo_url | VARCHAR(500) | NULL | 学校LOGO |
| contact_name | VARCHAR(50) | NOT NULL | 联系人姓名 |
| contact_phone | VARCHAR(20) | NOT NULL | 联系人手机号 |
| contact_email | VARCHAR(100) | NULL | 联系人邮箱 |
| contact_title | VARCHAR(100) | NULL | 联系人职务/部门 |
| status | SMALLINT | NOT NULL DEFAULT 1 | 状态：1待审核 2已通过 3已拒绝 |
| reviewer_id | BIGINT | NULL | 审核人ID（超管） |
| reviewed_at | TIMESTAMP | NULL | 审核时间 |
| reject_reason | VARCHAR(500) | NULL | 拒绝原因 |
| school_id | BIGINT | NULL | 审核通过后关联的学校ID |
| previous_application_id | BIGINT | NULL | 重新申请时关联的上一次申请ID |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 申请时间 |

**索引：**

```sql
CREATE INDEX idx_school_applications_status ON school_applications(status);
CREATE INDEX idx_school_applications_contact_phone ON school_applications(contact_phone);
CREATE INDEX idx_school_applications_created_at ON school_applications(created_at);
```

**状态枚举：**

```go
const (
    ApplicationStatusPending  = 1  // 待审核
    ApplicationStatusApproved = 2  // 已通过
    ApplicationStatusRejected = 3  // 已拒绝
)
```

### 2.3 school_sso_configs — SSO配置表

> 每个学校独立的SSO接入配置，支持CAS和OAuth2.0两种协议。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| school_id | BIGINT | NOT NULL, FK, UNIQUE | 关联学校ID |
| provider | VARCHAR(20) | NOT NULL | SSO协议类型：cas / oauth2 |
| is_enabled | BOOLEAN | NOT NULL DEFAULT FALSE | 是否启用 |
| is_tested | BOOLEAN | NOT NULL DEFAULT FALSE | 是否已通过连接测试 |
| config | JSONB | NOT NULL | SSO配置参数（JSON格式，见下方说明） |
| tested_at | TIMESTAMP | NULL | 最后测试时间 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |
| updated_by | BIGINT | NULL | 最后修改人ID |

**索引：**

```sql
CREATE UNIQUE INDEX uk_school_sso_configs_school_id ON school_sso_configs(school_id);
```

**config 字段结构（CAS协议）：**

```json
{
  "cas_server_url": "https://cas.example.edu/cas",
  "cas_service_url": "https://lianjing.com/api/v1/auth/sso/callback",
  "cas_version": "3.0",
  "user_id_attribute": "student_id"
}
```

**config 字段结构（OAuth2.0协议）：**

```json
{
  "authorize_url": "https://oauth.example.edu/authorize",
  "token_url": "https://oauth.example.edu/token",
  "userinfo_url": "https://oauth.example.edu/userinfo",
  "client_id": "lianjing_client",
  "client_secret": "encrypted_secret_value",
  "redirect_uri": "https://lianjing.com/api/v1/auth/sso/callback",
  "scope": "openid profile",
  "user_id_attribute": "student_id"
}
```

> **安全说明**：`client_secret` 等敏感字段在数据库中加密存储，API返回时脱敏显示。

### 2.4 school_notifications — 学校通知记录表

> 记录**已存在学校租户**的生命周期通知流水（到期提醒、缓冲期、冻结、审核通过后开通通知等）。
> 该表要求 `school_id` 必填，因此**不承载“审核拒绝”这类仍处于申请阶段、尚未创建学校记录的外部联系人通知**。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| school_id | BIGINT | NOT NULL, FK | 关联学校ID |
| type | SMALLINT | NOT NULL | 通知类型：1到期提醒 2缓冲期通知 3冻结通知 4审核通过 |
| title | VARCHAR(200) | NOT NULL | 通知标题 |
| content | TEXT | NOT NULL | 通知内容 |
| is_sent | BOOLEAN | NOT NULL DEFAULT FALSE | 是否已发送 |
| sent_at | TIMESTAMP | NULL | 发送时间 |
| target_phone | VARCHAR(20) | NULL | 短信通知目标手机号 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |

**索引：**

```sql
CREATE INDEX idx_school_notifications_school_id ON school_notifications(school_id);
CREATE INDEX idx_school_notifications_type ON school_notifications(type);
CREATE INDEX idx_school_notifications_is_sent ON school_notifications(is_sent);
```

## 三、Redis 数据结构

### 3.1 学校状态缓存

```
# 学校状态缓存（登录时快速判断学校是否可用）
Key:   school:status:{school_id}
Value: { "status": 2, "license_end_at": "2027-04-07T00:00:00Z" }
TTL:   1小时（状态变更时主动刷新）
```

### 3.2 到期检查任务

```
# 需要检查到期的学校ID集合（定时任务使用）
Key:   school:expiring_soon
Value: Sorted Set，score为到期时间戳，member为school_id
TTL:   无过期（定时任务维护）
```

## 四、定时任务

| 任务 | 执行频率 | 说明 |
|------|---------|------|
| 到期提醒检查 | 每天凌晨1:00 | 检查7天内即将到期的学校，发送提醒通知 |
| 缓冲期转冻结 | 每天凌晨2:00 | 检查缓冲期已满7天的学校，自动转为冻结状态 |
| 到期转缓冲期 | 每天凌晨0:30 | 检查已到期的激活学校，转为缓冲期状态 |
