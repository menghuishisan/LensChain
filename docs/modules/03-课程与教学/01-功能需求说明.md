# 课程与教学模块 — 功能需求说明

> 模块状态：✅ 已确认
> 文档版本：v1.0

---

## 一、模块概述

### 1.1 模块职责

课程与教学模块是平台的核心业务模块，负责：

- 课程的创建、编辑、发布、归档等全生命周期管理
- 课程内容组织（章节 → 课时，支持多种内容形式）
- 学生选课与课程加入管理（教师指定 + 邀请码）
- 作业与测验管理（客观题、主观题、编程题、实验报告）
- 学习进度跟踪（课时完成、视频进度、学习时长）
- 课程讨论区与公告
- 课程评价（评分 + 评论）
- 单课程成绩计算与管理（可配置权重、成绩汇总、手动调整、Excel导出）
- 课程共享与克隆
- 课程表管理
- 课程统计与数据导出

### 1.2 模块边界

| 本模块负责 | 本模块不负责 |
|-----------|-------------|
| 课程CRUD、内容管理、发布流程 | 用户身份认证（→ 模块01） |
| 作业/测验的题目管理和提交 | 学校/租户管理（→ 模块02） |
| 学习进度记录 | 实验环境的创建与编排（→ 模块04） |
| 课程讨论区 | CTF竞赛（→ 模块05） |
| 成绩权重配置、单课程成绩汇总、手动调整、Excel导出 | 成绩的最终审核、GPA、跨课程汇总、成绩单PDF（→ 模块06） |
| 课程共享与克隆 | 全局通知推送（→ 模块07） |
| 课程统计数据展示 | 系统资源监控（→ 模块08） |
| 课程内公告 | |

### 1.3 依赖关系

```
本模块依赖：
  → 01-用户与认证（教师、学生身份）
  → 02-学校与租户管理（课程归属学校）
  → 04-实验环境（课时/作业关联实验环境模板）

本模块被以下模块依赖：
  ← 04-实验环境（实验可关联到课程）
  ← 06-评测与成绩（成绩数据来源）
```

### 2.2 用户故事

#### US-01：教师创建并发布课程

```
作为 教师
我想要 创建一门区块链课程并发布给学生
以便于 开展本学期的教学工作

验收条件：
1. 填写课程名称、描述、类型（理论/实验/混合/项目）、难度、主题分类
2. 上传封面图或使用平台自动生成的封面
3. 设置课程开始/结束时间、所属学期、课程学分
4. 设置每周上课时间（课程表）
5. 创建章节和课时，上传/编写内容
6. 课程保存为草稿，可随时编辑
7. 点击发布后课程状态变为"已发布"
8. 到达开始时间后状态自动变为"进行中"
9. 生成邀请码供学生加入
```

#### US-02：学生通过邀请码加入课程

```
作为 学生
我想要 通过教师提供的邀请码加入课程
以便于 开始学习课程内容

验收条件：
1. 在"加入课程"页面输入邀请码
2. 系统验证邀请码有效性（课程存在、未结束、邀请码正确）
3. 加入成功后课程出现在"我的课程"列表中
4. 可立即开始浏览课程内容
```

#### US-03：学生学习课时内容

```
作为 学生
我想要 按顺序学习课程的每个课时
以便于 系统地掌握区块链知识

验收条件：
1. 进入课程后看到章节→课时的树形目录
2. 每个课时显示完成状态图标（未开始/进行中/已完成）
3. 点击课时进入学习页面，根据内容类型展示：
   - 视频：播放器，支持倍速、进度条、续播
   - 图文：Markdown渲染的教程内容
   - 附件：文件列表，可预览或下载
   - 实验：显示实验描述和"启动实验环境"按钮
4. 视频观看完成/图文阅读到底部/实验完成后，课时标记为已完成
5. 学习时长实时记录
```

#### US-04：教师创建作业并批改

```
作为 教师
我想要 为课程创建作业并批改学生提交
以便于 检验学生的学习效果

验收条件：
1. 创建作业：设置标题、关联章节、截止时间、最大提交次数
2. 添加不同题型：客观题（自动批改）、主观题、编程题、实验报告
3. 设置每题分值
4. 发布作业后学生可见
5. 客观题提交后即时显示得分
6. 主观题和实验报告需教师手动批改：给分 + 评语
7. 编程题判题方式留到模块04定义
8. 教师可查看每个学生的提交历史和各次提交的详情
```

#### US-05：迟交处理

```
作为 教师
我想要 配置作业的迟交策略
以便于 灵活处理学生的迟交情况

验收条件：
1. 创建作业时可设置迟交策略：
   - 不允许迟交（截止后无法提交）
   - 允许迟交，扣分比例可配置（如每天扣10%）
   - 允许迟交，不扣分（仅标记为迟交）
2. 迟交的提交在教师端明确标记为"迟交"
3. 设置了扣分策略的，系统自动计算扣分后的成绩
```

#### US-06：课程成绩计算

```
作为 教师
我想要 配置课程成绩的计算规则
以便于 自动生成学生的课程总成绩

验收条件：
1. 在成绩设置中配置各项权重（如：作业1占20%、作业2占20%、实验占30%、期末占30%）
2. 所有权重之和必须等于100%
3. 系统根据权重自动计算每个学生的总成绩
4. 教师可手动调整某学生的最终成绩（覆盖自动计算值）
5. 可导出成绩单为Excel
```

#### US-07：课程克隆

```
作为 教师
我想要 克隆一门已有课程作为新学期的基础
以便于 不用从头创建课程内容

验收条件：
1. 可克隆自己的课程或共享课程库中的课程
2. 克隆内容包括：课程信息、章节结构、课时内容、作业题目
3. 克隆不包括：学生名单、提交记录、成绩数据、讨论帖子
4. 克隆后生成新的草稿课程，教师可自由修改
5. 克隆课程与原课程完全独立，互不影响
```

#### US-08：课程共享

```
作为 教师
我想要 将我的课程共享到平台共享课程库
以便于 其他学校的教师可以参考和使用

验收条件：
1. 在课程设置中开启"共享到课程库"
2. 共享后课程出现在全平台的共享课程库中
3. 其他教师可浏览共享课程的简介和目录结构
4. 其他教师可克隆共享课程到自己的学校
5. 教师可随时取消共享
```

## 三、业务规则与约束

### 3.1 课程规则

| 规则编号 | 规则描述 |
|---------|---------|
| BR-01 | 课程归属创建它的教师和学校 |
| BR-02 | 课程类型：理论课(1)、实验课(2)、混合课(3)、项目实战课(4) |
| BR-03 | 课程难度：入门(1)、进阶(2)、高级(3)、研究(4) |
| BR-04 | 课程分类主题由平台预设（区块链基础、智能合约、DApp开发、底层原理、密码学、联盟链等） |
| BR-05 | 课程封面：教师自定义上传，未上传则平台根据主题自动生成 |
| BR-06 | 一门课程仅有一个负责教师 |
| BR-07 | 邀请码为6位字母数字组合，教师可刷新重新生成 |
| BR-08 | 仅草稿状态的课程可删除（软删除） |

### 3.2 课程状态规则

| 规则编号 | 规则描述 |
|---------|---------|
| BR-09 | 课程状态：草稿(1) → 已发布(2) → 进行中(3) → 已结束(4) → 已归档(5) |
| BR-10 | 草稿 → 已发布：教师手动发布，至少有一个章节和课时 |
| BR-11 | 已发布 → 进行中：到达课程开始时间后自动转换 |
| BR-12 | 进行中 → 已结束：到达课程结束时间自动转换，或教师手动结束 |
| BR-13 | 已结束 → 已归档：教师手动归档 |
| BR-14 | 已归档课程学生不可见，仅教师可查看和导出数据 |
| BR-15 | 已发布/进行中的课程可编辑内容，但不能删除已有学生提交过的作业 |
| BR-15A | 学生名单管理（教师添加/移除学生、学生通过邀请码加入）仅在已发布/进行中阶段允许；草稿课程学生不可见，无需管理 |

### 3.3 作业规则

| 规则编号 | 规则描述 |
|---------|---------|
| BR-16 | 作业有独立的截止时间 |
| BR-17 | 学生可在截止前多次提交，以最后一次为准（受最大次数限制） |
| BR-18 | 迟交策略由教师在创建作业时配置：不允许/允许扣分/允许不扣分 |
| BR-19 | 客观题提交后即时自动批改并反馈 |
| BR-20 | 主观题和实验报告需教师手动批改 |
| BR-21 | 编程题判题方式在模块04中定义 |

### 3.4 成绩规则

| 规则编号 | 规则描述 |
|---------|---------|
| BR-22 | 教师可配置各作业/实验在总成绩中的权重百分比 |
| BR-23 | 所有权重之和必须等于100% |
| BR-24 | 系统根据权重自动计算总成绩 |
| BR-25 | 教师可手动覆盖某学生的最终成绩 |
| BR-26 | 手动调整的成绩标记为"已调整"，保留自动计算值作为参考 |
| BR-26A | 单课程成绩汇总、手动调整、Excel导出属于模块03职责 |
| BR-26B | 课程学分 `credits` 和所属学期 `semester_id` 由模块03维护，模块06只读使用 |
| BR-26C | 手动调整成绩需持久化保存到 `course_grade_overrides`，用于后续查询与导出 |

### 3.5 共享与克隆规则

| 规则编号 | 规则描述 |
|---------|---------|
| BR-27 | 仅已发布/进行中/已结束状态的课程可共享 |
| BR-28 | 共享课程对全平台所有教师可见 |
| BR-29 | 克隆生成独立的草稿课程，与原课程无关联 |
| BR-30 | 克隆包含：课程信息、章节、课时内容、作业题目 |
| BR-31 | 克隆不包含：学生名单、提交记录、成绩、讨论、公告 |

```
  ┌──────────┐     教师批改     ┌──────────┐
  │  待批改   │ ─────────────▶ │  已批改   │
  │(reviewing)│                │ (graded)  │
  └──────────┘                └──────────┘

  注：截止时间后提交的标记为"迟交"(late)
  注：学生可在截止前重新提交（受次数限制），状态回到"已提交"
```

### 4.3 作业草稿保存与恢复

1. 学生进入作业作答页后，前端先尝试从 localStorage 读取本地草稿
2. 前端调用服务端草稿接口获取当前学生在该作业下的最新草稿
3. 前端按“服务端草稿优先，本地草稿兜底”的原则恢复作答内容
4. 学生作答过程中每60秒自动触发一次草稿保存
5. 学生也可手动点击“保存草稿”立即保存
6. 草稿保存时同时写入 localStorage 和服务端，服务端按“单学生单作业一份草稿”覆盖更新
7. 网络中断时仅保存到 localStorage，网络恢复后由前端补偿同步到服务端
8. 草稿不计入提交次数，不参与成绩统计，不触发自动判分
9. 作业正式提交后，服务端草稿立即失效；后续重新进入作答页时不再以旧草稿作为默认作答内容
