# 课程与教学模块 — 验收标准

> 模块状态：✅ 已确认
> 文档版本：v1.0

---

## 一、功能验收条件

### AC-01：创建课程

```
Given 教师已登录系统
When 填写课程名称、类型、难度、主题分类并提交
Then 系统创建草稿状态课程，自动生成6位邀请码和默认封面
And 课程出现在教师的课程列表中
```

### AC-02：课程发布

```
Given 教师拥有一门草稿状态的课程
When 课程至少有1个章节且章节下至少有1个课时，教师点击发布
Then 课程状态变为"已发布"，学生可通过邀请码加入
```

```
Given 教师拥有一门草稿状态的课程
When 课程没有任何章节或课时，教师点击发布
Then 系统拒绝发布，提示"请至少添加一个章节和一个课时"
```

### AC-03：课程状态自动流转

```
Given 课程状态为"已发布"，设置了开始时间
When 系统时间到达课程开始时间
Then 定时任务自动将课程状态更新为"进行中"
```

```
Given 课程状态为"进行中"，设置了结束时间
When 系统时间到达课程结束时间
Then 定时任务自动将课程状态更新为"已结束"
```

### AC-04：课程手动结束与归档

```
Given 课程状态为"进行中"
When 教师手动点击"结束课程"
Then 课程状态变为"已结束"，学生不可再提交作业
```

```
Given 课程状态为"已结束"
When 教师手动点击"归档课程"
Then 课程状态变为"已归档"，学生不可见，仅教师可查看和导出数据
```

### AC-05：课程删除

```
Given 课程状态为"草稿"
When 教师点击删除课程
Then 课程被软删除，不再出现在任何列表中
```

```
Given 课程状态不是"草稿"（已发布/进行中/已结束/已归档）
When 教师尝试删除课程
Then 系统拒绝操作，提示"仅草稿状态的课程可删除"
```

### AC-06：通过邀请码加入课程

```
Given 学生已登录，课程状态为"已发布"或"进行中"
When 学生输入正确的6位邀请码
Then 学生成功加入课程，课程出现在"我的课程"列表中
```

```
Given 学生已加入某课程
When 学生再次输入该课程邀请码
Then 系统提示"您已加入该课程"
```

```
Given 课程状态为"已结束"或"已归档"
When 学生输入该课程邀请码
Then 系统拒绝加入，提示"课程已结束，无法加入"
```

```
Given 课程设置了最大学生数且已满
When 新学生输入该课程邀请码
Then 系统拒绝加入，提示"课程人数已满"
```

### AC-07：章节与课时管理

```
Given 教师进入课程内容管理页
When 创建章节，填写章节标题并保存
Then 章节出现在内容列表中，可继续在其下创建课时
```

```
Given 教师在某章节下
When 创建课时，选择内容类型（视频/图文/附件/实验），填写标题并保存
Then 课时出现在章节下，可进入编辑页完善内容
```

```
Given 教师在内容管理页
When 拖拽章节或课时到新位置
Then 排序立即更新，刷新页面后顺序保持不变
```

### AC-08：视频课时学习

```
Given 学生进入视频类型的课时学习页
When 播放视频
Then 视频播放器支持倍速（0.5x/1x/1.5x/2x）、进度拖拽、全屏播放
And 前端每30秒自动上报播放进度
And 页面离开时立即上报当前进度
```

```
Given 学生之前观看了某视频到15:30处并离开
When 再次进入该课时
Then 视频自动从15:30处续播
```

```
Given 学生观看视频
When 播放进度达到视频总时长的95%以上
Then 该课时自动标记为"已完成"
```

### AC-09：实验课时学习

```
Given 学生进入实验类型的课时学习页
When 页面加载完成
Then 显示实验说明（名称、时长、目标、步骤）和"启动实验环境"按钮
```

```
Given 学生点击"启动实验环境"
When 实验环境启动成功
Then 页面跳转或嵌入实验操作界面（具体实现在模块04定义）
```

### AC-10：创建作业

```
Given 教师进入课程作业管理页
When 创建作业，填写标题、截止时间、最大提交次数、迟交策略，添加至少一道题目
Then 作业以"未发布"状态保存，教师可继续编辑
```

```
Given 作业至少包含一道题目
When 教师点击"发布作业"
Then 作业状态变为"已发布"，学生可见并可作答
```

### AC-11：作业题目管理

```
Given 教师在作业编辑页
When 添加客观题（单选/多选/判断/填空），设置题目内容、选项、正确答案、分值
Then 题目保存成功，总分自动累加
```

```
Given 教师在作业编辑页
When 添加主观题（简答/论述），设置题目内容、参考答案、分值
Then 题目保存成功，标记为"需手动批改"
```

```
Given 教师在作业编辑页
When 添加编程题，设置题目描述和分值
Then 题目保存成功（判题配置留到模块04定义）
```

```
Given 教师在作业编辑页
When 添加实验报告题，设置要求说明和分值
Then 题目保存成功，支持学生上传PDF/Word文件
```

### AC-12：学生提交作业

```
Given 学生进入已发布的作业作答页，作业未截止
When 作答所有题目后点击"提交作业"
Then 系统保存提交记录，客观题即时自动批改并反馈得分
And 主观题和实验报告显示"待批改"状态
And 显示剩余提交次数
```

```
Given 学生作答过程中
When 每隔60秒
Then 系统自动保存草稿到localStorage和服务端
```

```
Given 学生在作答页面有未保存内容
When 尝试离开页面
Then 浏览器弹出确认提示"有未保存的作答，是否离开？"
```

### AC-13：作业重新提交

```
Given 学生已提交作业，剩余提交次数 > 0，作业未截止
When 学生重新作答并提交
Then 系统保存新的提交记录，submission_no递增
And 以最后一次提交为最终成绩依据
```

```
Given 学生已达最大提交次数
When 尝试再次提交
Then 系统拒绝，提示"已达最大提交次数"
```

### AC-14：迟交处理

```
Given 作业迟交策略为"不允许迟交"
When 截止时间后学生尝试提交
Then 系统拒绝提交，提示"作业已截止且不允许迟交"
```

```
Given 作业迟交策略为"允许迟交，每天扣10%"
When 截止时间后2天学生提交
Then 系统接受提交，标记为迟交，is_late=true，late_days=2
And 扣分后得分 = 原始得分 × (1 - 10% × 2)
```

```
Given 作业迟交策略为"允许迟交，不扣分"
When 截止时间后学生提交
Then 系统接受提交，标记为迟交但不扣分
```

### AC-15：教师批改作业

```
Given 教师进入某学生的提交详情页
When 对主观题逐题批改：给分（不超过题目分值）+ 评语，最后提交批改结果
Then 提交状态变为"已批改"，总得分自动计算
And 学生可查看批改结果和教师评语
```

### AC-16：成绩权重配置与计算

```
Given 教师进入成绩管理页
When 配置各作业的权重百分比（如作业1占20%、作业2占30%、期末占50%）
Then 所有权重之和必须等于100%，否则提示"权重总和必须为100%"
And 保存后系统自动根据权重计算每个学生的总成绩
```

```
Given 教师查看成绩汇总表
When 手动修改某学生的最终成绩
Then 该学生成绩标记为"已调整"，保留自动计算值作为参考
```

### AC-17：成绩导出

```
Given 教师在成绩管理页
When 点击"导出成绩单"
Then 系统生成Excel文件并下载
And 文件包含所有学生的各项成绩和加权总成绩
```

### AC-18：学习进度跟踪

```
Given 学生正在学习课程
When 查看课程学习主页
Then 显示总体进度百分比（已完成课时数/总课时数）
And 显示累计学习时长
And 每个课时显示完成状态（未开始/进行中/已完成）
```

```
Given 教师查看学生管理页
When 查看某学生的学习进度
Then 显示该学生每个课时的完成状态和学习时长
```

### AC-19：课程讨论区

```
Given 学生或教师进入课程讨论区
When 发帖，填写标题和内容（Markdown格式）
Then 帖子出现在讨论区列表中，按最新回复时间排序
And 置顶帖始终在列表最前
```

```
Given 课程教师
When 点击某帖子的"置顶"按钮
Then 该帖子被置顶，置顶状态可切换
```

```
Given 用户查看帖子
When 点击点赞按钮
Then 点赞数+1，再次点击取消点赞
```

### AC-20：课程公告

```
Given 课程教师
When 发布公告，填写标题和内容
Then 公告出现在课程公告列表中，所有课程学生可见
```

### AC-21：课程克隆

```
Given 教师查看自己的课程或共享课程库中的课程
When 点击"克隆课程"
Then 系统创建新的草稿课程，包含：课程基本信息、章节结构、课时内容、作业题目
And 不包含：学生名单、提交记录、成绩数据、讨论帖子、公告
And 新课程与原课程完全独立
```

### AC-22：课程共享

```
Given 教师的课程状态为"已发布"/"进行中"/"已结束"
When 在课程设置中开启"共享到课程库"
Then 课程出现在全平台共享课程库中，其他教师可浏览和克隆
```

```
Given 课程状态为"草稿"
When 教师尝试开启共享
Then 系统拒绝，提示"仅已发布/进行中/已结束的课程可共享"
```

### AC-23：课程表

```
Given 教师在课程设置页
When 设置每周上课时间段（星期几、开始时间、结束时间、地点）
Then 课程表保存成功
And 学生在"我的课程表"中可见
```

```
Given 学生查看"我的课程表"
When 页面加载完成
Then 以周视图形式展示所有已加入课程的上课时间安排
And 点击课程块可跳转到对应课程学习页
```

### AC-24：课程评价

```
Given 课程状态为"已结束"
When 学生提交评价（1-5星评分 + 评论内容）
Then 评价保存成功，教师可在评价列表中查看
And 每个学生对同一课程只能评价一次，可修改
```

### AC-25：课程统计

```
Given 教师进入课程统计页
When 页面加载完成
Then 显示课程整体数据：学生数、课时数、完课率、平均分、活跃度
And 显示学习进度分布（未开始/进行中/已完成的学生占比）
And 显示作业统计（每次作业的提交率、平均分、分数分布）
```

### AC-26：课程状态对操作的限制

```
Given 课程状态为"草稿"
When 教师操作
Then 教师可编辑所有内容；学生不可见该课程
```

```
Given 课程状态为"已发布"
When 教师/学生操作
Then 教师可编辑内容和管理学生；学生可加入和浏览内容
```

```
Given 课程状态为"进行中"
When 教师/学生操作
Then 教师可编辑内容和批改作业；学生可学习和提交作业
And 教师不可删除已有学生提交过的作业
```

```
Given 课程状态为"已结束"
When 教师/学生操作
Then 教师可批改作业和查看数据；学生可查看内容和成绩，不可提交作业
```

```
Given 课程状态为"已归档"
When 教师/学生操作
Then 教师可查看和导出数据；学生不可见该课程
```

## 二、性能指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 课程列表加载 | ≤ 500ms | 含分页，单页20条 |
| 课程详情加载 | ≤ 500ms | 含章节和课时目录 |
| 章节课时目录加载 | ≤ 300ms | 一次性加载完整目录树 |
| 视频进度上报 | ≤ 200ms | 每30秒一次，不阻塞播放 |
| 作业提交 | ≤ 1s | 含客观题即时批改反馈 |
| 作业草稿自动保存 | ≤ 500ms | 每60秒一次，不阻塞作答 |
| 成绩汇总表加载 | ≤ 1s | 50人课程 |
| 成绩导出 | ≤ 5s | 50人课程生成Excel |
| 讨论区列表加载 | ≤ 500ms | 含分页，单页20条 |
| 课程克隆 | ≤ 10s | 含完整章节、课时、作业题目的复制 |
| 学习进度查询 | ≤ 500ms | 单个学生的课程进度 |
| 课程统计数据 | ≤ 2s | 含完课率、活跃度等聚合计算 |
| 拖拽排序响应 | ≤ 300ms | 章节/课时排序操作 |

## 三、安全要求

| 要求 | 说明 |
|------|------|
| 课程数据隔离 | 教师只能管理自己创建的课程，学生只能访问已加入的课程 |
| school_id 隔离 | 所有课程数据带 school_id，跨学校不可访问 |
| 权限校验 | 所有教师端API校验请求者是否为课程负责教师 |
| 学生数据保护 | 学生间不可查看彼此的提交内容和成绩 |
| 邀请码安全 | 邀请码6位字母数字组合，支持刷新重新生成 |
| 文件上传安全 | 附件上传校验文件类型和大小（视频≤500MB，文档≤50MB） |
| XSS 防护 | Markdown内容渲染时过滤危险HTML标签和脚本 |
| SQL 注入防护 | 所有数据库操作使用参数化查询 |
| CSRF 防护 | 所有写操作接口校验 CSRF Token 或依赖 JWT Bearer 机制 |
| 接口限流 | 视频进度上报接口限流（每用户每分钟最多3次） |
| 作业提交防刷 | 提交接口校验剩余提交次数，服务端二次校验 |

## 四、边界条件与异常处理

| 场景 | 处理方式 |
|------|---------|
| 课程无章节时发布 | 拒绝发布，提示"请至少添加一个章节和一个课时" |
| 课程无课时时发布 | 拒绝发布，提示"请至少添加一个章节和一个课时" |
| 邀请码输入错误 | 提示"邀请码无效" |
| 重复加入课程 | 提示"您已加入该课程" |
| 课程人数已满 | 提示"课程人数已满" |
| 已结束课程加入 | 提示"课程已结束，无法加入" |
| 已截止作业提交（不允许迟交） | 拒绝提交，提示"作业已截止且不允许迟交" |
| 超过最大提交次数 | 拒绝提交，提示"已达最大提交次数" |
| 未发布作业学生访问 | 返回404或提示"作业未发布" |
| 批改给分超过题目分值 | 前端限制，后端校验拒绝 |
| 成绩权重总和≠100% | 拒绝保存，提示"权重总和必须为100%" |
| 删除已有提交的作业 | 拒绝删除，提示"该作业已有学生提交，不可删除" |
| 视频文件上传超限 | 提示"视频文件不能超过500MB" |
| 文档附件上传超限 | 提示"文档文件不能超过50MB" |
| 网络断开时自动保存 | 草稿保留在localStorage，网络恢复后同步到服务端 |
| 并发编辑课程内容 | 采用最后写入优先策略，提示用户内容已被更新 |
| 视频播放中网络中断 | 缓存已加载部分继续播放，进度上报暂存后补报 |
| 课程状态定时任务失败 | 记录错误日志，下次定时任务执行时重试 |
| 克隆大型课程超时 | 异步执行克隆，完成后通知教师 |
