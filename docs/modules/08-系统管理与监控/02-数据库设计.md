# 系统管理与监控模块 — 数据库设计

> 模块状态：✅ 已确认
> 文档版本：v1.0

---

## 一、ER 关系总览

```
                    ┌─────────────────────────────────────────────┐
                    │           外部模块日志表（只读聚合查询）       │
                    │  login_logs (模块01)                        │
                    │  operation_logs (模块01)                    │
                    │  instance_operation_logs (模块04)           │
                    └─────────────────────────────────────────────┘

system_configs (系统配置)
  └── 1:N ── config_change_logs (配置变更记录)

alert_rules (告警规则)
  └── 1:N ── alert_events (告警事件)

platform_statistics (平台统计日表)

backup_records (备份记录)
```

## 二、表结构设计

### 2.1 system_configs — 系统配置表

> 存储平台全局配置，key-value结构，支持分组。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| config_group | VARCHAR(50) | NOT NULL | 配置分组 |
| config_key | VARCHAR(100) | NOT NULL | 配置键 |
| config_value | TEXT | NOT NULL | 配置值（JSON字符串或纯文本） |
| value_type | SMALLINT | NOT NULL DEFAULT 1 | 值类型：1字符串 2数字 3布尔 4JSON |
| description | VARCHAR(200) | NULL | 配置说明 |
| is_sensitive | BOOLEAN | NOT NULL DEFAULT FALSE | 是否敏感配置（敏感配置值不在API中明文返回） |
| updated_by | BIGINT | NULL | 最后修改人ID |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |

**索引：**
| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| uk_system_configs_group_key | (config_group, config_key) | 唯一 | 分组+键唯一 |
| idx_system_configs_group | config_group | 普通 | 按分组查询 |

**预置配置分组和键：**

| config_group | config_key | 默认值 | 说明 |
|-------------|-----------|--------|------|
| `platform` | `name` | "链镜" | 平台名称 |
| `platform` | `logo_url` | "" | 平台Logo URL |
| `platform` | `icp_number` | "" | ICP备案号 |
| `platform` | `copyright` | "" | 版权信息 |
| `platform` | `description` | "" | 平台描述 |
| `storage` | `default_school_quota_gb` | "100" | 学校默认存储配额(GB) |
| `storage` | `max_upload_size_mb` | "50" | 单文件最大上传大小(MB) |
| `security` | `session_timeout_hours` | "24" | 会话超时时间(小时) |
| `security` | `max_login_fail_count` | "5" | 最大登录失败次数 |
| `security` | `lock_duration_minutes` | "30" | 账号锁定时长(分钟) |
| `backup` | `auto_backup_enabled` | "true" | 是否启用自动备份 |
| `backup` | `auto_backup_cron` | "0 2 * * *" | 自动备份cron表达式 |
| `backup` | `backup_retention_count` | "30" | 备份保留份数 |

> **与模块01安全策略的关系：** 模块01的 `security_policies` 接口管理登录锁定等安全策略。模块08的 `system_configs` 中 `security` 分组存储的是**全局默认值**，模块01可以读取这些默认值作为初始配置。两者不冲突：模块01管理运行时安全策略，模块08管理全局默认配置。

### 2.2 config_change_logs — 配置变更记录表

> 记录所有系统配置的变更历史，不可删除。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| config_group | VARCHAR(50) | NOT NULL | 配置分组 |
| config_key | VARCHAR(100) | NOT NULL | 配置键 |
| old_value | TEXT | NULL | 变更前的值（新增时为NULL） |
| new_value | TEXT | NOT NULL | 变更后的值 |
| changed_by | BIGINT | NOT NULL | 操作人ID |
| changed_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 变更时间 |
| ip | VARCHAR(45) | NOT NULL | 操作人IP |

**索引：**
| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_config_change_logs_group_key | (config_group, config_key) | 普通 | 按配置项查询变更历史 |
| idx_config_change_logs_changed_by | changed_by | 普通 | 按操作人查询 |
| idx_config_change_logs_changed_at | changed_at | 普通 | 按时间查询 |

### 2.3 alert_rules — 告警规则表

> 定义系统告警规则。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| name | VARCHAR(100) | NOT NULL | 规则名称 |
| description | VARCHAR(500) | NULL | 规则描述 |
| alert_type | SMALLINT | NOT NULL | 告警类型：1阈值告警 2事件告警 3服务状态告警 |
| level | SMALLINT | NOT NULL DEFAULT 2 | 告警级别：1信息 2警告 3严重 4紧急 |
| condition | JSONB | NOT NULL | 告警条件配置 |
| silence_period | INT | NOT NULL DEFAULT 1800 | 静默期（秒），同一规则在此期间不重复告警 |
| is_enabled | BOOLEAN | NOT NULL DEFAULT TRUE | 是否启用 |
| created_by | BIGINT | NOT NULL | 创建人ID |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |
| deleted_at | TIMESTAMP | NULL | 软删除 |

**`condition` JSONB 结构：**

阈值告警：
```json
{
  "metric": "cpu_usage",
  "operator": ">",
  "value": 80,
  "duration": 300
}
```

事件告警：
```json
{
  "event_source": "login_logs",
  "event_filter": { "action": 2 },
  "group_by": "ip",
  "count_threshold": 10,
  "time_window": 600
}
```

服务状态告警：
```json
{
  "service_name": "postgresql",
  "check_url": "localhost:5432",
  "check_interval": 30,
  "fail_threshold": 3
}
```

**索引：**
| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_alert_rules_alert_type | alert_type | 普通 | 按类型筛选 |
| idx_alert_rules_is_enabled | is_enabled | 普通 | 查询启用的规则 |

**告警类型枚举：**
| 值 | 常量名 | 说明 |
|----|--------|------|
| 1 | AlertTypeThreshold | 阈值告警 |
| 2 | AlertTypeEvent | 事件告警 |
| 3 | AlertTypeService | 服务状态告警 |

**告警级别枚举：**
| 值 | 常量名 | 说明 |
|----|--------|------|
| 1 | AlertLevelInfo | 信息 |
| 2 | AlertLevelWarning | 警告 |
| 3 | AlertLevelCritical | 严重 |
| 4 | AlertLevelUrgent | 紧急 |

### 2.4 alert_events — 告警事件表

> 记录告警规则触发的告警事件。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| rule_id | BIGINT | NOT NULL | 关联告警规则ID |
| level | SMALLINT | NOT NULL | 告警级别（冗余，便于查询） |
| title | VARCHAR(200) | NOT NULL | 告警标题 |
| detail | JSONB | NOT NULL | 告警详情 |
| status | SMALLINT | NOT NULL DEFAULT 1 | 状态：1待处理 2已处理 3已忽略 |
| handled_by | BIGINT | NULL | 处理人ID |
| handled_at | TIMESTAMP | NULL | 处理时间 |
| handle_note | TEXT | NULL | 处理备注 |
| triggered_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 触发时间 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |

**`detail` JSONB 结构示例：**

阈值告警：
```json
{
  "metric": "cpu_usage",
  "current_value": 92.5,
  "threshold": 80,
  "duration_seconds": 320,
  "node": "k8s-node-01"
}
```

事件告警：
```json
{
  "event_source": "login_logs",
  "group_value": "192.168.1.100",
  "event_count": 15,
  "time_window": 600,
  "sample_events": [
    { "user_id": "...", "fail_reason": "密码错误", "created_at": "..." }
  ]
}
```

**索引：**
| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_alert_events_rule_id | rule_id | 普通 | 按规则查询 |
| idx_alert_events_level | level | 普通 | 按级别筛选 |
| idx_alert_events_status | status | 普通 | 按状态筛选 |
| idx_alert_events_triggered_at | triggered_at | 普通 | 按时间排序 |

**状态枚举：**
| 值 | 常量名 | 说明 |
|----|--------|------|
| 1 | AlertEventPending | 待处理 |
| 2 | AlertEventHandled | 已处理 |
| 3 | AlertEventIgnored | 已忽略 |

### 2.6 backup_records — 备份记录表

> 记录数据备份的历史。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| backup_type | SMALLINT | NOT NULL | 备份类型：1自动 2手动 |
| status | SMALLINT | NOT NULL DEFAULT 1 | 状态：1进行中 2成功 3失败 |
| file_path | VARCHAR(500) | NULL | 备份文件路径（MinIO） |
| file_size | BIGINT | NULL | 文件大小（字节） |
| database_name | VARCHAR(100) | NOT NULL | 数据库名称 |
| started_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 开始时间 |
| completed_at | TIMESTAMP | NULL | 完成时间 |
| error_message | TEXT | NULL | 错误信息（失败时） |
| triggered_by | BIGINT | NULL | 触发人ID（手动备份时） |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |

**索引：**
| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_backup_records_status | status | 普通 | 按状态筛选 |
| idx_backup_records_started_at | started_at | 普通 | 按时间排序 |

**备份类型枚举：**
| 值 | 常量名 | 说明 |
|----|--------|------|
| 1 | BackupTypeAuto | 自动备份 |
| 2 | BackupTypeManual | 手动备份 |

**备份状态枚举：**
| 值 | 常量名 | 说明 |
|----|--------|------|
| 1 | BackupStatusRunning | 进行中 |
| 2 | BackupStatusSuccess | 成功 |

## 四、Redis 缓存设计

| Key 格式 | 类型 | TTL | 说明 |
|---------|------|-----|------|
| `system_config:{group}:{key}` | String | 1h | 系统配置缓存 |
| `alert_rule:last_trigger:{rule_id}` | String(timestamp) | 永久 | 告警规则最后触发时间（用于静默期判断） |
| `platform_stats:today` | Hash | 5min | 当日实时统计缓存 |
| `service_health:{service_name}` | String(JSON) | 1min | 服务健康状态缓存 |

## 五、定时任务

| 任务 | 执行频率 | 说明 |
|------|---------|------|
| 阈值告警检测 | 每分钟 | 检查CPU/内存/磁盘等指标是否超过阈值 |
| 事件告警检测 | 每5分钟 | 扫描日志表检查异常事件（如连续登录失败） |
| 服务健康检查 | 每30秒 | 检查各服务（PostgreSQL、Redis、NATS、MinIO）可达性 |
| 平台统计聚合 | 每日凌晨1:00 | 聚合前一天的平台使用数据，写入 `platform_statistics` |
| 自动备份 | 按配置（默认每日2:00） | 执行PostgreSQL全量备份，上传到MinIO |
| 备份清理 | 每日凌晨3:00 | 删除超出保留份数的旧备份 |
| 统计数据清理 | 每月1日 | 清理超过365天的统计数据 |
