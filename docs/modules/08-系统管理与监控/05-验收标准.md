# 系统管理与监控模块 — 验收标准

> 模块状态：✅ 已确认
> 文档版本：v1.0

---

## 一、功能验收条件

### AC-01：统一审计 — 聚合查询

```
Given 超级管理员进入统一审计中心
When 不选择日志来源（查询全部）
Then 系统并行查询 login_logs、operation_logs、instance_operation_logs
And 结果按时间倒序合并展示
And 每条记录标注来源（登录日志/操作日志/实验操作日志）
```

```
Given 超级管理员选择"登录日志"来源
When 输入IP地址筛选
Then 仅查询 login_logs 表，按IP过滤
And 结果显示登录成功/失败/登出等操作
```

```
Given 超级管理员选择"实验操作日志"来源
When 搜索关键词"docker"
Then 仅查询 instance_operation_logs 表
And 匹配 command 字段包含"docker"的记录
```

### AC-02：审计日志导出

```
Given 超级管理员设置了筛选条件
When 点击"导出"
Then 系统根据当前筛选条件生成Excel文件并下载
And 文件包含所有匹配记录（不受分页限制，但有最大导出量限制10万条）
```

### AC-03：全局配置管理

```
Given 超级管理员进入全局配置页
When 修改平台名称为"链镜教学平台"并保存
Then 配置更新成功
And config_change_logs 记录变更（old: "链镜", new: "链镜教学平台"）
And Redis缓存刷新
And 前端页面标题立即更新
```

```
Given 超级管理员修改安全配置（最大登录失败次数改为10）
When 保存成功
Then 模块01的登录锁定逻辑使用新的阈值
```

### AC-04：配置变更记录

```
Given 超级管理员查看配置变更记录
When 筛选"platform"分组
Then 显示所有平台基本信息的变更历史
And 每条记录包含：变更时间、配置项、变更前值、变更后值、操作人
```

### AC-05：告警规则 — 创建阈值告警

```
Given 超级管理员创建告警规则：CPU > 80% 持续5分钟，级别"严重"
When 保存成功
Then 规则出现在告警规则列表中，状态为"启用"
And 告警检测定时任务开始监控该指标
```

### AC-06：告警规则 — 创建事件告警

```
Given 超级管理员创建告警规则：同一IP连续登录失败 > 10次/10分钟，级别"警告"
When 保存成功
Then 规则出现在告警规则列表中
And 事件告警检测定时任务开始扫描 login_logs
```

### AC-07：告警规则 — 启用/禁用

```
Given 某告警规则处于启用状态
When 超级管理员点击"禁用"
Then 规则状态变为"禁用"
And 该规则不再触发告警
```

### AC-08：告警触发

```
Given CPU使用率告警规则已启用（阈值80%，持续5分钟）
When K8s节点CPU使用率达到92%并持续超过5分钟
Then 系统创建告警事件，级别"严重"
And 通过模块07发送站内信通知超级管理员
And 告警事件出现在告警事件列表中
```

### AC-09：告警静默期

```
Given CPU告警已触发，静默期30分钟
When 30分钟内CPU使用率仍然超过阈值
Then 不重复创建告警事件
And 不重复发送通知
```

```
Given 静默期已过
When CPU使用率仍然超过阈值
Then 创建新的告警事件并通知
```

### AC-10：告警处理

```
Given 超级管理员查看待处理告警
When 填写处理备注"已扩容节点"并标记为"已处理"
Then 告警状态变为"已处理"
And 记录处理人和处理时间
```

```
Given 超级管理员查看低级别告警
When 点击"忽略"
Then 告警状态变为"已忽略"
```

### AC-11：运维仪表盘

```
Given 超级管理员进入运维仪表盘
When 页面加载完成
Then 显示各服务健康状态（绿色/黄色/红色）
And 显示CPU/内存/存储使用率进度条
And 显示在线用户数、活跃实验数、进行中竞赛数
And 显示最近告警列表
And 数据每30秒自动刷新
```

```
Given PostgreSQL服务不可达
When 健康检查执行
Then 仪表盘中PostgreSQL状态变为红色
And 如果有对应的服务状态告警规则，触发告警
```

### AC-12：平台使用统计

```
Given 超级管理员进入平台统计页
When 页面加载完成
Then 显示核心指标卡片（总用户、总学校、总课程、总实验、总竞赛）
And 显示活跃用户趋势图（默认最近30天）
And 显示学校活跃度排行
```

```
Given 超级管理员切换趋势周期为"90天"
When 数据加载完成
Then 趋势图显示最近90天的数据点
```

### AC-13：数据备份 — 手动触发

```
Given 超级管理员点击"手动备份"
When 确认后
Then 系统异步执行PostgreSQL全量备份
And 备份记录状态为"进行中"
And 备份完成后状态变为"成功"，记录文件大小和耗时
And 备份文件上传到MinIO
```

```
Given 备份执行过程中发生错误
When 备份失败
Then 备份记录状态变为"失败"
And 记录错误信息
And 管理员可查看错误详情
```

### AC-14：数据备份 — 自动备份

```
Given 自动备份已启用，配置为每日2:00执行
When 到达2:00
Then 系统自动执行备份
And 备份记录类型标记为"自动"
```

```
Given 备份保留份数配置为30
When 备份总数超过30份
Then 自动删除最旧的备份（MinIO文件 + 数据库记录）
```

### AC-15：数据备份 — 下载

```
Given 备份记录状态为"成功"
When 超级管理员点击"下载"
Then 从MinIO下载备份文件
```

## 二、跨模块联动验收

### AC-C01：审计中心聚合模块01日志

```
Given 用户在模块01登录成功
When 超级管理员在审计中心搜索该用户
Then 能查到来源为"登录日志"的登录成功记录
And 记录包含用户名、IP、登录方式、时间
```

### AC-C02：审计中心聚合模块04日志

```
Given 学生在模块04的实验环境中执行终端命令
When 超级管理员在审计中心搜索"terminal_command"
Then 能查到来源为"实验操作日志"的终端命令记录
And 记录包含学生名、实验实例ID、命令内容
```

### AC-C03：告警通知通过模块07

```
Given 告警规则触发
When 系统调用模块07的通知接口
Then 超级管理员在站内信收件箱收到告警通知
And 通知分类为"系统通知"
And 通知内容包含告警级别、规则名称、触发详情
```

### AC-C04：安全配置联动模块01

```
Given 超级管理员在模块08修改"最大登录失败次数"为10
When 用户连续登录失败
Then 模块01的登录锁定逻辑使用新阈值（第10次失败后锁定，而非默认的第5次）
```

### AC-C05：平台统计聚合各模块数据

```
Given 平台有10所学校、5000用户、150门课程
When 超级管理员查看平台统计总览
Then 各指标数值与实际数据一致
And 数据来源：users表(模块01)、schools表(模块02)、courses表(模块03)、experiment_instances表(模块04)、competitions表(模块05)
```

## 三、性能指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 审计日志查询（单来源） | ≤ 1s | 单表查询，单页20条 |
| 审计日志查询（全来源） | ≤ 3s | 三表并行查询+合并排序 |
| 审计日志导出 | ≤ 30s | 最大10万条 |
| 全局配置加载 | ≤ 500ms | Redis缓存 |
| 配置更新 | ≤ 500ms | 含变更记录写入+缓存刷新 |
| 告警规则列表 | ≤ 500ms | 单页20条 |
| 告警事件列表 | ≤ 500ms | 单页20条 |
| 阈值告警检测 | ≤ 10s | 每分钟执行一次 |
| 事件告警检测 | ≤ 30s | 每5分钟执行一次 |
| 服务健康检查 | ≤ 5s | 每30秒执行一次 |
| 运维仪表盘加载 | ≤ 2s | 含所有服务状态+资源使用 |
| 平台统计总览 | ≤ 2s | 含各模块COUNT查询 |
| 统计趋势数据 | ≤ 1s | 从platform_statistics表查询 |
| 备份触发响应 | ≤ 1s | 异步执行，立即返回 |
| 全量备份执行 | ≤ 30min | PostgreSQL pg_dump |

## 四、安全要求

| 要求 | 说明 |
|------|------|
| 仅超级管理员访问 | 模块08所有接口和页面仅超级管理员可访问 |
| 审计日志不可删除 | 审计日志只读，不提供删除接口 |
| 审计日志不可篡改 | 日志表不提供UPDATE接口 |
| 配置变更审计 | 所有配置变更自动记录到config_change_logs |
| 敏感配置脱敏 | 敏感配置值在API响应中脱敏显示 |
| 备份文件加密 | 备份文件在MinIO中加密存储 |
| 备份下载鉴权 | 备份文件下载需要超级管理员权限 |
| 告警规则权限 | 只有超级管理员可创建/修改/删除告警规则 |

## 五、边界条件与异常处理

| 场景 | 处理方式 |
|------|---------|
| 审计查询全来源数据量过大 | 强制要求时间范围不超过30天，建议选择单来源 |
| 审计导出超过10万条 | 提示"数据量过大，请缩小筛选范围" |
| 配置值类型不匹配 | 后端校验拒绝，提示"配置值类型错误" |
| 告警规则条件配置错误 | 后端校验JSONB结构，拒绝无效配置 |
| 告警检测服务异常 | 记录错误日志，下次定时任务重试 |
| 服务健康检查超时 | 标记该服务为"未知"状态，不立即标记为"不可达" |
| 连续3次健康检查失败 | 标记服务为"不可达"，触发告警 |
| 备份过程中磁盘空间不足 | 备份失败，记录错误信息，通知管理员 |
| 备份文件上传MinIO失败 | 备份标记为失败，保留本地临时文件，管理员可重试 |
| 统计聚合任务失败 | 记录错误日志，下次定时任务重试，不影响历史数据 |
| Redis缓存不可用 | 配置查询降级为直接读数据库 |
| 并发修改同一配置 | 乐观锁（基于updated_at），后提交者提示"配置已被修改，请刷新后重试" |
