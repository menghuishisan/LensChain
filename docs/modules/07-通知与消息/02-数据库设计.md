# 通知与消息模块 — 数据库设计

> 模块状态：✅ 已确认
> 文档版本：v1.0

---

## 一、ER 关系总览

```
notifications (站内信主表)
  └── 关联 users.id (receiver_id)
  └── 关联 notification_templates.event_type

system_announcements (系统公告)
  └── 关联 users.id (published_by)

notification_templates (消息模板)

user_notification_preferences (用户通知偏好)
  └── 关联 users.id
```

---

## 二、表结构设计

### 2.1 notifications — 站内信主表

> 存储所有用户的站内通知消息。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| receiver_id | BIGINT | NOT NULL | 接收者用户ID（引用 `users.id`） |
| school_id | BIGINT | NULL | 接收者所属学校ID（冗余，便于按学校统计） |
| category | SMALLINT | NOT NULL | 分类：1系统通知 2课程通知 3实验通知 4竞赛通知 5成绩通知 |
| event_type | VARCHAR(50) | NOT NULL | 事件类型编码（如 `assignment.published`） |
| title | VARCHAR(200) | NOT NULL | 通知标题 |
| content | TEXT | NOT NULL | 通知内容 |
| source_module | VARCHAR(20) | NOT NULL | 来源模块（如 `module_03`） |
| source_id | BIGINT | NULL | 来源对象ID（如课程ID、作业ID） |
| source_type | VARCHAR(50) | NULL | 来源对象类型（如 `course`、`assignment`） |
| is_read | BOOLEAN | NOT NULL DEFAULT FALSE | 是否已读 |
| read_at | TIMESTAMP | NULL | 阅读时间 |
| is_deleted | BOOLEAN | NOT NULL DEFAULT FALSE | 用户是否已删除（软删除） |
| deleted_at | TIMESTAMP | NULL | 用户删除时间 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |

**索引：**
| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_notifications_receiver_id | receiver_id | 普通 | 按接收者查询（核心查询） |
| idx_notifications_receiver_read | (receiver_id, is_read) | 普通 | 未读消息查询 |
| idx_notifications_receiver_category | (receiver_id, category) | 普通 | 按分类筛选 |
| idx_notifications_school_id | school_id | 普通 | 按学校统计 |
| idx_notifications_created_at | created_at | 普通 | 按时间排序、过期清理 |
| idx_notifications_event_type | event_type | 普通 | 按事件类型统计 |

**分类枚举：**
| 值 | 常量名 | 说明 |
|----|--------|------|
| 1 | CategorySystem | 系统通知 |
| 2 | CategoryCourse | 课程通知 |
| 3 | CategoryExperiment | 实验通知 |
| 4 | CategoryCompetition | 竞赛通知 |
| 5 | CategoryGrade | 成绩通知 |

> **设计说明：** `notifications` 表采用"每个接收者一条记录"的模式。当一条通知发给N个人时，会生成N条记录。这种设计虽然数据量较大，但查询简单、支持每个用户独立的已读/删除状态。通过定期归档清理（180天）控制数据量。

### 2.2 system_announcements — 系统公告表

> 平台级系统公告，全员可见，置顶显示。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| title | VARCHAR(200) | NOT NULL | 公告标题 |
| content | TEXT | NOT NULL | 公告内容（富文本） |
| published_by | BIGINT | NOT NULL | 发布者ID（超级管理员） |
| status | SMALLINT | NOT NULL DEFAULT 1 | 状态：1草稿 2已发布 3已下架 |
| is_pinned | BOOLEAN | NOT NULL DEFAULT TRUE | 是否置顶 |
| published_at | TIMESTAMP | NULL | 发布时间 |
| scheduled_at | TIMESTAMP | NULL | 定时发布时间（NULL=立即发布） |
| unpublished_at | TIMESTAMP | NULL | 下架时间 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |
| deleted_at | TIMESTAMP | NULL | 软删除 |

**索引：**
| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_system_announcements_status | status | 普通 | 按状态筛选 |
| idx_system_announcements_published_at | published_at | 普通 | 按发布时间排序 |

**状态枚举：**
| 值 | 常量名 | 说明 |
|----|--------|------|
| 1 | AnnouncementDraft | 草稿 |
| 2 | AnnouncementPublished | 已发布 |
| 3 | AnnouncementUnpublished | 已下架 |

### 2.3 notification_templates — 消息模板表

> 各类通知事件的消息模板，支持变量占位符。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| event_type | VARCHAR(50) | NOT NULL, UNIQUE | 事件类型编码（如 `assignment.published`） |
| category | SMALLINT | NOT NULL | 通知分类（同 notifications.category） |
| title_template | VARCHAR(200) | NOT NULL | 标题模板（支持 `{variable}` 占位符） |
| content_template | TEXT | NOT NULL | 内容模板（支持 `{variable}` 占位符） |
| variables | JSONB | NOT NULL | 可用变量列表及说明 |
| is_enabled | BOOLEAN | NOT NULL DEFAULT TRUE | 是否启用 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |

**`variables` JSONB 结构示例：**
```json
[
  { "name": "course_name", "description": "课程名称", "required": true },
  { "name": "assignment_name", "description": "作业名称", "required": true },
  { "name": "deadline", "description": "截止时间", "required": false }
]
```

**系统预置模板数据（部分示例）：**

| event_type | category | title_template | content_template |
|-----------|----------|----------------|------------------|
| `user.created` | 1 | 账号创建通知 | 您的账号已创建，初始密码已通过短信发送，请登录后修改密码。 |
| `assignment.published` | 2 | 新作业发布 | 课程《{course_name}》发布了新作业《{assignment_name}》，截止时间：{deadline}。 |
| `assignment.deadline_reminder` | 2 | 作业截止提醒 | 作业《{assignment_name}》将于{hours}小时后截止，请尽快提交。 |
| `assignment.graded` | 2 | 作业批改完成 | 您的作业《{assignment_name}》已批改，得分：{score}。 |
| `experiment.expiring` | 3 | 实验环境即将超时 | 您的实验环境将于{minutes}分钟后到期，请及时保存进度。 |
| `competition.published` | 4 | 新竞赛发布 | 新竞赛《{competition_name}》开始报名，报名截止：{deadline}。 |
| `grade.review_approved` | 5 | 成绩已发布 | 课程《{course_name}》{semester_name}成绩已发布，请查看。 |
| `grade.academic_warning` | 5 | 学业预警 | 您的GPA({gpa})低于预警线({threshold})，请关注学业表现。 |
| `system.maintenance` | 1 | 系统维护通知 | 平台将于{time}进行维护，预计持续{duration}，届时将暂停服务。 |

### 2.4 user_notification_preferences — 用户通知偏好表

> 用户可配置各分类通知的开启/关闭。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| user_id | BIGINT | NOT NULL | 用户ID |
| category | SMALLINT | NOT NULL | 通知分类 |
| is_enabled | BOOLEAN | NOT NULL DEFAULT TRUE | 是否接收该分类通知 |
| created_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 更新时间 |

**索引：**
| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| uk_user_notification_prefs | (user_id, category) | 唯一 | 每用户每分类一条记录 |

> **说明：** 系统通知（category=1）和成绩通知（category=5）强制接收，即使用户设置 `is_enabled=false`，后端仍会发送。

---

### 2.5 announcement_read_status — 公告阅读状态表

> 记录用户对系统公告的阅读状态。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGINT | PK | 雪花算法ID |
| announcement_id | BIGINT | NOT NULL | 公告ID |
| user_id | BIGINT | NOT NULL | 用户ID |
| read_at | TIMESTAMP | NOT NULL DEFAULT NOW() | 阅读时间 |

**索引：**
| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| uk_announcement_read | (announcement_id, user_id) | 唯一 | 每用户每公告一条记录 |
| idx_announcement_read_user | user_id | 普通 | 按用户查询已读公告 |

> **设计说明：** 系统公告不写入 `notifications` 表（避免全员广播产生海量记录），而是单独存储，通过 `announcement_read_status` 跟踪阅读状态。前端查询时合并系统公告和站内信。

## 三、与外部模块的数据依赖

### 3.1 读取的外部表（只读）

| 外部表 | 所属模块 | 读取字段 | 用途 |
|--------|---------|---------|------|
| `users` | 模块01 | id, name, school_id | 接收者信息 |

### 3.2 写入的外部表

无。模块07不写入其他模块的表。

### 3.3 被其他模块调用

各模块通过模块07的**内部通知接口**（NATS消息或直接函数调用）发送通知事件。模块07负责：
1. 查询消息模板
2. 渲染通知内容
3. 检查接收者通知偏好
4. 写入 `notifications` 表
5. 更新 Redis 未读计数
6. WebSocket 推送

## 四、Redis 缓存设计

| Key 格式 | 类型 | TTL | 说明 |
|---------|------|-----|------|
| `notification:unread:{user_id}` | String(int) | 永久 | 用户未读消息计数 |
| `notification:template:{event_type}` | String(JSON) | 1h | 消息模板缓存 |
| `notification:pref:{user_id}` | Hash | 1h | 用户通知偏好缓存（category → enabled） |
| `announcement:latest` | String(JSON) | 10min | 最新已发布系统公告列表缓存 |
| `notification:dedup:{event_type}:{receiver_id}:{source_id}` | String | 24h | 通知去重标记 |

## 五、定时任务

| 任务 | 执行频率 | 说明 |
|------|---------|------|
| 作业截止提醒 | 每小时 | 扫描24小时内即将截止的作业，向未提交学生发送提醒 |
| 竞赛报名截止提醒 | 每日 | 扫描48小时内即将截止报名的竞赛，向未报名学生发送提醒 |
| 竞赛开始提醒 | 每小时 | 扫描1小时内即将开始的竞赛，向已报名参赛者发送提醒 |
| 定时公告发布 | 每分钟 | 检查是否有到达 `scheduled_at` 时间的公告，自动发布 |
| 站内信归档清理 | 每日凌晨2:00 | 清理超过180天的已读站内信 |
| 未读计数校验 | 每日凌晨3:00 | 校验Redis未读计数与数据库的一致性 |
