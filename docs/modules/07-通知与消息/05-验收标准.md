# 通知与消息模块 — 验收标准

> 模块状态：✅ 已确认
> 文档版本：v1.0

---

## 一、功能验收条件

### AC-01：站内信收件箱

```
Given 用户已登录系统
When 进入消息中心页面
Then 显示所有站内通知，按时间倒序排列
And 未读消息有蓝色圆点标记和加粗字体
And 支持按分类Tab切换（系统/课程/实验/竞赛/成绩）
And 每个Tab显示该分类的消息数量
```

### AC-02：消息标记已读

```
Given 用户有未读消息
When 点击某条消息查看详情
Then 该消息自动标记为已读
And 未读计数减1
And 铃铛图标数字更新
```

```
Given 用户有多条未读消息
When 点击"全部标记已读"
Then 所有未读消息标记为已读
And 未读计数归零
```

### AC-03：消息删除

```
Given 用户查看站内信列表
When 删除某条消息
Then 该消息从列表中消失（软删除）
And 如果是未读消息，未读计数减1
```

```
Given 消息为系统公告
When 用户尝试删除
Then 系统拒绝，系统公告不可删除
```

### AC-04：未读消息实时推送

```
Given 用户在线（WebSocket已连接）
When 其他模块发送通知事件（如模块03发布新作业）
Then 用户的铃铛未读数实时+1
And 铃铛下拉面板显示新消息预览
```

```
Given 用户WebSocket断线
When 重新连接成功
Then 自动同步最新未读计数
```

### AC-05：系统公告发布

```
Given 超级管理员创建公告并点击"发布"
When 发布操作完成
Then 公告状态变为"已发布"
And 所有在线用户通过WebSocket收到新公告提醒
And 公告在所有用户的消息中心置顶显示
```

```
Given 超级管理员设置定时发布（scheduled_at）
When 到达定时发布时间
Then 系统自动发布公告
And 效果同立即发布
```

### AC-06：系统公告下架

```
Given 公告状态为"已发布"
When 超级管理员点击"下架"
Then 公告状态变为"已下架"
And 公告从用户的消息中心消失
```

### AC-07：定向通知发送

```
Given 学校管理员选择"全校用户"发送通知
When 填写标题和内容后点击"发送"
Then 本校所有用户的收件箱收到该通知
And 通知通过NATS异步处理，不阻塞发送接口
```

```
Given 教师选择"指定课程学生"发送通知
When 选择课程并填写内容后发送
Then 仅该课程的选课学生收到通知
```

### AC-08：通知偏好设置

```
Given 用户关闭"竞赛通知"偏好
When 模块05发送竞赛相关通知事件
Then 该用户不收到竞赛类站内信
And 其他用户正常收到
```

```
Given 用户尝试关闭"系统通知"或"成绩通知"
When 操作提交
Then 系统忽略该请求，强制分类不可关闭
```

### AC-09：消息模板渲染

```
Given 模板为"课程《{course_name}》发布了新作业《{assignment_name}》"
When 模块03发送事件 params={course_name:"区块链原理", assignment_name:"作业1"}
Then 生成的通知内容为"课程《区块链原理》发布了新作业《作业1》"
```

```
Given 模板被禁用（is_enabled=false）
When 对应事件触发
Then 不生成站内信
```

### AC-10：消息模板管理

```
Given 超级管理员编辑消息模板
When 修改标题模板和内容模板后保存
Then 模板更新成功
And 后续新生成的通知使用新模板
And 已发送的历史通知不受影响
```

```
Given 超级管理员预览模板
When 输入示例参数并点击"预览"
Then 显示渲染后的标题和内容
```

### AC-11：定时通知 — 作业截止提醒

```
Given 某作业将于24小时内截止
When 定时任务扫描到该作业
Then 向所有未提交该作业的学生发送截止提醒
And 每个学生每个作业只提醒一次（去重）
```

### AC-12：定时通知 — 竞赛提醒

```
Given 某竞赛报名将于48小时内截止
When 定时任务扫描到该竞赛
Then 向符合条件但未报名的学生发送报名提醒
```

```
Given 某竞赛将于1小时内开始
When 定时任务扫描到该竞赛
Then 向所有已报名参赛者发送开始提醒
```

### AC-13：通知去重

```
Given 作业截止提醒已发送给某学生
When 下一次定时任务再次扫描到同一作业
Then 不重复发送提醒给该学生
```

### AC-14：消息来源跳转

```
Given 用户收到作业发布通知（source_type=assignment）
When 在消息详情页点击"前往查看"
Then 跳转到模块03的作业详情页
```

```
Given 用户收到成绩发布通知（source_type=grade_review）
When 在消息详情页点击"前往查看"
Then 跳转到模块06的我的成绩页
```

## 二、跨模块联动验收

### AC-C01：模块03作业发布联动

```
Given 教师在模块03发布新作业
When 模块03调用模块07的内部通知接口
Then 课程所有选课学生收到"新作业发布"站内信
And 通知内容根据模板渲染，包含课程名和作业名
```

### AC-C02：模块04实验超时联动

```
Given 学生的实验环境即将超时（模块04检测到）
When 模块04调用模块07的内部通知接口
Then 该学生收到"实验环境即将超时"站内信
```

### AC-C03：模块05竞赛发布联动

```
Given 管理员在模块05发布新竞赛
When 模块05调用模块07的内部通知接口
Then 符合参赛条件的学生收到"新竞赛发布"站内信
```

### AC-C04：模块06成绩审核联动

```
Given 学校管理员在模块06审核通过某课程成绩
When 模块06调用模块07的内部通知接口
Then 课程教师收到"成绩审核已通过"站内信
And 课程所有学生收到"成绩已发布"站内信
```

### AC-C05：模块06学业预警联动

```
Given 模块06检测到学生GPA低于预警线
When 模块06调用模块07的内部通知接口
Then 该学生收到"学业预警"站内信
And 通知内容包含当前GPA和预警阈值
```

### AC-C06：模块08系统告警联动

```
Given 模块08检测到系统告警
When 模块08调用模块07的内部通知接口
Then 超级管理员收到"系统告警"站内信
```

## 三、性能指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 收件箱列表加载 | ≤ 500ms | 单页20条，含分类计数 |
| 未读计数查询 | ≤ 50ms | Redis缓存读取 |
| 消息标记已读 | ≤ 200ms | 含Redis计数更新 |
| 全部标记已读 | ≤ 1s | 批量更新 |
| WebSocket未读推送 | ≤ 500ms | 事件触发到用户端更新 |
| 单条通知生成 | ≤ 100ms | 模板渲染 + 写入DB + 更新Redis |
| 批量通知生成（50人） | ≤ 3s | 异步队列处理 |
| 批量通知生成（500人全校） | ≤ 30s | 异步队列处理 |
| 系统公告发布 | ≤ 1s | 含WebSocket广播 |
| 定时任务扫描 | ≤ 30s | 每小时扫描截止提醒 |
| 消息模板渲染 | ≤ 10ms | 单条模板变量替换 |
| 通知偏好查询 | ≤ 50ms | Redis缓存读取 |

## 四、安全要求

| 要求 | 说明 |
|------|------|
| 消息隔离 | 用户只能查看自己的站内信 |
| 发送权限控制 | 教师只能向自己课程的学生发送通知 |
| 学校隔离 | 学校管理员只能向本校用户发送通知 |
| 内部接口保护 | `POST /internal/send-event` 仅内部网络可访问，不对外暴露 |
| WebSocket认证 | WebSocket连接需携带JWT Token验证身份 |
| 模板注入防护 | 模板变量渲染时转义HTML特殊字符，防止XSS |
| 批量发送限流 | 单次定向通知最多发送给1000人，超过需分批 |
| 公告内容审查 | 系统公告内容过滤敏感词（可选） |

## 五、边界条件与异常处理

| 场景 | 处理方式 |
|------|---------|
| 接收者不存在 | 跳过该接收者，记录警告日志 |
| 消息模板不存在 | 使用默认模板（事件类型 + 原始参数） |
| 模板变量缺失 | 缺失的变量显示为空字符串，记录警告日志 |
| NATS消息队列不可用 | 降级为同步处理，记录错误日志 |
| WebSocket推送失败 | 用户下次打开页面时通过HTTP接口同步 |
| Redis未读计数不一致 | 每日凌晨定时任务校验并修正 |
| 大量通知并发写入 | NATS队列缓冲，消费者按批次处理 |
| 站内信归档清理失败 | 记录错误日志，下次定时任务重试 |
| 定时公告发布失败 | 记录错误日志，管理员可手动发布 |
| 用户通知偏好未配置 | 默认所有分类开启 |
