## 二、页面详细设计

### C-01 通知铃铛（全局组件）

**布局：**
```
┌──────────────────────────────────────────────────────────┐
│ Logo  链镜平台    课程  实验  竞赛    🔔(3)  👤 张三    │
└──────────────────────────────────────┬───────────────────┘
                                       │ 点击展开
                                       ▼
                              ┌────────────────────┐
                              │ 通知              │
                              ├────────────────────┤
                              │ 🔵 新作业发布      │
                              │   课程《区块链原理》│
                              │   10分钟前         │
                              ├────────────────────┤
                              │ 🔵 成绩已发布      │
                              │   课程《智能合约》  │
                              │   1小时前          │
                              ├────────────────────┤
                              │    系统维护通知     │
                              │   平台将于4/15...  │
                              │   昨天             │
                              ├────────────────────┤
                              │   查看全部 →       │
                              │   全部标记已读      │
                              └────────────────────┘
```

**交互：**
- 铃铛图标右上角显示未读数（红色圆点+数字）
- 未读数 > 99 时显示 "99+"
- 点击铃铛展开下拉面板，显示最近5条消息
- 未读消息前有蓝色圆点标记
- 点击消息跳转到消息详情页
- "查看全部"跳转到消息中心
- "全部标记已读"一键清零
- 新消息到达时铃铛抖动动画 + 未读数更新（WebSocket驱动）

## 三、全局交互规范

### 3.1 WebSocket连接管理

- 用户登录后自动建立通知WebSocket连接
- 与模块04（实验）和模块05（竞赛）的WebSocket连接独立
- 断线自动重连（指数退避，最大间隔30秒）
- 重连后同步最新未读计数

### 3.2 通知铃铛更新机制

- 初始加载：调用 `GET /inbox/unread-count` 获取未读数
- 实时更新：通过WebSocket接收 `unread_count_update` 消息
- 用户操作（标记已读/删除）后立即更新本地计数

### 3.3 消息来源跳转映射

| source_type | 跳转目标 | 说明 |
|-------------|---------|------|
| `assignment` | 模块03 作业详情页 | 查看作业内容或批改结果 |
| `experiment` | 模块04 实验详情页 | 查看实验或评分结果 |
| `competition` | 模块05 竞赛详情页 | 查看竞赛信息 |
| `grade_review` | 模块06 我的成绩页 | 查看成绩 |
| `grade_appeal` | 模块06 成绩申诉页 | 查看申诉结果 |
| `academic_warning` | 模块06 GPA总览页 | 查看GPA详情 |
| 无source | 不显示跳转按钮 | 纯文本通知 |

### 3.4 响应式布局

- 桌面端：消息中心为独立页面，铃铛下拉为预览面板
- 移动端：消息中心全屏，铃铛点击直接进入消息中心
- 消息列表支持下拉刷新和无限滚动
