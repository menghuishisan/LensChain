## 三、全局交互规范

### 3.1 数据来源标注

由于模块06的成绩数据来源于模块03，页面上需要明确标注数据来源：
- 各项作业/实验成绩来自模块03的成绩计算结果
- 加权总成绩来自模块03的 `course_grade_configs` 权重配置
- 等级和绩点由模块06根据学校等级映射配置计算

### 3.2 状态颜色规范

| 状态 | 颜色 | 场景 |
|------|------|------|
| 已审核/已通过 | 绿色 ✅ | 成绩已审核确认 |
| 待审核/待处理 | 黄色 🟡 / 红色 🔴 | 等待处理的审核/申诉/预警 |
| 未提交 | 灰色 ⚪ | 教师尚未提交审核 |
| 已驳回 | 红色 ❌ | 审核驳回/申诉驳回 |
| 已调整 | 蓝色 🔵 | 成绩经过手动调整 |

### 3.3 权限控制

- 学生只能查看自己的成绩和GPA
- 教师只能查看自己课程的学生成绩
- 学校管理员可查看本校所有数据
- 超级管理员可查看全平台数据
- 所有数据按 `school_id` 严格隔离

### 3.4 与模块03页面的关系

- 模块03的"成绩管理"页面（`/teacher/courses/:id/grades`）负责**单课程内**的成绩权重配置、批改、手动调整、Excel导出
- 模块06的页面负责**跨课程**的成绩汇总、审核、GPA、申诉
- 教师在模块03完成成绩计算后，到模块06提交审核
- 学生在模块03查看单课程详细成绩，在模块06查看跨课程汇总和GPA
