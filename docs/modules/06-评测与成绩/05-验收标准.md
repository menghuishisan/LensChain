# 评测与成绩模块 — 验收标准

> 模块状态：✅ 已确认
> 文档版本：v1.0

---

## 一、功能验收条件

### AC-01：学校管理员配置等级映射

```
Given 学校管理员已登录系统
When 配置等级映射（A:90-100→4.0, B:80-89→3.0, C:70-79→2.0, D:60-69→1.0, F:0-59→0）并保存
Then 配置保存成功，后续GPA计算使用新配置
```

```
Given 学校管理员配置的分数区间有重叠（如 A:85-100, B:80-90）
When 点击保存
Then 系统拒绝，提示"分数区间不可重叠"
```

```
Given 学校管理员配置的分数区间未覆盖全范围（缺少0-59区间）
When 点击保存
Then 系统拒绝，提示"分数区间必须覆盖0-100全范围"
```

### AC-02：学期管理

```
Given 学校管理员已登录系统
When 创建学期（名称、编码、起止日期）
Then 学期创建成功，出现在学期列表中
```

```
Given 学校已有学期编码"2025-2026-1"
When 再次创建相同编码的学期
Then 系统拒绝，提示"学期编码已存在"
```

```
Given 学校管理员将某学期设为当前学期
When 操作完成
Then 该学期标记为当前学期，其他学期取消当前标记
```

### AC-03：教师提交成绩审核

```
Given 教师的课程所有学生成绩已计算完成
When 教师点击"提交审核"并填写备注
Then 审核记录创建，状态为"待审核"
And 学校管理员收到站内信通知（通过模块07）
```

```
Given 课程中有学生成绩未计算完成（模块03中未全部批改）
When 教师尝试提交审核
Then 系统拒绝，提示"课程成绩未全部计算完成，尚有X名学生未出成绩"
```

```
Given 该课程该学期已有待审核或已通过的审核记录
When 教师再次提交
Then 系统拒绝，提示"该课程本学期已有审核记录"
```

### AC-04：学校管理员审核通过

```
Given 审核记录状态为"待审核"
When 学校管理员查看成绩汇总和分布后点击"通过"
Then 审核状态变为"已通过"，成绩锁定
And 系统从模块03读取各学生加权总成绩
And 根据等级映射配置计算等级和绩点
And 批量写入 student_semester_grades 表
And 更新所有相关学生的GPA缓存
And 触发学业预警检测
And 教师收到站内信通知"审核已通过"（通过模块07）
```

### AC-05：学校管理员审核驳回

```
Given 审核记录状态为"待审核"
When 学校管理员填写驳回原因并点击"驳回"
Then 审核状态变为"已驳回"
And 教师收到站内信通知"审核已驳回"及驳回原因（通过模块07）
And 教师可在模块03修改成绩后重新提交
```

### AC-06：成绩锁定与解锁

```
Given 成绩审核已通过，成绩已锁定
When 教师尝试在模块03修改该课程的学生成绩
Then 模块03拒绝修改，提示"成绩已锁定，如需修改请联系学校管理员解锁"
```

> **跨模块联动说明：** 此验收条件要求模块03在成绩修改接口中检查模块06的锁定状态。实现方式：模块03调用模块06的内部接口查询锁定状态，或通过 `grade_reviews.is_locked` 字段直接查询。

```
Given 学校管理员对已锁定的成绩执行解锁
When 填写解锁原因并确认
Then 成绩解锁，审核状态回退为"未提交"
And 教师可在模块03修改成绩后重新提交审核
And 操作记录写入 operation_logs（模块01）
```

### AC-07：学生查看学期成绩

```
Given 学生已登录系统
When 进入"我的成绩"页面
Then 显示当前学期所有课程的成绩列表
And 每门课程显示：课程名、学分、最终成绩、等级、绩点
And 已审核的成绩标记为"已审核"，未审核的标记为"待审核"
And 底部显示本学期GPA和总学分
```

### AC-08：GPA计算

```
Given 学生有3门已审核通过的课程成绩：
  - 课程A：3学分，绩点4.0
  - 课程B：2学分，绩点3.0
  - 课程C：3学分，绩点2.0
When 系统计算学期GPA
Then GPA = (4.0×3 + 3.0×2 + 2.0×3) / (3+2+3) = 24/8 = 3.00
```

```
Given 学生有多个学期的已审核成绩
When 系统计算累计GPA
Then 累计GPA = 所有学期的 Σ(绩点×学分) / Σ(学分)
And 仅计算审核状态为"已通过"的成绩
```

### AC-09：成绩申诉提交

```
Given 学生的某课程成绩已审核通过，且在30天申诉期内
When 学生填写申诉理由（≥20字）并提交
Then 申诉记录创建，状态为"待处理"
And 课程教师收到站内信通知（通过模块07）
```

```
Given 成绩审核通过已超过30天
When 学生尝试申诉
Then 系统拒绝，提示"已超过申诉时效（30天）"
```

```
Given 学生对该课程该学期已有申诉记录
When 学生再次申诉
Then 系统拒绝，提示"每门课程每学期最多申诉1次"
```

### AC-10：教师处理申诉 — 同意

```
Given 教师查看学生的成绩申诉
When 选择"同意修改"，填写新成绩和处理意见
Then 申诉状态变为"已同意"
And student_semester_grades 中的成绩更新为新成绩
And 等级和绩点根据等级映射重新计算
And 该学生的学期GPA和累计GPA重新计算
And 检查学业预警是否需要解除
And 学生收到站内信通知"申诉已处理"（通过模块07）
```

### AC-11：教师处理申诉 — 驳回

```
Given 教师查看学生的成绩申诉
When 选择"驳回"并填写驳回理由
Then 申诉状态变为"已驳回"
And 成绩不变
And 学生收到站内信通知"申诉已驳回"及驳回理由（通过模块07）
```

### AC-12：学业预警自动触发

```
Given 某学校预警配置：GPA阈值2.0，连续挂科阈值2门
When 某学期成绩审核完成后，学生张三的学期GPA为1.85
Then 系统自动生成低GPA预警记录
And 预警详情包含当前GPA和挂科课程列表
And 学生收到站内信通知"您的GPA低于预警线"（通过模块07）
```

```
Given 学生在连续2门课程中成绩低于60分（等级F）
When 学业预警检测执行
Then 系统自动生成连续挂科预警记录
```

### AC-13：学业预警处理

```
Given 学校管理员查看预警列表
When 选择某条预警并填写处理备注"已约谈学生"
Then 预警状态变为"已处理"
```

```
Given 学生GPA回升到预警阈值以上
When 下一次预警检测执行
Then 该学生的预警自动解除，状态变为"已解除"
```

### AC-14：成绩单生成

```
Given 学生选择要包含的学期
When 点击"生成成绩单"
Then 系统生成PDF文件，包含学校名称/Logo、学生信息、各学期成绩表、GPA汇总
And 仅包含已审核通过的成绩
And PDF文件上传到MinIO，返回下载链接
```

### AC-15：成绩分析

```
Given 教师查看课程成绩分析
When 进入分析页面
Then 显示：学生数、平均分、中位数、最高分、最低分、及格率
And 显示成绩分布柱状图和等级分布饼图
```

```
Given 学校管理员查看全校成绩分析
When 选择某学期
Then 显示：全校平均GPA、GPA分布、不及格率、预警人数
And 显示课程排行（最高/最低平均分课程）
```

## 二、跨模块联动验收

> 以下验收条件涉及多个模块的协同工作，需要在集成测试中验证。

### AC-C01：模块03成绩锁定联动

```
Given 模块06中某课程成绩审核已通过（is_locked=true）
When 教师在模块03尝试修改该课程学生的成绩（调用 PATCH /api/v1/courses/:id/grades/:student_id）
Then 模块03返回错误"成绩已锁定"
```

### AC-C02：模块04实验成绩回传链路

```
Given 学生完成实验并获得评分（模块04）
And 实验成绩通过模块04 BR-50自动回传到模块03的课程成绩
When 教师在模块06提交该课程的成绩审核
Then 审核通过后，student_semester_grades 中的 final_score 已包含实验成绩的加权贡献
```

### AC-C03：模块07通知联动

```
Given 成绩审核通过
When 系统通过模块07发送通知
Then 教师在站内信收件箱看到"课程《XXX》成绩审核已通过"
And 学生在站内信收件箱看到"课程《XXX》成绩已发布"
```

### AC-C04：模块05 CTF成绩隔离

```
Given 学生参加了CTF竞赛并获得排名
When 系统计算该学生的GPA
Then CTF竞赛成绩不参与GPA计算
And GPA仅基于课程成绩
```

## 三、性能指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 学期成绩列表加载 | ≤ 500ms | 单学期，含GPA计算 |
| GPA计算 | ≤ 200ms | 单个学生的累计GPA |
| 成绩审核通过处理 | ≤ 5s | 含批量写入student_semester_grades + GPA更新 + 预警检测（50人课程） |
| 成绩审核列表加载 | ≤ 500ms | 单学期，单页20条 |
| 等级映射配置保存 | ≤ 500ms | 含校验逻辑 |
| 成绩单PDF生成 | ≤ 10s | 单个学生，含3个学期数据 |
| 批量成绩单生成 | ≤ 60s | 50人班级 |
| 成绩分析（课程级） | ≤ 1s | 50人课程 |
| 成绩分析（全校级） | ≤ 3s | 500人学校 |
| 学业预警检测 | ≤ 30s | 500人学校全量检测 |
| 申诉处理（含GPA重算） | ≤ 3s | 单个学生 |

## 四、安全要求

| 要求 | 说明 |
|------|------|
| 多租户隔离 | 所有数据按 school_id 隔离，学校管理员只能查看本校数据 |
| 学生数据隔离 | 学生只能查看自己的成绩和GPA |
| 教师数据隔离 | 教师只能查看自己课程的学生成绩 |
| 成绩锁定不可绕过 | 锁定后的成绩修改必须经过解锁流程，后端强制校验 |
| 审核操作审计 | 所有审核通过/驳回/解锁操作记录到 operation_logs |
| 申诉时效强制 | 后端校验申诉时效，不可通过前端绕过 |
| 成绩单防伪 | PDF成绩单包含生成时间戳和唯一编号 |
| GPA计算一致性 | GPA计算结果与缓存保持一致，定时任务每日校验 |

## 五、边界条件与异常处理

| 场景 | 处理方式 |
|------|---------|
| 课程无学分信息 | 提交审核时提示"请先在课程设置中配置学分" |
| 课程未关联学期 | 提交审核时提示"请先关联课程所属学期" |
| 等级映射未配置 | 使用系统默认五级制（A/B/C/D/F） |
| 审核通过后教师修改成绩 | 模块03拒绝修改，提示"成绩已锁定" |
| 申诉修改成绩后GPA缓存不一致 | 申诉处理时同步刷新缓存，定时任务兜底校验 |
| 学生退课后成绩处理 | 退课学生的成绩不参与GPA计算 |
| 成绩单生成失败 | 返回错误信息，用户可重试 |
| 预警检测中发现数据异常 | 记录错误日志，跳过异常学生，继续检测其他学生 |
| 并发提交审核 | 数据库唯一索引保证同一课程同一学期只有一条审核记录 |
| Redis缓存与数据库GPA不一致 | 每日凌晨定时任务全量校验并修正 |
