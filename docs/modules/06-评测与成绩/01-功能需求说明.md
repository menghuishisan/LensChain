# 评测与成绩模块 — 功能需求说明

> 模块状态：✅ 已确认
> 文档版本：v1.0

## 一、模块概述

### 1.1 模块定位

评测与成绩模块是**跨课程成绩聚合层**，负责将模块03（课程与教学）中各课程的成绩数据进行汇总、审核、分析和管理。

**核心职责：**
- 跨课程成绩汇总与GPA计算
- 成绩审核工作流（教师提交 → 学校管理员审核）
- 成绩单生成（PDF）
- 成绩申诉（学生 → 教师）
- 成绩分析与学业预警

### 1.2 模块边界

| 本模块负责 | 不属于本模块（归属说明） |
|-----------|----------------------|
| 跨课程成绩汇总与GPA计算 | 单课程内成绩权重配置与自动计算（→ 模块03 `course_grade_configs`） |
| 成绩审核工作流 | 作业批改与评分（→ 模块03 `assignment_submissions`） |
| 成绩单PDF生成 | 单课程成绩Excel导出（→ 模块03 `GET /courses/:id/grades/export`） |
| 成绩申诉处理 | 实验检查点评分（→ 模块04 `checkpoint_results`） |
| 学业预警 | CTF竞赛计分（→ 模块05，独立体系，不纳入GPA） |
| 成绩分析与统计 | 实验成绩回传课程体系（→ 模块04 BR-25/BR-50 自动完成） |
| 等级映射配置（学校级） | 课程统计与数据导出（→ 模块03） |

### 1.3 与其他模块的联动关系

| 关联模块 | 联动方式 | 数据流向 | 具体说明 |
|---------|---------|---------|---------|
| 模块01 用户与认证 | 读取 | 01 → 06 | 读取 `users` 表获取学生信息（姓名、学号、学校ID）；成绩归属用户 |
| 模块02 学校与租户管理 | 读取 | 02 → 06 | 读取 `schools` 表获取学校信息；等级映射配置按学校隔离 |
| 模块03 课程与教学 | 读取 | 03 → 06 | **核心数据源**：读取 `courses`（课程信息、学分）、`course_grade_configs`（权重配置）、`assignment_submissions`（各项成绩）、`course_enrollments`（选课关系）计算加权总成绩和GPA |
| 模块04 实验环境 | 间接（经模块03） | 04 → 03 → 06 | 实验成绩通过模块04的 BR-50 自动回传到模块03的课程成绩体系，模块06从模块03读取已包含实验成绩的汇总数据 |
| 模块05 CTF竞赛 | **无联动** | — | CTF竞赛是独立子系统（模块05 BR-01），不与课程关联，不回传成绩，不纳入GPA计算 |
| 模块07 通知与消息 | 发送 | 06 → 07 | 成绩审核通过/驳回、申诉处理结果、学业预警 通过模块07的通知接口发送站内信 |
| 模块08 系统管理与监控 | 被聚合 | 06 → 08 | 成绩审核操作记录写入 `operation_logs`，模块08统一审计中心可查询 |

### 1.4 依赖关系

```
模块06 依赖：
  → 01-用户与认证（用户数据）
  → 02-学校与租户管理（学校数据、租户隔离）
  → 03-课程与教学（课程成绩数据 —— 核心数据源）
  → 04-实验环境（间接，实验成绩经模块03回传）

模块06 被依赖：
  ← 07-通知与消息（成绩相关通知事件）
  ← 08-系统管理与监控（操作审计）
```

## 三、用户故事

### US-01：学校管理员配置等级映射

```
作为 学校管理员
我想要 配置本校的百分制到等级和绩点的映射规则
以便于 系统能按本校标准计算学生的GPA

验收条件：
1. 可配置多个等级区间（如 A: 90-100 → 4.0, B: 80-89 → 3.0 ...）
2. 分数区间不可重叠，必须覆盖0-100全范围
3. 保存后立即生效，影响后续的GPA计算
4. 提供默认模板（五级制 A/B/C/D/F），可基于模板修改
```

### US-02：教师提交成绩审核

```
作为 教师
我想要 将课程最终成绩提交给学校管理员审核
以便于 成绩经过审核后正式生效

验收条件：
1. 提交前系统检查：所有学生的成绩是否已计算完成
2. 提交时可附加说明（如特殊情况备注）
3. 提交后成绩进入"待审核"状态，教师不可再修改
4. 学校管理员审核通过后成绩锁定
5. 学校管理员驳回后退回教师，教师可修改后重新提交
```

### US-03：学校管理员审核成绩

```
作为 学校管理员
我想要 审核教师提交的课程成绩
以便于 确保成绩的准确性和规范性

验收条件：
1. 可查看课程成绩汇总表（各项成绩、加权总成绩、等级、绩点）
2. 可查看成绩分布图（正态分布检查）
3. 审核通过：成绩锁定，学生可查看最终成绩，GPA更新
4. 审核驳回：填写驳回原因，退回教师修改
```

### US-04：学生查看GPA

```
作为 学生
我想要 查看我的学期GPA和累计GPA
以便于 了解自己的学业表现

验收条件：
1. 显示当前学期所有课程的成绩、等级、绩点、学分
2. 显示当前学期GPA = Σ(课程绩点 × 学分) / Σ(学分)
3. 显示累计GPA（所有已审核通过学期的加权平均）
4. 成绩状态标记：已审核（正式）/ 待审核（暂定）
```

### US-05：学生申诉成绩

```
作为 学生
我想要 对某课程成绩提出申诉
以便于 在成绩有误时得到纠正

验收条件：
1. 只能对已审核通过（锁定）的成绩申诉
2. 申诉需填写申诉理由（必填，至少20字）
3. 申诉提交后通知对应课程教师
4. 教师可同意修改（填写新成绩和修改原因）或驳回（填写驳回理由）
5. 同意修改后成绩更新，GPA重新计算
6. 每门课程每学期最多申诉1次
```

### US-06：学业预警

```
作为 学校管理员
我想要 系统自动识别学业困难的学生
以便于 及时干预和帮助

验收条件：
1. 可配置预警规则：GPA阈值（默认 < 2.0）、连续挂科门数（默认 ≥ 2）
2. 每学期成绩审核完成后自动触发预警检测
3. 预警列表显示：学生信息、当前GPA、触发原因
4. 预警通知发送给学生本人（通过模块07站内信）
5. 管理员可标记预警为"已处理"并填写处理备注
```

### US-07：成绩单生成

```
作为 学生/教师/学校管理员
我想要 生成正式的成绩单PDF
以便于 用于学业证明

验收条件：
1. 成绩单包含：学校名称和Logo、学生基本信息、各学期课程成绩表、GPA汇总
2. 仅包含已审核通过的成绩
3. 学生可下载自己的成绩单
4. 教师可下载本课程学生的成绩单
5. 学校管理员可批量生成某班级/年级的成绩单
```

## 四、业务规则

| 编号 | 规则 | 说明 |
|------|------|------|
| BR-01 | GPA计算公式 | GPA = Σ(课程绩点 × 学分) / Σ(学分)，仅计算已审核通过的成绩 |
| BR-02 | 默认等级映射 | A(90-100, 4.0)、B(80-89, 3.0)、C(70-79, 2.0)、D(60-69, 1.0)、F(<60, 0)，学校可自定义 |
| BR-03 | 等级区间约束 | 分数区间不可重叠，必须覆盖0-100全范围，绩点值0.0-4.0 |
| BR-04 | 成绩审核流程 | 教师提交 → 学校管理员审核 → 通过则锁定 / 驳回则退回教师 |
| BR-05 | 成绩锁定 | 审核通过后成绩锁定，教师不可修改；仅学校管理员可解锁 |
| BR-06 | 解锁后重新审核 | 管理员解锁成绩后，教师修改完需重新提交审核 |
| BR-07 | CTF不纳入GPA | CTF竞赛成绩独立（模块05 BR-01），不参与GPA计算 |
| BR-08 | 实验成绩经模块03 | 实验成绩通过模块04 BR-50自动回传到模块03课程成绩体系，模块06从模块03读取 |
| BR-09 | 成绩申诉时效 | 成绩审核通过后30天内可申诉 |
| BR-10 | 申诉次数限制 | 每门课程每学期最多申诉1次 |
| BR-11 | 申诉处理 | 学生提交 → 教师处理（同意修改/驳回），无需多级审批 |
| BR-12 | 申诉修改后GPA更新 | 教师同意修改成绩后，系统自动重新计算该学生的学期GPA和累计GPA |
| BR-13 | 学业预警触发 | 每学期成绩审核完成后自动检测，GPA < 阈值 或 连续挂科 ≥ 阈值 |
| BR-14 | 预警阈值可配置 | 学校管理员可配置GPA预警线（默认2.0）和连续挂科预警线（默认2门） |
| BR-15 | 成绩单仅含已审核成绩 | PDF成绩单只包含审核通过的正式成绩 |
| BR-16 | 学分来源 | 课程学分在 `courses` 表中定义（需模块03新增 `credits` 字段），模块06读取 |
| BR-17 | 学期概念 | 学期由学校管理员定义（如 2025-2026第一学期），课程关联学期 |
| BR-18 | 多租户隔离 | 所有成绩数据按 `school_id` 隔离，学校管理员只能查看本校数据 |
| BR-19 | 单课程能力边界 | 单课程成绩汇总、手动调整、Excel导出、课程统计仍归模块03，模块06只做跨课程聚合与审核 |

## 五、状态流转

### 5.1 成绩审核状态

```
未提交 ──[教师提交审核]──→ 待审核 ──[管理员通过]──→ 已通过（锁定）
                              │                         │
                              │                    [管理员解锁]
                              │                         │
                              ←──[管理员驳回]──→ 已驳回    ←─────┘
                                                  │
                                            [教师修改后重新提交]
                                                  │
                                                  → 待审核
```

状态枚举：
| 值 | 状态 | 说明 |
|----|------|------|
| 1 | 未提交 | 教师尚未提交审核 |
| 2 | 待审核 | 已提交，等待学校管理员审核 |
| 3 | 已通过 | 审核通过，成绩锁定 |
| 4 | 已驳回 | 审核驳回，退回教师修改 |

### 5.2 成绩申诉状态

```
[学生提交] → 待处理 ──[教师同意]──→ 已同意（成绩已修改）
                       │
                  [教师驳回]──→ 已驳回
```

状态枚举：
| 值 | 状态 | 说明 |
|----|------|------|
| 1 | 待处理 | 学生已提交，等待教师处理 |
| 2 | 已同意 | 教师同意修改成绩 |
| 3 | 已驳回 | 教师驳回申诉 |

### 5.3 学业预警状态

| 值 | 状态 | 说明 |
|----|------|------|
| 1 | 待处理 | 系统自动生成，等待管理员处理 |
| 2 | 已处理 | 管理员已处理并填写备注 |
| 3 | 已解除 | 学生GPA回升，预警自动解除 |

## 六、与模块03的数据衔接说明

模块06的成绩数据**完全来源于模块03**，不重复存储原始成绩，而是通过读取模块03的数据进行聚合计算：

| 模块03数据 | 模块06用途 |
|-----------|-----------|
| `courses.id/title/school_id/teacher_id/status` | 课程基本信息，用于成绩汇总展示 |
| `courses.credits`（需新增字段） | 学分，用于GPA加权计算 |
| `courses.semester_id` | 关联学期ID，确定课程所属学期 |
| `course_grade_configs.config` | 权重配置，用于展示成绩构成 |
| `assignment_submissions.score_after_deduction` | 各项作业/实验的最终得分 |
| `course_enrollments.student_id` | 选课关系，确定学生选了哪些课程 |

**模块03需要配合的改动：**
- `courses` 表新增 `credits DECIMAL(3,1) NULL` 字段（课程学分，如 3.0、1.5）
- `courses` 表新增 `semester_id BIGINT NULL` 字段（关联模块06的 `semesters.id`）

> 注意：这两个字段的新增需要在模块03的数据库设计文档中补充说明。

**职责统一说明：**
- 模块03负责单课程成绩汇总、手动调整、Excel导出与课程统计
- 模块06负责跨课程成绩审核、GPA、申诉、成绩单PDF与学业预警
