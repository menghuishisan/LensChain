# 链镜 —— 区块链教学与实践平台

> 文档驱动开发（Documentation-Driven Development）
> 一切开发以文档和规范为引擎，模块文档确认无误后方可进入开发阶段。

---

## 一、项目概述

**链镜** 是一个覆盖高校场景的区块链综合教学平台，集 **教学、实验实践、CTF竞赛** 三位一体，支持多链生态、多层次学生、多学校混合部署。

### 核心特性

| 特性 | 说明 |
|------|------|
| 教学+实践一体化 | 课程学习与实验环境深度融合，学完即练、练中学习 |
| CTF竞赛系统 | 支持解题赛、攻防对抗赛、抢占赛、实时对战赛四种赛制 |
| 多链全覆盖 | 以太坊/EVM系、Hyperledger Fabric、国产链（长安链/FISCO BCOS）等 |
| 动态实验环境 | Docker + K8s 多镜像动态编排，按实验类型自动组合环境 |
| 全层次支撑 | 从零基础本科生到研究生，分层学习路径 |
| 混合部署 | 支持私有化部署到各学校 + 公网统一运营 |

## 三、用户角色

| 角色 | 说明 |
|------|------|
| 超级管理员 | 平台级管理，管理学校租户、全局配置、系统监控 |
| 学校管理员 | 管理本校教师和学生、SSO配置、本校数据（由教师兼任） |
| 教师 | 创建课程、设计实验、出题、组织和管理竞赛 |
| 学生 | 学习课程、完成实验、参加竞赛 |

### 账户体系

- **不开放注册**，仅支持管理员/教师批量导入或手动添加
- 登录方式：手机号 + 密码 / 学校SSO统一认证（CAS + OAuth2.0）
- 手机号作为全局唯一标识，一人一校严格限制

## 六、文档规范

### 6.1 每个模块文档必须包含以下5个文件

| 文件 | 内容 |
|------|------|
| `01-功能需求说明.md` | 模块概述、功能清单、用户故事、业务规则、状态流转 |
| `02-数据库设计.md` | 表结构、字段说明、索引设计、ER关联关系 |
| `03-API接口设计.md` | RESTful路径、请求/响应结构、权限要求 |
| `04-前端页面设计.md` | 页面清单、布局描述、交互流程、组件结构 |
| `05-验收标准.md` | Given-When-Then验收条件、性能指标、安全要求、边界处理 |

### 6.2 文档状态标记

| 标记 | 含义 |
|------|------|
| ⚪ 未开始 | 尚未开始讨论 |
| 🔵 待讨论 | 正在讨论细化中 |
| 🟡 待确认 | 文档已完成，等待最终确认 |
| ✅ 已确认 | 确认无误，可进入开发 |
| 🔄 需修改 | 确认过程中发现问题，需要修改 |
