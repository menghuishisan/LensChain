# SimEngine

`sim-engine/` 是 LensChain 的可视化仿真引擎子系统，用于把区块链中的节点、消息、结构、交易、共识过程、攻击行为和经济模型，以可交互、可观察的方式呈现给学习者。

它既可以作为平台实验模块的一部分使用，也可以作为一个独立的仿真能力子系统进行理解和扩展。

## SimEngine 能做什么

- 运行内置区块链仿真场景
- 将场景状态实时推送到前端渲染层
- 支持播放、暂停、单步、变速、重置与定点回退
- 支持多场景联动观察
- 支持教师监控摘要与预览
- 支持教师扩展自定义场景
- 支持在内置领域之外扩展自定义渲染器
- 支持混合实验的数据采集与可视化接入

## 目录结构

```text

├── core/          # SimEngine Core 运行时服务
├── renderers/     # 前端领域渲染器
├── scenarios/     # 内置仿真场景
├── sdk/           # 自定义场景开发包
├── proto/         # gRPC 与协议定义
├── buf.yaml
├── buf.gen.yaml

```

## 各目录职责

### `core/`

负责会话管理、场景调度、时间控制、联动状态、快照恢复、消息推送和采集接入，是仿真引擎的运行时核心。

### `renderers/`

负责前端绘制与交互，把标准场景状态转换为画布上的节点、连线、结构和动画表现。

### `scenarios/`

负责内置仿真场景的算法实现与状态推进逻辑。每个场景输出统一的状态、事件和渲染数据。

### `sdk/`

用于开发自定义场景。通过 SDK，可以定义场景元信息、初始化逻辑、推进逻辑、交互处理和渲染输出。

### `proto/`

定义 Core 与场景容器之间的 gRPC 协议，以及仿真引擎对外使用的共享契约。

## 工作方式

SimEngine 的基本运行链路如下：

```text
前端渲染层
  ↕ WebSocket
SimEngine Core
  ↕ gRPC
场景算法容器
```

典型流程是：

1. Core 启动仿真会话并调度场景
2. 场景输出标准状态与事件
3. Core 通过 WebSocket 向前端推送状态帧
4. 前端根据 `category` 选择对应领域渲染器
5. 用户交互再通过标准消息回传到 Core 和场景

在混合实验模式下，Core 还可以接入实验环境中的采集数据，将真实数据与仿真状态统一推送到前端。

## 统一协议约束

当前 SimEngine 在外部可见层面遵循以下统一约束：

- 运行时领域使用字符串 `category` 标识
- 平台内置场景使用 8 个标准领域
- 自定义场景在运行时允许扩展新的领域字符串
- 定点回退统一通过 `rewind_to` 消息并传递 `target_tick`
- `process` 模式使用 `play`
- `continuous` 模式使用 `resume`

如果你在扩展场景或渲染器，建议保持这些约束不变，避免出现额外兼容分支。

## 自定义场景

如果你想增加新的仿真场景，通常有两种方式：

- 在 `scenarios/` 中增加平台内置场景
- 基于 `sdk/` 开发自定义场景，并接入自己的场景容器

一个场景通常需要提供：

- 场景元信息
- 默认状态
- 初始化逻辑
- 单步推进逻辑
- 交互处理逻辑
- 渲染状态输出

## 自定义渲染器

如果你的场景属于现有的 8 个标准领域，通常可以直接复用已有渲染器。

如果你的场景属于全新领域，则可以提供对应的自定义渲染器。渲染器扩展的核心是：

- 声明新的领域编码
- 实现该领域的状态绘制逻辑
- 处理交互事件
- 支持状态插值与过渡
